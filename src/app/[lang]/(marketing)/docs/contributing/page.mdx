## Quick Start

### 1. Fork & Clone

**01. Clone your fork**
```bash
git clone https://github.com/YOUR_USERNAME/hogwarts.git
```

---

**02. Navigate to project**
```bash
cd hogwarts
```

---

**03. Add upstream remote**
```bash
git remote add upstream https://github.com/databayt/hogwarts.git
```

### 2. Create Feature Branch

**01. Create new branch**
```bash
git checkout -b feature/your-feature-name
```

Branch naming conventions: `feature/`, `fix/`, `docs/`, `refactor/`, `test/`, `chore/`

### 3. Make Changes

Follow these critical standards:

**Multi-Tenant Safety**

Every database query MUST include schoolId:

```typescript
// CORRECT
await db.student.findMany({
  where: { schoolId, grade: "10" }
})

// WRONG - Missing schoolId
await db.student.findMany({
  where: { grade: "10" }
})
```

---

**Typography System**

Use semantic HTML only:

```tsx
// CORRECT
<h2>Page Title</h2>
<p>Body text</p>

// WRONG
<div className="text-2xl">Title</div>
```

### 4. Validate Changes

**01. TypeScript check**
```bash
pnpm tsc --noEmit
```
Must pass with 0 errors.

---

**02. Lint check**
```bash
pnpm lint
```
Fix any linting issues.

---

**03. Run tests**
```bash
pnpm test
```
All tests must pass.

---

**04. Build check**
```bash
pnpm build
```
Verify build succeeds.

### 5. Commit Changes

**01. Stage changes**
```bash
git add .
```

---

**02. Commit with message**
```bash
git commit -m "feat: add student attendance report"
```

Commit format: `feat:`, `fix:`, `docs:`, `style:`, `refactor:`, `test:`, `chore:`

### 6. Push & Create PR

**01. Push to your fork**
```bash
git push origin feature/your-feature-name
```

Then go to GitHub and click "Compare & pull request".

## PR Checklist

- [ ] TypeScript passes (`pnpm tsc --noEmit`)
- [ ] All tests pass (`pnpm test`)
- [ ] Build succeeds (`pnpm build`)
- [ ] Database queries include `schoolId`
- [ ] Used semantic HTML (no hardcoded styles)

## Server Actions Pattern

```typescript
"use server"

import { auth } from "@/auth"
import { db } from "@/lib/db"
import { revalidatePath } from "next/cache"

export async function createItem(data: FormData) {
  // 1. Authenticate
  const session = await auth()
  const schoolId = session?.user?.schoolId
  if (!schoolId) throw new Error("Unauthorized")

  // 2. Validate
  const validated = schema.parse(Object.fromEntries(data))

  // 3. Execute with schoolId
  const item = await db.item.create({
    data: { ...validated, schoolId }
  })

  // 4. Revalidate
  revalidatePath("/items")
  return { success: true, item }
}
```

## Common Pitfalls

- **Missing schoolId** - Always include in database queries
- **Hardcoded text styles** - Use semantic HTML elements
- **Skipping TypeScript checks** - Always run `pnpm tsc --noEmit`
- **Large PRs** - Keep PRs focused and small (< 400 lines)
