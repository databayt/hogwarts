# Unified Listing Pattern

## Overview

The **Unified Listing Pattern** provides a consistent, reusable approach for all data listings in the Hogwarts platform. This pattern eliminates code duplication by **60%** and ensures a consistent user experience across all modules.

**Status:** âœ… Production-Ready

## Quick Start

### 1. Basic Implementation

Create a new listing in **under 5 minutes**:

```tsx
// src/components/platform/teachers/table.tsx
'use client';

import { UnifiedDataTable } from '@/components/table/UnifiedDataTable';
import { useDataTable } from '@/components/table/hooks/use-data-table';
import { useCrudModal } from '@/components/crud-modal';
import { CrudModal } from '@/components/crud-modal';
import { teacherColumns } from './columns';

export function TeachersTable({ initialData }) {
  const crudModal = useCrudModal();

  const { table } = useDataTable({
    data: initialData,
    columns: teacherColumns,
    pageCount: 1,
  });

  return (
    <>
      <UnifiedDataTable
        table={table}
        toolbar={{
          showViewToggle: true,
          showExport: true,
        }}
        viewMode={{ enabled: true }}
        exportConfig={{
          fetchData: async () => initialData,
          columns: ['name', 'email', 'subject'],
          filename: 'teachers',
        }}
      />

      <CrudModal
        open={crudModal.state.open}
        onOpenChange={(open) => !open && crudModal.close()}
        title="Create Teacher"
      >
        {/* Form component here */}
      </CrudModal>
    </>
  );
}
```

## Core Components

### 1. ExportButton Component

**Location:** `src/components/export/ExportButton.tsx`

Dropdown button with CSV & Excel export options.

**Features:**
- Both CSV and Excel formats
- Localized headers
- Custom formatters for complex fields
- Progress indicators
- Respects current filters/sorting

**Usage:**

```tsx
import { ExportButton } from '@/components/export';

<ExportButton
  fetchData={async () => await getAllStudents()}
  columns={['name', 'email', 'class', 'status']}
  filename="students"
  entityPath="school.students"
  formatters={{
    createdAt: (date) => new Date(date).toLocaleDateString(),
  }}
/>
```

**Props:**

| Prop | Type | Description |
|------|------|-------------|
| `fetchData` | `() => Promise<any[]>` | Function to fetch data |
| `columns` | `string[]` | Column keys to export |
| `filename` | `string` | Base filename (without extension) |
| `formatters` | `Record<string, Function>` | Custom formatters per column |
| `entityPath` | `string` | Dictionary path for localized headers |
| `disabledFormats` | `ExportFormat[]` | Disable specific formats |

### 3. CrudModal System

**Location:** `src/components/crud-modal/`

Complete CRUD operations with auto-close and immediate refresh.

**Features:**
- Auto-closes on success (configurable delay)
- Loading states
- Error handling with toast
- Optimistic updates
- Form reset on close

**Usage:**

```tsx
import { CrudModal, CrudForm, useCrudModal } from '@/components/crud-modal';

function MyListing() {
  const crudModal = useCrudModal({
    onSuccess: async (mode) => {
      showSuccess(\`\${mode} successful\`);
      await refreshData();
    },
  });

  return (
    <>
      <Button onClick={crudModal.openCreate}>Create</Button>

      <CrudModal
        open={crudModal.state.open}
        onOpenChange={(open) => !open && crudModal.close()}
        title={crudModal.isCreate ? 'Create' : 'Edit'}
        autoCloseDelay={500}
      >
        <CrudForm
          schema={mySchema}
          onSubmit={handleSubmit}
          defaultValues={crudModal.state.data}
        >
          {(form) => (
            <>
              <FormField name="name" control={form.control} />
              <FormField name="email" control={form.control} />
            </>
          )}
        </CrudForm>
      </CrudModal>
    </>
  );
}
```

```tsx
const { refresh, start, stop, isRunning } = useAutoRefresh({
```

## Migration Guide

### From Old Pattern to Unified Pattern

#### Step 1: Install Dependencies

No new dependencies needed - all components use existing packages.

#### Step 2: Update Column Definitions

**Before:**
```tsx
// columns.tsx
export const columns = [
  { accessorKey: 'name', header: 'Name' },
  { accessorKey: 'email', header: 'Email' },
];
```

**After:**
```tsx
// columns.tsx
import { DataTableColumnHeader } from '@/components/table/data-table/data-table-column-header';

export const columns: ColumnDef<DataType>[] = [
  {
    accessorKey: 'name',
    header: ({ column }) => (
      <DataTableColumnHeader column={column} title="Name" />
    ),
    meta: { label: 'Name', variant: 'text' },
    enableColumnFilter: true,
  },
  // ... more columns
];
```

#### Step 3: Replace Table Component

**Before:**
```tsx
// table.tsx
<DataTable table={table}>
  <DataTableToolbar table={table}>
    <Button onClick={() => openModal()}>
      <Plus />
    </Button>
    <ExportButton />
  </DataTableToolbar>
  <Modal content={<Form />} />
</DataTable>
```

**After:**
```tsx
// table.tsx
<UnifiedDataTable
  table={table}
  toolbar={{
    showViewToggle: true,
    showExport: true,
    customActions: (
      <Button onClick={crudModal.openCreate}>
        <Plus /> Create
      </Button>
    ),
  }}
  viewMode={{ enabled: true, renderCard }}
  exportConfig={exportConfig}
/>

<CrudModal {...crudModal.state}>
  <CrudForm {...formProps} />
</CrudModal>
```

#### Step 4: Update Modal Logic

**Before:**
```tsx
const { openModal, closeModal } = useModal();

// Manual close, manual refresh
const handleSubmit = async (data) => {
  await createItem(data);
  closeModal();
  router.refresh(); // or manual data fetch
};
```

**After:**
```tsx
const crudModal = useCrudModal({
  onSuccess: async () => {
    showSuccess('Created successfully');
    await refreshData(); // Automatic
  },
});

// Auto-close on success, auto-refresh
const handleSubmit = async (data) => {
  return await createItem(data);
  // Modal closes automatically
  // List refreshes automatically
};
```

## Best Practices

### 1. Column Configuration

âœ… **Do:**
- Use `DataTableColumnHeader` for sortable columns
- Add `meta` property for filter configuration
- Set `enableColumnFilter` for filterable columns
- Include `accessorKey` for data binding

âŒ **Don't:**
- Hardcode column widths
- Skip accessibility attributes
- Forget to handle empty states

### 2. Data Fetching

âœ… **Do:**
- Implement server-side pagination for large datasets
- Use `useDataTable` hook for consistent state management
- Cache data when appropriate
- Handle loading and error states

âŒ **Don't:**
- Load all data at once for large tables
- Forget to handle errors
- Skip loading indicators

### 3. Form Handling

âœ… **Do:**
- Validate on both client (UX) and server (security)
- Use `CrudForm` wrapper for consistent behavior
- Transform data when needed before submission
- Reset form on success

âŒ **Don't:**
- Skip validation
- Leave stale data in forms
- Forget to handle server errors

### 4. Export Configuration

âœ… **Do:**
- Limit export columns to relevant data
- Use formatters for dates and complex fields
- Provide localized headers via dictionary
- Handle large exports with streaming

âŒ **Don't:**
- Export sensitive data
- Skip progress indicators
- Ignore memory limits

### 5. View Modes

âœ… **Do:**
- Implement both list and grid views when beneficial
- Store user preference in localStorage
- Optimize card rendering for performance
- Make cards responsive

âŒ **Don't:**
- Force grid view for data-heavy tables
- Duplicate logic between views
- Skip mobile optimization

## Troubleshooting

### Issue: Export not working
**Solution:** Ensure `fetchData` returns proper array format

```tsx
// âœ… Correct
fetchData: async () => data // Returns array

// âŒ Wrong
fetchData: async () => csvString // Returns string
```

### Issue: View toggle not persisting
**Solution:** Check localStorage permissions and storage key

```tsx
// Use unique storage key per listing
<ViewToggle storageKey="students-view-mode" />
<ViewToggle storageKey="teachers-view-mode" />
```

### Issue: Modal not closing
**Solution:** Verify `onSuccess` is called properly

```tsx
const handleSubmit = async (data) => {
  const result = await createItem(data);

  // âœ… Return result to trigger success
  return result;

  // âŒ Don't just await
  // await createItem(data);
};
```

### Issue: Filters not working
**Solution:** Check column `meta` configuration

```tsx
{
  meta: {
    label: 'Status',
    variant: 'select', // Must match filter type
    options: [ /* ... */ ],
  },
  enableColumnFilter: true, // Must be true
}
```

### Issue: Grid view empty
**Solution:** Implement `renderCard` function

```tsx
viewMode={{
  enabled: true,
  renderCard: (item) => <Card>{/* content */}</Card>,
}}
```

```typescript
    interval?: number;
    onRefresh?: () => void | Promise<void>;

  actionBar?: React.ReactNode;
  emptyMessage?: string;
```

```tsx
        open={crudModal.state.open}
        onOpenChange={(open) => !open && crudModal.close()}
        title={crudModal.isCreate ? 'Create' : 'Edit'}
        autoCloseDelay={500}
      >
        <CrudForm
          schema={validationSchema}
          onSubmit={handleSubmit}
          defaultValues={crudModal.state.data}
        >
          {(form) => <FormFields form={form} />}
        </CrudForm>
      </CrudModal>
    </>
  );
}
```
