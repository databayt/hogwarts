### Prerequisites
```bash
# Required tools
Node.js 18+
pnpm 9.x
PostgreSQL 14+
Git 2.x
```

### Setup Development Environment

1. **Fork the Repository**
```bash
# Fork on GitHub, then clone
git clone https://github.com/YOUR_USERNAME/hogwarts.git
cd hogwarts
```

2. **Install Dependencies**
```bash
pnpm install
```

3. **Setup Database**
```bash
# Copy environment template
cp .env.example .env

# Update DATABASE_URL in .env
# Run migrations
pnpm prisma migrate dev

# Seed sample data
pnpm db:seed
```

4. **Start Development Server**
```bash
pnpm dev
# Open http://localhost:3000
```

## ğŸ”„ Development Workflow

### 1. Choose an Issue

Browse issues labeled:
- `good-first-issue` - Perfect for beginners
- `help-wanted` - Community help needed
- `hacktoberfest` - Hacktoberfest eligible
- `bounty` - Paid contributions

### 2. Create a Branch

```bash
# Create feature branch
git checkout -b feature/issue-123-add-amazing-feature

# Or bug fix branch
git checkout -b fix/issue-456-fix-attendance-bug
```

### 3. Make Changes

Follow our coding standards:
- **TypeScript**: Use strict types, no `any`
- **React**: Functional components with hooks
- **Styling**: Tailwind CSS with cn() utility
- **Testing**: Write tests for new code
- **Commits**: Use conventional commits

### 4. Commit Your Changes

```bash
# Stage changes
git add .

# Commit with conventional message
git commit -m "feat: add student bulk import feature

- Add CSV parser for student data
- Implement validation logic
- Add progress indicator
- Update documentation

Fixes #123"
```

Commit message format:
- `feat:` New feature
- `fix:` Bug fix
- `docs:` Documentation
- `style:` Formatting
- `refactor:` Code restructuring
- `test:` Testing
- `chore:` Maintenance

### 5. Push & Create PR

```bash
# Push to your fork
git push origin feature/issue-123-add-amazing-feature
```

PR checklist:
- [ ] Tests pass locally
- [ ] Documentation updated
- [ ] Follows code style
- [ ] Linked to issue
- [ ] Screenshots for UI changes
- [ ] Performance impact considered

### 6. Code Review Process

Your PR will be reviewed for:
- **Functionality**: Does it work as expected?
- **Code Quality**: Is it maintainable?
- **Performance**: No regressions?
- **Security**: No vulnerabilities?
- **Tests**: Adequate coverage?
- **Documentation**: Clear and complete?

### Component Structure
```typescript
// Follow our mirror pattern
src/
  app/[feature]/page.tsx      // Route
  components/[feature]/
    content.tsx               // UI composition
    actions.ts               // Server actions
    validation.ts            // Zod schemas
    types.ts                // TypeScript types
```

### Multi-Tenant Safety
```typescript
// Always include schoolId
const result = await db.model.findMany({
  where: {
    schoolId: session.user.schoolId // CRITICAL
  }
})
```

### Server Actions Pattern
```typescript
"use server"

export async function createItem(data: FormData) {
  const session = await auth()
  const validated = schema.parse(data)

  await db.model.create({
    data: { ...validated, schoolId: session.user.schoolId }
  })

  revalidatePath('/items')
}
```
