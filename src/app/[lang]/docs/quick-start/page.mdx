# Quick Start Guide

Get your Hogwarts school management system up and running in just 5 minutes! This guide will walk you through the essential steps to deploy and configure your first school.

## Prerequisites

Before you begin, ensure you have:

- **Node.js 18+** installed on your system
- **PostgreSQL database** (we recommend [Neon](https://neon.tech) for quick setup)
- **pnpm** package manager (`npm install -g pnpm`)
- **Git** for version control

## üöÄ 5-Minute Setup

### Step 1: Clone the Repository

```bash
git clone https://github.com/your-org/hogwarts.git
cd hogwarts
```

### Step 2: Install Dependencies

```bash
pnpm install
```

### Step 3: Configure Environment

Create a `.env.local` file in the root directory:

```bash
cp .env.example .env.local
```

Update the following essential variables:

```env
# Database (PostgreSQL)
DATABASE_URL="postgresql://user:password@host:5432/dbname"

# Authentication
AUTH_SECRET="your-secret-key" # Generate with: openssl rand -base64 32

# Application URL
NEXT_PUBLIC_APP_URL="http://localhost:3000"

# Email Service (Resend)
RESEND_API_KEY="re_your_api_key"

# Optional: OAuth Providers
GOOGLE_CLIENT_ID="your-google-client-id"
GOOGLE_CLIENT_SECRET="your-google-client-secret"
```

### Step 4: Setup Database

Run Prisma migrations to create your database schema:

```bash
# Generate Prisma client
pnpm prisma generate

# Run migrations
pnpm prisma migrate dev

# Seed with sample data (optional)
pnpm db:seed
```

### Step 5: Start Development Server

```bash
pnpm dev
```

Your application is now running at `http://localhost:3000` üéâ

## üìù First School Setup

### 1. Create Admin Account

Navigate to `http://localhost:3000/join` and create your admin account:

- Enter your email and password
- Verify your email (check inbox)
- Log in with your credentials

### 2. School Onboarding

After logging in, you'll be guided through the school setup wizard:

1. **Basic Information**
   - School name
   - School type (Primary, Secondary, etc.)
   - Contact details

2. **Academic Configuration**
   - Academic year setup
   - Terms/semesters
   - Grading system

3. **Branding**
   - Upload school logo
   - Set color theme
   - Configure subdomain

4. **Initial Data**
   - Import students (CSV/Excel)
   - Add teachers
   - Create classes

## üåê Production Deployment

### Deploy to Vercel (Recommended)

1. Push your code to GitHub
2. Connect your repository to Vercel
3. Configure environment variables
4. Deploy with one click

```bash
# Install Vercel CLI
npm i -g vercel

# Deploy
vercel
```

### Custom Domain Setup

1. Add your domain in Vercel dashboard
2. Configure DNS records
3. Enable SSL (automatic)
4. Set up subdomains for multi-tenant access

1. **Use Demo Data**: Run `pnpm db:seed` to populate with sample data for testing
2. **Enable Debug Mode**: Set `DEBUG=true` in `.env.local` for detailed logs

## üÜò Troubleshooting

### Common Issues

<details>
<summary>Database connection failed</summary>

Check your `DATABASE_URL` format:
- Ensure PostgreSQL is running
- Verify credentials
- Check network connectivity
- For Neon, ensure project is active

</details>

<details>
<summary>Prisma client errors</summary>

Regenerate Prisma client:
```bash
pnpm prisma generate
pnpm prisma migrate reset # Warning: This resets your database
```

</details>

<details>
<summary>OAuth login not working</summary>

Verify OAuth configuration:
- Check callback URLs match your domain
- Ensure OAuth app is approved/live
- Verify client ID and secret
- Check redirect URI settings

</details>

<details>
<summary>Email not sending</summary>

Check email service:
- Verify Resend API key
- Check sender domain verification
- Review email templates
- Check spam folder

</details>
