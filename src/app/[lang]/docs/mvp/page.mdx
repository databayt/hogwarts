## Core Capabilities

### Foundation & Security

<div className="rounded-lg border bg-card p-6 my-6">
  <h4>Multi-Tenant Architecture</h4>
  <p className="muted mb-4">Enterprise-grade tenant isolation with subdomain-based routing ensures complete data separation between schools.</p>

  <ul className="space-y-2 text-sm text-muted-foreground">
    <li>✅ Each school operates on its own subdomain (e.g., <code>schoolname.databayt.org</code>)</li>
    <li>✅ Complete data isolation with mandatory <code>schoolId</code> scoping on all queries</li>
    <li>✅ Zero cross-tenant data leaks (verified through automated test suite)</li>
    <li>✅ Automatic subdomain provisioning during onboarding</li>
  </ul>
</div>

<div className="rounded-lg border bg-card p-6 my-6">

### Implementation Philosophy

<div className="grid gap-6 md:grid-cols-2 my-8">
  <div className="rounded-lg border bg-card p-6">
    <h4>Vertical Slicing</h4>
    <p className="muted">Each story delivers end-to-end value from database to UI, ensuring deployable increments at every step. No horizontal layer stories that don't provide user value.</p>
  </div>

  <div className="rounded-lg border bg-card p-6">
    <h4>No Forward Dependencies</h4>
    <p className="muted">Stories only depend on previous epics, never future ones. This ensures linear progress without blocking dependencies or rework.</p>
  </div>

  <div className="rounded-lg border bg-card p-6">
    <h4>Multi-Tenant Safety</h4>
    <p className="muted">Every story includes tenant isolation verification with automated tests ensuring no cross-school data leaks.</p>
  </div>

  <div className="rounded-lg border bg-card p-6">
    <h4>Deployable Incrementally</h4>
    <p className="muted">Each epic completion results in a deployable increment that provides value to schools, enabling early feedback and validation.</p>
  </div>
</div>
