The **Principal Dashboard** provides school principals with a comprehensive view of institutional performance, critical alerts, staff management, and executive-level metrics.

## Target Users

- **PRINCIPAL**: School principals and headmasters

## Layout Structure

The principal dashboard follows a **multi-section layout**:

```tsx
<div className="space-y-6">
  {/* Hero Section - Performance Scorecard */}
  <PrincipalDashboardStats ... />

  {/* Critical Alerts (conditional) */}
  {criticalAlerts.length > 0 && <AlertsCard />}

  {/* Quick Actions */}
  <QuickActions actions={...} locale={locale} />

  {/* Main Content Grid (2 columns) */}
  <div className="grid gap-6 md:grid-cols-2">
    <TodaysPrioritiesCard />
    <AcademicPerformanceCard />
    <DisciplinarySummaryCard />
    <StaffEvaluationsCard />
    <BudgetStatusCard />
    <ParentFeedbackCard />
  </div>

  {/* Executive Summary */}
  <ExecutiveSummaryCard />

  {/* Board Meetings */}
  <BoardMeetingsCard />

  {/* Financial Health */}
  <FinancialHealthCard />
</div>
```

## Performance Scorecard

Hero section with 5 key metrics using `PrincipalDashboardStats`:

| Metric              | Description                     | Visual       |
| ------------------- | ------------------------------- | ------------ |
| Overall Score       | Composite school performance    | Large number |
| Academic Score      | Academic achievement percentage | Progress bar |
| Attendance          | Average attendance rate         | Progress bar |
| Discipline Score    | Behavioral compliance           | Progress bar |
| Parent Satisfaction | Survey results                  | Progress bar |

```tsx
<PrincipalDashboardStats
  overallScore={performanceScorecard.overall}
  academicScore={performanceScorecard.academic}
  attendance={performanceScorecard.attendance}
  disciplineScore={performanceScorecard.discipline}
  parentSatisfaction={performanceScorecard.parentSatisfaction}
  dictionary={t.performanceScorecard}
/>
```

## Critical Alerts Section

Conditional display when alerts exist:

```tsx
{
  criticalAlerts.length > 0 && (
    <Card>
      <CardHeader>
        <CardTitle className="flex items-center space-x-2">
          <TriangleAlert className="text-destructive h-5 w-5" />
          <span>Critical Alerts</span>
        </CardTitle>
      </CardHeader>
      <CardContent>
        {criticalAlerts.map((alert) => (
          <AlertRow
            type={alert.type}
            message={alert.message}
            severity={alert.severity}
            action={alert.action}
          />
        ))}
      </CardContent>
    </Card>
  )
}
```

Severity levels:

- **critical** / **high**: Red destructive badge
- **medium** / **low**: Amber warning badge

## Academic Performance Trends

Subject-wise performance tracking:

| Field           | Display                          |
| --------------- | -------------------------------- |
| Subject         | Subject name                     |
| Current Average | Percentage                       |
| Trend           | Arrow indicator (up/down/stable) |
| Improvement     | Change percentage                |

Trend badges:

- **up**: Default badge with ↗
- **down**: Destructive badge with ↘
- **stable**: Secondary badge with →

## Data Fetching

Uses `getPrincipalDashboardData()` from `actions/principal.ts`:

```tsx
const dashboardData = await getPrincipalDashboardData()

const {
  performanceScorecard,
  criticalAlerts,
  todaysPriorities,
  academicTrends,
  disciplinarySummary,
  staffEvaluations,
  budgetStatus,
  parentFeedback,
  goalProgress,
  boardMeetings,
  monthlyHighlights,
} = dashboardData
```

## Error Handling

```tsx
try {
  dashboardData = await getPrincipalDashboardData()
} catch (error) {
  return (
    <DashboardSectionError
      title={t.dashboardUnavailable}
      message={t.unableToLoadData}
    />
  )
}
```
