## Architecture Overview

Translation operates on two tiers:

```
┌─────────────────────────────────────────────────────────────┐
│                     Tier 1: Static UI                        │
│  Dictionary JSON files → getDictionary() → {dictionary.key} │
│  5,600+ keys synced across ar.json / en.json pairs          │
│  Covers: sidebar, table headers, status badges, labels      │
├─────────────────────────────────────────────────────────────┤
│                  Tier 2: Dynamic Content                     │
│  Single-language DB storage → getDisplayText() → translate   │
│  Google Translate API + TranslationCache (DB caching)        │
│  Covers: announcements, subjects, departments, events        │
└─────────────────────────────────────────────────────────────┘
```

### Infrastructure Files

| File                                                                | Purpose                                          |
| ------------------------------------------------------------------- | ------------------------------------------------ |
| `src/lib/content-display.ts`                                        | `getDisplayText()`, `getDisplayFields()`         |
| `src/lib/translate.ts`                                              | `translateWithCache()`, server actions           |
| `src/lib/google-translate.ts`                                       | Google Translate API client                      |
| `src/lib/auto-translate.ts`                                         | Auto-translation utilities                       |
| `src/components/internationalization/helpers/index.ts`              | `ValidationHelper`, `ToastHelper`, `ErrorHelper` |
| `src/components/internationalization/dictionaries.ts`               | Route-specific dictionary loaders                |
| `src/components/internationalization/dictionaries/en/messages.json` | Validation, toast, error message keys            |

### The Infrastructure-Coverage Gap

The infrastructure is solid - dictionary files have 5,600+ keys, `ValidationHelper` / `ToastHelper` / `ErrorHelper` classes exist in `src/components/internationalization/helpers/index.ts`, and `getDisplayText()` works well. The gap is **adoption**: ~2,985 hardcoded English strings across 500+ files in the school-dashboard layer haven't been wired to use these helpers.

## Gaps & Blockers

### Gap 1: Forms Layer (~40 form files, ~327 strings)

All labels, placeholders, and helper text are hardcoded English. ~15 form files (~19%) use `useDictionary()`.

**Scope**: `src/components/school-dashboard/**/form.tsx` (~40 files with hardcoded labels)

**Example** (student contact form):

```typescript
// Current - hardcoded
<FormLabel>Phone Number</FormLabel>
<Input placeholder="Enter phone number" />

// Target - dictionary-driven
<FormLabel>{dictionary.school.students.form.phoneNumber}</FormLabel>
<Input placeholder={dictionary.school.students.form.phoneNumberPlaceholder} />
```

### Gap 2: Config / Select Options (24 config files, ~415 strings)

Static option arrays use hardcoded English labels.

**Scope**: `src/components/school-dashboard/**/config.ts` (24 files with hardcoded labels)

**Example** (student config):

```typescript
// Current - hardcoded
const GENDER_OPTIONS = [
  { value: "male", label: "Male" },
  { value: "female", label: "Female" },
]

// Target - dictionary-driven
const getGenderOptions = (d: Dictionary) => [
  { value: "male", label: d.school.students.gender.male },
  { value: "female", label: d.school.students.gender.female },
]
```

### Gap 3: Validation Messages (90 validation files, ~660 strings)

Zod schemas use hardcoded English error messages. `ValidationHelper` exists but is only used in onboarding + auth (not school-dashboard).

**Scope**: `src/components/school-dashboard/**/validation.ts` (90 files)

**Example**:

```typescript
// Current - hardcoded
export const studentSchema = z.object({
  name: z.string().min(1, "Name is required"),
  email: z.string().email("Invalid email"),
})

// Target - using ValidationHelper
export const createStudentSchema = (v: ValidationHelper) =>
  z.object({
    name: z.string().min(1, v.required()),
    email: z.string().email(v.email()),
  })
```

**Blocker**: Zod schemas are currently defined at module level (static). To use `ValidationHelper`, they need to become factory functions that accept the helper as a parameter. This is a significant refactor across 90 files. The pattern already works in onboarding (`createOnboardingSchema(v)`) and auth (`createLoginSchema(v)`).

### Gap 4: Toast Messages (~79 hardcoded calls across 27 files)

Success/error toasts use hardcoded English strings. `ToastHelper` exists but is unused.

**Scope**: 79 hardcoded toast calls across 27 files in school-dashboard

**Example**:

```typescript
// Current - hardcoded
toast.success("Student created successfully")
toast.error("Failed to create student")

// Target - using ToastHelper
toast.success(t.success.student.created())
toast.error(t.error.student.createFailed())
```

**Blocker**: Toast calls happen in client components, but `ToastHelper` needs dictionary context. Requires either a `useToastHelper()` hook or passing dictionary through props.

### Gap 5: Server Action Errors (66 action files, ~302 strings)

Error messages returned from server actions are hardcoded English. `ErrorHelper` exists but is unused.

**Scope**: `src/components/school-dashboard/**/actions.ts` (66 files with hardcoded errors)

**Example**:

```typescript
// Current - hardcoded
return { success: false, error: "Not authenticated" }
return { success: false, error: "Missing school context" }
return { success: false, error: "Failed to create student" }

// Option A - error codes (recommended)
return { success: false, errorCode: "NOT_AUTHENTICATED" }
// Client translates: dictionary.messages.errors.auth.notAuthenticated

// Option B - translate in action
const d = await getDictionary(locale)
return { success: false, error: d.messages.errors.auth.notAuthenticated }
```

**Blocker**: Server actions don't have access to the user's locale. Two approaches: (A) return error codes and translate on the client, or (B) accept `locale` as a parameter.

### Gap 6: Button Text (across all forms)

Common button labels are hardcoded.

**Common patterns**:

- `>Save<` / `>Cancel<` / `>Delete<` / `>Submit<`
- `>Create<` / `>Update<` / `>Edit<`
- `>Back<` / `>Next<` / `>Confirm<`

**Fix**: Use `dictionary.common.save`, `dictionary.common.cancel`, etc. These keys already exist in the dictionary system.

### Gap 7: Wizard Tab / Group Labels (11 wizard configs, ~92 strings)

Step names, group labels, and select options are hardcoded in wizard config objects.

**Scope**: 11 wizard config files + 4 FormFooter preset configs

**Example** (student wizard):

```typescript
// Current - hardcoded
groupLabels: ["Essentials", "Contact Details", "Health & History"],
finalLabel: "Complete",

// Target - dictionary-driven (at render time)
// Config stays static with keys, wizard-layout resolves labels from dictionary
groupLabels: ["essentials", "contactDetails", "healthHistory"],
// wizard-layout.tsx maps: dictionary.school.students.wizard[key]
```

### Gap 8: Sidebar Missing Keys

Two navigation items lack dictionary entries:

- `"Cards"` - not in `platform.sidebar`
- `"Lab"` - not in `platform.sidebar`

**Fix**: Add keys to `school-en.json` and `school-ar.json`.

## Best Practices

### Form Pattern

Pass dictionary to forms and use translated labels:

```typescript
// content.tsx (server component)
const dictionary = await getDictionary(lang)
return <StudentForm dictionary={dictionary.school.students} />

// form.tsx (client component)
interface Props {
  dictionary: Dictionary["school"]["students"]
}

export function StudentForm({ dictionary }: Props) {
  return (
    <FormField
      name="name"
      render={({ field }) => (
        <FormItem>
          <FormLabel>{dictionary.form.name}</FormLabel>
          <Input
            placeholder={dictionary.form.namePlaceholder}
            {...field}
          />
        </FormItem>
      )}
    />
  )
}
```

### Select Pattern

Translate static options via dictionary:

```typescript
// config.ts - export a factory function
export const getGenderOptions = (dictionary: Dictionary["school"]) => [
  { value: "male", label: dictionary.students.gender.male },
  { value: "female", label: dictionary.students.gender.female },
]

// form.tsx - call with dictionary
const options = getGenderOptions(dictionary)
```

### Validation Pattern

Use `ValidationHelper` from i18n helpers:

```typescript
import { getValidationMessages } from "@/components/internationalization/helpers"

// Create schema factory
export const createStudentSchema = (
  v: ReturnType<typeof getValidationMessages>
) =>
  z.object({
    name: z.string().min(1, v.required()),
    email: z.string().email(v.email()),
    phone: z.string().min(10, v.minLength(10)),
  })

// In form component
const dictionary = await getDictionary(lang)
const v = getValidationMessages(dictionary)
const schema = createStudentSchema(v)
```

### Toast Pattern

Use `ToastHelper` with dictionary context:

```typescript
import { getToastMessages } from "@/components/internationalization/helpers"

// In client component
function StudentForm({ dictionary }: Props) {
  const t = useMemo(() => getToastMessages(dictionary), [dictionary])

  const onSubmit = async (data: FormData) => {
    const result = await createStudent(data)
    if (result.success) {
      toast.success(t.success.student.created())
    } else {
      toast.error(t.error.student.createFailed())
    }
  }
}
```

### Server Action Error Pattern

Return error codes instead of hardcoded strings:

```typescript
// actions.ts - return codes
export async function createStudent(
  input: StudentInput
): Promise<ActionResponse> {
  const session = await auth()
  if (!session?.user) {
    return { success: false, errorCode: "NOT_AUTHENTICATED" }
  }

  const { schoolId } = await getTenantContext()
  if (!schoolId) {
    return { success: false, errorCode: "MISSING_SCHOOL_CONTEXT" }
  }

  // ...
}

// form.tsx - translate on client
const ERROR_MAP: Record<string, string> = {
  NOT_AUTHENTICATED: dictionary.messages.errors.auth.notAuthenticated,
  MISSING_SCHOOL_CONTEXT:
    dictionary.messages.errors.tenant.missingSchoolContext,
}

if (!result.success) {
  toast.error(
    ERROR_MAP[result.errorCode] ??
      dictionary.messages.errors.server.internalError
  )
}
```
