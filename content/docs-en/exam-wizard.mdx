## Overview

Two wizard systems, each with two phases:

**Exam Wizard** (exam paper creation)

- **Phase 1 - Create Template**: Design exam paper layout (slots, variants, scoring, print settings) → saves `ExamTemplate`
- **Phase 2 - Generate Exam**: Consume a template, select questions from bank, configure paper → saves `GeneratedExam` + `ExamPaperConfig`

**Certificate Wizard** (certificate creation)

- **Phase 1 - Create Template**: Design certificate composition (slots, decorations, text, signatures) → saves `ExamCertificateConfig`
- **Phase 2 - Generate Certificates**: Consume a config, issue certificates to students (single/batch) → saves `ExamCertificate`

### Route Group Separation

- **Phase 1 + Exam Phase 2**: `(exam-wizard)/` route group — full-screen, no dashboard chrome
- **Cert Phase 2**: `(school-dashboard)/exams/certificates/` — within dashboard layout
- **Auth guard**: ADMIN, DEVELOPER, TEACHER only

## Wizard Steps

### Exam Phase 1: Template Steps (dynamic count)

| #   | Step ID           | Component        | Description                                                                   |
| --- | ----------------- | ---------------- | ----------------------------------------------------------------------------- |
| 1   | gallery           | GalleryStep      | Regional preset (SD, SA, US, MENA) or blank                                   |
| 2   | info              | InfoStep         | Name, subjects, grades, exam type, duration, marks, question types            |
| 3   | layout            | LayoutStep       | 6 slots: header, studentInfo, instructions, answerSheet, cover, footer        |
| 4+  | question-\{type\} | QuestionTypeStep | Dynamic — one per selected Q type, difficulty distribution (easy/medium/hard) |
| N-2 | scoring           | ScoringStep      | Passing score, grade boundaries (add/remove/reset-to-defaults)                |
| N-1 | footer-print      | FooterPrintStep  | Footer variant + decorations (accent bar/watermark/frame) + print settings    |
| N   | preview           | PreviewStep      | 4 summary cards + paper mockup + save (creates/updates ExamTemplate)          |

### Exam Phase 2: Generation Steps (fixed 5)

| #   | Step ID      | Component          | Description                                                                 |
| --- | ------------ | ------------------ | --------------------------------------------------------------------------- |
| 1   | template     | SelectTemplateStep | Pick from school's saved templates                                          |
| 2   | exam         | SelectExamStep     | Create new exam or pick existing PLANNED exam                               |
| 3   | questions    | QuestionBankStep   | Auto-fill matching questions + manual select with filters                   |
| 4   | paper-config | PaperConfigStep    | Style (classic/modern/formal/custom), page size, versions (1-5), toggles    |
| 5   | preview      | PreviewStep        | Summary + generate (creates GeneratedExam + ExamPaperConfig in transaction) |

### Certificate Phase 1: Template Steps (fixed 12)

| #   | Step ID     | Component       | Description                                                                                        |
| --- | ----------- | --------------- | -------------------------------------------------------------------------------------------------- |
| 1   | gallery     | GalleryStep     | Regional preset selection                                                                          |
| 2   | info        | InfoStep        | Name, type (6 options), theme, bilingual title/body, eligibility, signatures                       |
| 3   | header      | HeaderStep      | Slot picker: crest / ministry / bilingual / minimal                                                |
| 4   | title       | TitleStep       | Slot picker: elegant / modern / classic / arabic-calligraphy                                       |
| 5   | recipient   | RecipientStep   | Slot picker: centered / underline / framed / photo                                                 |
| 6   | body        | BodyStep        | Slot picker: achievement / report-summary / transcript / custom                                    |
| 7   | scores      | ScoresStep      | Slot picker: badge-row / table-grid / gauge / hidden                                               |
| 8   | signatures  | SignaturesStep  | Slot picker: dual / triple / single / stamps                                                       |
| 9   | footer      | FooterStep      | Slot picker: verification / minimal / dated / numbered                                             |
| 10  | decorations | DecorationsStep | 5 toggles: border (style+width), corners, seal (position), watermark (text+opacity), ribbon (text) |
| 11  | print       | PrintStep       | Page size (A4/Letter), orientation (landscape/portrait)                                            |
| 12  | preview     | PreviewStep     | Summary + full cert mockup + save (creates/updates ExamCertificateConfig)                          |

### Certificate Phase 2: Issue (dashboard UI, not wizard steps)

- **Single issue**: `generateCertificate()` — checks eligibility, calculates rank, generates verification code
- **Batch issue**: `batchGenerateCertificates()` via `BatchIssueDialog` — issues to all eligible students for an exam
- **Share**: `shareCertificate()` — generates public share token
- **Verify**: `verifyCertificate()` — public endpoint, no auth required
- **Revoke**: `revokeCertificate()` — soft revoke with reason

## State Management

All wizard phases use the same pattern:

```
useReducer(reducer, INITIAL_STATE)
  + localStorage persistence (keyed by schoolId)
  + draft resume dialog on mount
  + Context provider for child access
```

| Wizard       | State Type            | Action Count | Context                                  | Storage Key                  |
| ------------ | --------------------- | ------------ | ---------------------------------------- | ---------------------------- |
| Exam Phase 1 | `TemplateWizardState` | 18 actions   | `WizardProvider` / `useWizard()`         | `wizard-template-{schoolId}` |
| Exam Phase 2 | `ExamWizardState`     | 10 actions   | None (self-contained)                    | None                         |
| Cert Phase 1 | `CertWizardState`     | 17 actions   | `CertWizardProvider` / `useCertWizard()` | `cert-wizard-{schoolId}`     |

## Cross-Module Dependencies

- **Exam Phase 1** imports composition types from `exams/templates/` (`VARIANT_REGISTRY`, region presets: `SD_NATIONAL`, `SA_NATIONAL`, `US_STANDARD`, `MENA_PRIVATE`)
- **Cert Phase 1** imports composition types from `grades/templates/composition/` (`CERT_VARIANT_REGISTRY`, cert region presets)
- Both Phase 1 wizards share atom components (`WizardCard`, `VariantThumbnail`, `GalleryCard`)
- Cert Phase 2 (`certificates/`) is independent, only consuming `ExamCertificateConfig` records from the database
