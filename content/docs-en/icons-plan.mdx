## Automation Reference

### Commands

#### `/icon-add <name>`

Add a new icon to the unified system.

```bash
# Add a new icon
/icon-add activity

# Process:
# 1. Check if icon exists in lucide
# 2. Create component in appropriate category file
# 3. Add to Icons namespace in index.tsx
# 4. Register in registry.ts with metadata
# 5. Verify component renders correctly
```

#### `/icon-migrate <scope>`

Migrate lucide-react imports to unified system.

```bash
# Migrate a single file
/icon-migrate src/components/example.tsx

# Migrate in batches (10 files at a time)
/icon-migrate batch

# Migrate all files (with confirmation)
/icon-migrate all

# Show migration status/report
/icon-migrate report
```

#### `/icon-audit`

Audit icon usage across the codebase.

```bash
/icon-audit

# Output:
# - Unified system imports: 125 files (15.1%)
# - Direct lucide imports: 703 files (84.9%)
# - Unregistered icons: 0
# - Theme compatibility: 100%
# - Files needing migration: 703
```

#### `/icon-registry <mode>`

Manage the icon registry.

```bash
# Sync registry with Icons namespace
/icon-registry sync

# Add single icon to registry
/icon-registry add alertCircle

# Validate registry completeness
/icon-registry validate

# Export registry as JSON
/icon-registry export
```

#### `/icon-validate [scope]`

Validate icons against design system rules.

```bash
# Validate all icons
/icon-validate

# Validate specific category
/icon-validate system

# Validate specific file
/icon-validate src/components/icons/categories/system.tsx
```

**Validation rules:**

- Uses `currentColor` (theme-aware)
- Correct viewBox (0 0 24 24 for UI icons)
- Spreads `{...props}` for customization
- No hardcoded colors
- Proper xmlns attribute

#### `/icon-fetch <source>`

Fetch icons from external sources.

```bash
# Fetch from Anthropic design system
/icon-fetch anthropic

# Fetch education-style icons (ClickView)
/icon-fetch education

# Fetch school operations icons (Zenda)
/icon-fetch school

# Fetch from all sources
/icon-fetch all
```

#### `/icon-generate`

AI-generate icon from description.

```bash
/icon-generate "a graduation cap for education"

# Process:
# 1. Generate SVG based on description
# 2. Optimize SVG (remove metadata, minify)
# 3. Create React component
# 4. Add to appropriate category
# 5. Register with metadata
```

### Agent: icon-manager


## Implementation Guidelines

### SVG Component Pattern

```tsx
// Standard icon component (UI icons)
export const IconName = (props: IconProps) => (
  <svg
    xmlns="http://www.w3.org/2000/svg"
    viewBox="0 0 24 24"
    fill="none"
    stroke="currentColor"
    strokeWidth="2"
    strokeLinecap="round"
    strokeLinejoin="round"
    {...props}
  >
    <path d="..." />
  </svg>
)

// Filled icon variant
export const IconNameFilled = (props: IconProps) => (
  <svg
    xmlns="http://www.w3.org/2000/svg"
    viewBox="0 0 24 24"
    fill="currentColor"
    {...props}
  >
    <path d="..." />
  </svg>
)
```

### SVG Standards

| Property    | UI Icons                | Anthropic Icons                     |
| ----------- | ----------------------- | ----------------------------------- |
| viewBox     | `0 0 24 24`             | `0 0 1000 1000` or `0 0 1680 1260`  |
| Colors      | `currentColor` only     | `#FAF9F5` (light), `#141413` (dark) |
| Fill/Stroke | `stroke="currentColor"` | `fill="currentColor"` or explicit   |
| Props       | `{...props}` last       | `{...props}` last                   |
| Naming      | camelCase               | camelCase                           |

### Do's and Don'ts

**DO:**

```tsx
// âœ… Use unified system
import { Icons } from "@/components/icons"
<Icons.check className="size-5" />

// âœ… Use size-* utilities
<Icons.github className="size-4" />

// âœ… Let icons inherit color
<div className="text-primary">
  <Icons.check />  {/* Inherits text-primary */}
</div>

// âœ… Use semantic color classes
<Icons.trash className="icon-destructive" />

// âœ… Spread props for customization
export const Icon = (props: IconProps) => (
  <svg {...props}>...</svg>
)

// âœ… Use currentColor
<path fill="currentColor" d="..." />
```

**DON'T:**

```tsx
// âŒ Direct lucide import
import { Check } from "lucide-react"

// âŒ Separate width/height
<Icons.github className="w-4 h-4" />

// âŒ Hardcoded colors
<path fill="#000000" d="..." />

// âŒ Inline SVG in components
<button>
  <svg>...</svg>  {/* Add to icons.tsx instead */}
</button>

// âŒ Inconsistent sizes
<Icons.a className="size-4" />
<Icons.b className="size-6" />  {/* Same context, different sizes */}
```

## Handoff Notes

### Ownership Scope

The icon system owner is responsible for:

- **Registry** - Icon metadata and search/filter
- **Fetching** - External icon acquisition
- **Migration** - Gradual lucide â†’ unified transition
- **Automation** - Agents, commands, hooks
- **Documentation** - Showcases, guides, patterns

### Quick Start for New Engineers

1. Read `src/components/icons/index.tsx` (shadcn/ui pattern)
2. Read `src/components/icons/registry.ts` (metadata pattern)
3. Check commands: `/icon-add`, `/icon-validate`, `/icon-audit`
4. Review `content/docs-en/icons.mdx` (user docs)
5. Run `/icon-audit` to see current health

### Key Decision Records

**Why unified system over direct lucide?**

- Consistent theming (currentColor)
- Better tree-shaking
- Centralized management
- Easier migrations
- Type-safe icon names

**Why registry pattern?**

- Searchable metadata
- Documentation generation
- Category filtering
- Validation automation

**Why category files?**

- Organized by domain
- Easier maintenance
- Better code splitting
- Clear ownership
