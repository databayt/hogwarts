Hogwarts uses Next.js App Router with feature-based components. Routes mirror components 1:1 — if you know the URL, you know where the code lives. See [Pattern](/docs/pattern) for code conventions and [Architecture](/docs/architecture) for the mirror pattern explained.

## Project root

```text
hogwarts/
├── public/                # Static assets
├── prisma/
│   ├── schema.prisma      # Datasource + generator + imports
│   ├── models/            # Multi-file schema
│   └── seeds/             # Seed scripts (run via pnpm db:seed:single)
├── src/
│   ├── app/               # Next.js App Router routes
│   ├── components/        # Feature components (mirror app/)
│   ├── lib/               # Utility modules
│   ├── types/             # Global TypeScript definitions
│   ├── styles/            # Global styles (typography.css)
│   └── proxy.ts           # Edge proxy — subdomain detection, URL rewriting
├── content/docs-en/       # MDX documentation (English)
├── content/docs-ar/       # MDX documentation (Arabic mirror)
├── scripts/               # Build + seed scripts
├── tests/                 # E2E test suites (Playwright)
├── .env                   # Central environment — never .env.local
├── next.config.ts
├── package.json           # pnpm only
└── tsconfig.json          # @/* → ./src/*
```

## Routes (`src/app/`)

All routes live under `[lang]/` for locale routing (`ar`, `en`). Four entry points, implemented as App Router groups.

### Entry points

| Group                | URL                             | Purpose                | Access                     |
| -------------------- | ------------------------------- | ---------------------- | -------------------------- |
| `(saas-marketing)`   | `ed.databayt.org/`              | SaaS public marketing  | Public                     |
| `(saas-dashboard)`   | `ed.databayt.org/dashboard`     | Operator dashboard     | DEVELOPER only             |
| `(school-marketing)` | `school.databayt.org/`          | Per-tenant public site | Public                     |
| `(school-dashboard)` | `school.databayt.org/dashboard` | Per-tenant platform    | Authenticated school roles |

The `s/[subdomain]/` segment is **internal only**. Middleware rewrites `school.databayt.org/dashboard` → `/[lang]/s/school/dashboard`. Client code uses clean tenant-relative paths.

## Components (`src/components/`)

Routes mirror components 1:1. The page file is thin — it imports and renders the feature's `content.tsx`.

```tsx
// src/app/[lang]/s/[subdomain]/(school-dashboard)/(listings)/students/page.tsx
import { Content } from "@/components/school-dashboard/listings/students/content"

export default function StudentsPage() {
  return <Content />
}
```

## Prisma schema (`prisma/`)

```text
prisma/
├── schema.prisma                # Datasource + generator + multi-file imports
├── models/                      # Multi-file schema (one .prisma per domain)
│   ├── school.prisma
│   ├── auth.prisma
│   ├── students.prisma
│   ├── staff.prisma
│   ├── academic.prisma
│   ├── timetable.prisma
│   ├── attendance.prisma
│   ├── catalog.prisma
│   ├── exam.prisma
│   ├── finance-core.prisma
│   ├── subscription.prisma
│   ├── stream.prisma
│   ├── transportation.prisma
│   └── …
└── seeds/                       # Seed scripts
    ├── index.ts                 # Orchestrator — never run directly
    ├── single.ts                # Single seed runner — always use this
    └── …
```

Run seeds via `pnpm db:seed:single <name>`. Never `pnpm db:seed`. See [Safe seeds](/docs/safe-seeds).

## Key rules

| Rule                   | Detail                                                                |
| ---------------------- | --------------------------------------------------------------------- |
| Package manager        | pnpm only (Vercel requirement)                                        |
| Environment            | Single `.env` — never `.env.local`, `.env.development`, etc.          |
| Multi-tenant isolation | Every DB query includes `schoolId`                                    |
| Mirror pattern         | `app/` routes map 1:1 to `components/` directories                    |
| Semantic HTML          | No hardcoded `text-*` / `font-*` classes                              |
| Locale routing         | All routes under `[lang]/` for `ar` / `en`                            |
| Subdomain routing      | School routes under `s/[subdomain]/` (internal only)                  |
| Logical CSS            | `ms-*`, `me-*`, `ps-*`, `pe-*` — never `ml-*`, `mr-*`, `pl-*`, `pr-*` |
| Import alias           | `@/*` → `./src/*`                                                     |
| Port                   | Always 3000 in dev                                                    |
| Seeding                | Always `pnpm db:seed:single <name>` — never `pnpm db:seed`            |
