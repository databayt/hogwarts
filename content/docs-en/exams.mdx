## Question Bank

### Overview

The Question Bank stores reusable questions organized by subject, type, difficulty, Bloom's taxonomy level, and tags. It supports 7 question types and includes psychometric analytics.

### Question Types

| Type            | Description                            | Auto-Gradable | Template Section             |
| --------------- | -------------------------------------- | ------------- | ---------------------------- |
| Multiple Choice | Select one correct answer from options | Yes           | `templates/multiple-choice/` |
| True/False      | Binary choice questions                | Yes           | `templates/true-false/`      |
| Fill in Blank   | Student provides exact answer          | Yes           | `templates/fill-blank/`      |
| Short Answer    | 1-2 sentence responses                 | AI-Assisted   | `templates/short-answer/`    |
| Essay           | Long-form written responses            | AI-Assisted   | `templates/essay/`           |
| Matching        | Match items from two columns           | Yes           | `templates/matching/`        |
| Ordering        | Arrange items in correct sequence      | Yes           | `templates/ordering/`        |

### Psychometrics

The `qbank/analytics/` module provides item-level psychometric analysis:

- **Item Discrimination Index** - How well a question differentiates strong/weak students
- **Distractor Analysis** - Effectiveness of wrong options in MCQ
- **Difficulty Calibration** - Actual vs. labeled difficulty
- **Success Rate** - Percentage of correct responses
- **Times Used** - Usage frequency across exams

### Catalog Integration

The `qbank/catalog-tab.tsx` provides a browsable catalog of shared questions that schools can import into their local question banks.

### Bulk Import

Import questions from CSV:

```csv
questionText,questionType,difficulty,bloomLevel,points,options,sampleAnswer,tags
"What is 2+2?",MULTIPLE_CHOICE,EASY,REMEMBER,1,"[{""text"":""4"",""isCorrect"":true},{""text"":""3"",""isCorrect"":false}]","","math,arithmetic"
```

## Exam Generation

### Templates

Templates define the structure and distribution of questions. Managed at `/exams/generate/templates`.

**Distribution Editor** (`generate/distribution-editor.tsx`):

- Define counts per question type and difficulty level
- Set Bloom's taxonomy distribution
- Visual editor with real-time pool availability

### Catalog

Browse and import shared exam templates from the platform catalog at `/exams/generate/catalog`.

### Version Library

Generate multiple versions of the same exam with different question orderings and option shuffling at `/exams/generate/versions`.

### Contributions

Track who contributed questions and templates at `/exams/generate/contributions`.

### Smart Selection Algorithm

1. Groups questions by type and difficulty
2. Fills distribution requirements from pool
3. Applies Bloom's level distribution if specified
4. Uses seeded randomization for reproducibility
5. Prevents duplicate questions across versions

## Paper Generation

### Overview

The paper generation system produces print-ready PDF exam papers using `@react-pdf/renderer`. It features a modular template architecture with composable section variants.

### Template Architecture

The system has 4 layers:

**Layer 1 - Atoms** (13 primitives in `templates/atom/`):

| Atom               | Purpose                           |
| ------------------ | --------------------------------- |
| `alignment-marker` | Alignment marks for scanning      |
| `answer-line`      | Blank answer lines                |
| `blank-slot`       | Fill-in-the-blank slots           |
| `bubble`           | OMR bubble circles                |
| `checkbox`         | Checkbox squares                  |
| `divider`          | Section dividers                  |
| `field-line`       | Student info field with underline |
| `logo-block`       | School logo with fallback         |
| `option-label`     | A/B/C/D option labels             |
| `points-badge`     | Points display badge              |
| `question-number`  | Question numbering                |
| `type-label`       | Question type label               |
| `watermark`        | Diagonal watermark overlay        |

**Layer 2 - Question Sections** (7 types in `templates/<type>/`):

- `multiple-choice/` - MCQ with option layout
- `true-false/` - Binary choice rendering
- `fill-blank/` - Blank slots with underlines
- `short-answer/` - Answer line blocks
- `essay/` - Extended answer space
- `matching/` - Two-column matching
- `ordering/` - Sequence ordering

**Layer 3 - Page Sections** (6 slot types with variants):

| Slot         | Variants                                         |
| ------------ | ------------------------------------------------ |
| Header       | standard, ministry, minimal, bilingual, centered |
| Footer       | standard, disclaimer, minimal, grading           |
| Student Info | standard, bubble-id, table, photo                |
| Instructions | standard, compact, rules, sectioned              |
| Answer Sheet | standard, omr, grid                              |
| Cover        | standard, toc, ministry                          |

**Layer 4 - Composition** (`templates/composition/`):
The composition system dynamically assembles sections based on a `CompositionConfig`:

```typescript
interface CompositionConfig {
  slots: {
    header: "standard" | "ministry" | "minimal" | "bilingual" | "centered"
    footer: "standard" | "disclaimer" | "minimal" | "grading"
    studentInfo: "standard" | "bubble-id" | "table" | "photo"
    instructions: "standard" | "compact" | "rules" | "sectioned"
    answerSheet: "standard" | "omr" | "grid"
    cover: "standard" | "toc" | "ministry"
  }
  decorations: {
    accentBar: {
      enabled: boolean
      height?: number
      colorKey?: "accent" | "primary"
    }
    watermark: { enabled: boolean; text?: string; opacity?: number }
    frame: { enabled: boolean; outerWidth?: number; innerWidth?: number }
  }
  slotProps: {
    /* per-slot overrides */
  }
}
```

### ComposableDocument

A single `ComposableDocument` component replaces the 4 legacy templates (Classic, Modern, Formal, Custom). It reads the composition config and renders chosen variants dynamically:

```
Resolution priority:
1. blockConfig JSON (explicit user composition) — highest
2. Regional preset features → composition overrides
3. DEFAULT_COMPOSITION — fallback (all "standard" variants)
```

The legacy templates still exist as deprecated files for backwards compatibility.

### Compose Tab

The `paper/composition-panel.tsx` provides a visual UI for selecting variants per slot:

- Toggle-group per slot with bilingual labels
- Decoration switches (accent bar, watermark, double frame)
- Saves as `blockConfig` JSON on `ExamPaperConfig`

### Regional Presets

Country-specific paper formatting in `templates/presets/`:

| Preset         | Key Features                                             |
| -------------- | -------------------------------------------------------- |
| `sd-national`  | Sudan - Ministry header, disclaimer footer, watermark    |
| `sa-national`  | Saudi Arabia - Ministry branding, bubble-id, OMR sheets  |
| `us-standard`  | USA - Standard layout, clean formatting                  |
| `mena-private` | MENA private schools - Bilingual header, custom branding |

### Theme System

Each template style (CLASSIC, MODERN, FORMAL, CUSTOM) has a `PaperTheme`:

- Colors: primary, accent, surface, text
- Typography: font family, sizes (title, subtitle, body, small)
- Spacing and border styles
- Locale-aware: RTL/LTR, number style (Arabic/Latin), text alignment

### Paper Config Form

`paper/config-form.tsx` provides the full configuration UI:

- Template selection (Classic, Modern, Formal, Custom)
- Page setup (size, orientation, layout)
- Toggle sections (logo, title, student info, instructions, page numbers)
- Answer sheet type (None, Standard, Bubble/OMR)
- Compose tab for variant selection
- Regional preset selection

### Server Actions

```typescript

      ├→ AUTO_GRADED (auto-marked, may need review)
```

```bash
## Troubleshooting

### Exam Generation Fails

- Check question bank has enough questions for the distribution
- Verify distribution can be satisfied with current pool
- Review template configuration matches available types

### AI Grading Issues

- Ensure OpenAI API key is configured in `.env`
- Check rate limits aren't exceeded
- Verify rubric exists for the question

### PDF Paper Generation Problems

- Check `@react-pdf/renderer` is installed
- Verify school branding is configured (logo URL, colors)
- For Arabic: ensure Rubik font is accessible
- For batch: ensure adequate server memory

### Digital Exam Taking Issues

- Verify exam status is `IN_PROGRESS`
- Check student is enrolled in the exam's class
- Proctor guard requires HTTPS in production

### Certificate QR Not Scanning

- Verify `QR_CODE_SECRET` env var is set
- Check certificate hasn't been revoked
- Ensure verification route is publicly accessible

## Performance Tips

1. **Result Caching** - `results/cache-manager.ts` caches computed results
2. **Batch Operations** - Use bulk grade for large classes
3. **Indexing** - All foreign keys have `@@index` directives
4. **Pagination** - Results and question bank are paginated
5. **Background Jobs** - Batch PDF and certificate generation runs in background
6. **Font Registration** - `templates/fonts.ts` registers fonts once (idempotent)
```
