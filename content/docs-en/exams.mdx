## Question bank

| Type            | Auto-gradable | Template section             |
| --------------- | :-----------: | ---------------------------- |
| Multiple choice |       ✓       | `templates/multiple-choice/` |
| True / false    |       ✓       | `templates/true-false/`      |
| Fill in blank   |       ✓       | `templates/fill-blank/`      |
| Short answer    |  AI-assisted  | `templates/short-answer/`    |
| Essay           |  AI-assisted  | `templates/essay/`           |
| Matching        |       ✓       | `templates/matching/`        |
| Ordering        |       ✓       | `templates/ordering/`        |

`qbank/analytics/` provides per-item discrimination index, distractor analysis, difficulty calibration, success rate, usage frequency.

`qbank/catalog-tab.tsx` browses shared questions for import.

CSV bulk import:

```csv
questionText,questionType,difficulty,bloomLevel,points,options,sampleAnswer,tags

```

## Generation

- **Templates** at `/exams/generate/templates` — distribution rules per type, difficulty, Bloom level.
- **Catalog** at `/exams/generate/catalog` — browse and import shared templates.
- **Versions** at `/exams/generate/versions` — multiple shuffled versions.
- **Contributions** at `/exams/generate/contributions` — author tracking.

Smart selection algorithm: group by type and difficulty, fill distribution, apply Bloom distribution, seeded randomization for reproducibility, prevent duplicates across versions.

`manage/analytics-dashboard.tsx` provides school-wide statistics, standards coverage, quick paper-gen, export.

## Paper generation

`@react-pdf/renderer`-based PDF system with a four-layer composable template:

1. **Atoms** (13 in `templates/atom/`) — `alignment-marker`, `answer-line`, `blank-slot`, `bubble`, `checkbox`, `divider`, `field-line`, `logo-block`, `option-label`, `points-badge`, `question-number`, `type-label`, `watermark`.
2. **Question sections** (7 in `templates/<type>/`) — one per question type.
3. **Page sections** (6 slots × variants):

   | Slot         | Variants                                         |
   | ------------ | ------------------------------------------------ |
   | Header       | standard, ministry, minimal, bilingual, centered |
   | Footer       | standard, disclaimer, minimal, grading           |
   | Student info | standard, bubble-id, table, photo                |
   | Instructions | standard, compact, rules, sectioned              |
   | Answer sheet | standard, omr, grid                              |
   | Cover        | standard, toc, ministry                          |

4. **Composition** (`templates/composition/`) — `ComposableDocument` reads a `CompositionConfig` and assembles slot variants dynamically.

```typescript
interface CompositionConfig {
  slots: Record<
    | "header"
    | "footer"
    | "studentInfo"
    | "instructions"
    | "answerSheet"
    | "cover",
    string
  >
  decorations: {
    accentBar: {
      enabled: boolean
      height?: number
      colorKey?: "accent" | "primary"
    }
    watermark: { enabled: boolean; text?: string; opacity?: number }
    frame: { enabled: boolean; outerWidth?: number; innerWidth?: number }
  }
  slotProps: Record<string, unknown>
}
```

Resolution priority: `blockConfig` JSON → regional preset overrides → `DEFAULT_COMPOSITION`.

`paper/composition-panel.tsx` provides the visual UI for selecting variants per slot. The four legacy templates (Classic, Modern, Formal, Custom) remain as deprecated wrappers around `ComposableDocument`.

Regional presets (`templates/presets/`):

| Preset         | Highlights                                               |
| -------------- | -------------------------------------------------------- |
| `sd-national`  | Sudan — ministry header, disclaimer footer, watermark    |
| `sa-national`  | Saudi Arabia — ministry branding, bubble-id, OMR sheets  |
| `us-standard`  | USA — standard layout                                    |
| `mena-private` | MENA private schools — bilingual header, custom branding |

Each style has a `PaperTheme` with locale-aware colour, typography, spacing, RTL/LTR direction, and number style (Arabic / Latin).

`paper/config-form.tsx` exposes the config UI: template, page setup, toggles, answer sheet type, compose tab, regional preset selection.

## Digital taking

`take/exam-player.tsx` is the full-screen runner: per-question nav, countdown timer (`take/timer.tsx`), auto-save, validated submit.

`take/proctor-guard.tsx` adds proctoring: device fingerprint, security flag reporting (tab switch, copy / paste, screen capture), per-exam proctor mode, retake-penalty calculation.

```typescript
startExamSession(input: StartExamSessionInput)
autoSaveAnswers(input: AutoSaveAnswersInput)
reportSecurityFlag(input: ReportSecurityFlagInput)
submitExamAnswers(input: SubmitExamAnswersInput)
```

## Marking

Result statuses progress: `NOT_STARTED` → `AUTO_GRADED` | `AI_GRADED` | `IN_PROGRESS` → `REVIEWED` → `COMPLETED`. Override is supported at any point.

```typescript
autoGradeAnswer(studentAnswerId: string)
aiGradeAnswer(studentAnswerId: string)
manualGrade(answerId: string, points: number, feedback?: string)
bulkGradeExam(examId: string, method: GradingMethod)
overrideGrade(resultId: string, newScore: number, reason: string)
```

OCR uses GPT-4 Vision for handwritten answers; AI grading uses essay rubrics in the question bank.

## Results and analytics

`/exams/result` lists exams with class average, pass rate, grade distribution. Per-exam at `/exams/result/[id]/analytics`:

- Class-level — average, median, mode, std dev, top / bottom performers, distribution chart, trend.
- Question-level — discrimination index, distractor analysis, most missed, average time per question.

Batch PDF at `/exams/result/[id]/pdf/batch` with background processing, multiple result-card templates, ZIP download. CSV export for raw data.

`results/cache-manager.ts` caches computed results for performance.

## Troubleshooting

| Symptom                      | Likely cause                                                                                           |
| ---------------------------- | ------------------------------------------------------------------------------------------------------ |
| Exam generation fails        | Pool too small for distribution; fix template or add questions                                         |
| AI grading errors            | `OPENAI_API_KEY` missing, rate limit hit, missing rubric                                               |
| PDF paper generation crashes | `@react-pdf/renderer` missing; missing logo URL; Rubik font unreachable; insufficient memory for batch |
| Digital exam won't start     | Exam status not `IN_PROGRESS`; student not enrolled in class                                           |
| Proctor guard inactive       | Production HTTPS required                                                                              |
| Certificate QR doesn't scan  | `QR_CODE_SECRET` missing; certificate revoked; verify route blocked                                    |
