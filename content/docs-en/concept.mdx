## What is a Concept?

A **concept** is a curriculum-agnostic identifier that maps every subject to shared visual assets (thumbnail, banner, color). Instead of storing images per-subject, the system stores them per-concept. This means Arabic in Sudan and World Languages in the US share the same `languages` concept and the same S3 images.

```
Subject (any curriculum)
  └── concept: "math"
        ├── thumbnail: catalog/concepts/g{grade}-math/thumbnail-{size}.webp
        ├── banner:    catalog/concepts/g{grade}-math/banner-{size}.webp
        └── cover:     catalog/concepts/math/cover
```

## S3 Storage Pattern

Each concept has grade-specific S3 paths with 4 size variants:

```
catalog/concepts/g1-math/thumbnail-sm.webp     (200px)
catalog/concepts/g1-math/thumbnail-md.webp     (600px)
catalog/concepts/g1-math/thumbnail-lg.webp     (1200px)
catalog/concepts/g1-math/thumbnail-original.webp
catalog/concepts/g1-math/banner-sm.webp
catalog/concepts/g1-math/banner-md.webp
catalog/concepts/g1-math/banner-lg.webp
catalog/concepts/g1-math/banner-original.webp
catalog/concepts/math/cover
```

All 12 grades share the same source image per concept. Grade-specific paths (`g1-math`, `g7-math`) support future grade-specific imagery.

## Image Resolution Flow

```
Database                          S3 / CloudFront
────────                          ────────────────
Subject.thumbnailKey              catalog/concepts/g1-math/thumbnail
  │                                 │
  ├── getCatalogImageUrl(key, "sm") ├── thumbnail-sm.webp   → 200px
  ├── getCatalogImageUrl(key, "md") ├── thumbnail-md.webp   → 600px
  ├── getCatalogImageUrl(key, "lg") ├── thumbnail-lg.webp   → 1200px
  └── getCatalogImageUrl(key, "original") └── thumbnail-original.webp
                                    │
                                    └── https://{CLOUDFRONT}/catalog/concepts/g1-math/thumbnail-md.webp
```

Key files:

- `src/lib/catalog-image-url.ts` -- client-safe URL construction
- `src/lib/catalog-image.ts` -- Sharp processing + S3 upload
- `src/components/ui/catalog-image.tsx` -- responsive srcSet component

## Seed Pipeline

Run in this order to populate images:

```bash
pnpm db:seed:single concept-images   # 1. Process JPGs → S3 thumbnails (all 12 grades)
pnpm db:seed:single banner-copy      # 2. Copy ClickView banners → concept paths
pnpm db:seed:single us-catalog       # 3. US subjects (sets concept + S3 keys)
pnpm db:seed:single sd-catalog       # 4. SD subjects (sets concept + S3 keys)
pnpm db:seed:single world-curricula  # 5. GB/IB/SA/EG/AE/QA/KW/JO subjects
```

## Adding a New Curriculum

To add a new curriculum that reuses concept images:

1. Define subjects with `concept` field pointing to one of the 23 concepts
2. Set `thumbnail`, `banner`, `cover` using the pattern:
   ```typescript
   thumbnail: `catalog/concepts/g${grade}-${concept}/thumbnail`
   banner: `catalog/concepts/g${grade}-${concept}/banner`
   cover: `catalog/concepts/${concept}/cover`
   ```
3. Assign `color` from `CONCEPT_COLORS` or a custom hex
4. The images are already on S3 -- no upload needed
