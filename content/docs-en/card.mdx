`card.tsx` renders feature-specific display cards composed from shadcn `Card` primitives. KPIs, entity summaries, onboarding step tiles. Pure presentation — no data fetching, no mutations.

## Categories

| Category        | Shape                                                                | Reference                             |
| --------------- | -------------------------------------------------------------------- | ------------------------------------- |
| Entity display  | Full / compact / health / stats variants for one domain entity.      | `saas-dashboard/billing/card.tsx`     |
| KPI / stats     | Single metric with label, trend, icon.                               | `school-dashboard/dashboard/card.tsx` |
| Onboarding step | Visual card representing a wizard option (level, curriculum, theme). | `onboarding/branding/card.tsx`        |
| Atom lab        | Showcase cards in `atom/lab/` for the design system.                 | `atom/lab/*-card.tsx`                 |

## Five rules

1. **Compose from shadcn `Card` primitives.** `Card`, `CardHeader`, `CardContent`, `CardFooter`, `CardTitle`, `CardDescription`. Don't reinvent.
2. **Typed entity props.** Import the entity type from `types.ts`. No `any`.
3. **Formatters from `util.ts`.** Currency, dates, names — formatting helpers live in `util.ts`, not in the card.
4. **Multiple variants per file.** When a feature needs both `EntityCard` and `CompactCard`, export both from one `card.tsx`. Don't split into two files.
5. **`showActions` prop for conditional footers.** Lets the same card render in interactive (table cell) and read-only (detail page) contexts.

```tsx
import { Badge } from "@/components/ui/badge"
import {
  Card,
  CardContent,
  CardDescription,
  CardFooter,
  CardHeader,
  CardTitle,
} from "@/components/ui/card"

import type { TenantDto } from "./types"
import { formatDate } from "./util"

interface TenantCardProps {
  tenant: TenantDto
  showActions?: boolean
}

export function TenantCard({ tenant, showActions = true }: TenantCardProps) {
  return (
    <Card>
      <CardHeader>
        <CardTitle>{tenant.name}</CardTitle>
        <CardDescription>{tenant.domain}</CardDescription>
      </CardHeader>
      <CardContent>
        <Badge variant={tenant.isActive ? "default" : "secondary"}>
          {tenant.isActive ? "Active" : "Suspended"}
        </Badge>
        <p className="text-muted-foreground text-sm">
          Created {formatDate(tenant.createdAt)}
        </p>
      </CardContent>
      {showActions && <CardFooter>{/* edit, archive, etc. */}</CardFooter>}
    </Card>
  )
}

export function TenantCompactCard({ tenant }: { tenant: TenantDto }) {
  return (
    <Card className="bg-muted/30">
      <CardHeader>
        <CardTitle className="text-base">{tenant.name}</CardTitle>
      </CardHeader>
    </Card>
  )
}
```

## Naming for multi-variant exports

Pick a consistent suffix system:

| Variant           | Suffix    | Example             |
| ----------------- | --------- | ------------------- |
| Full / default    | none      | `TenantCard`        |
| Compact / minimal | `Compact` | `TenantCompactCard` |
| Health / status   | `Health`  | `TenantHealthCard`  |
| Metric / KPI      | `Stats`   | `TenantStatsCard`   |
| Detail-only       | `Detail`  | `TenantDetailCard`  |

The codebase currently mixes `Compact`, `Health`, `Summary`, `Stats`. Settle on the table above for new code.

## Anti-patterns

- **Form logic inside cards** — 12 onboarding card files mix card display with form state. Cards are presentation; lift state to `client.tsx` or the form component.
- **Reinventing card primitives** — using raw `<div className="rounded-lg border p-6">` instead of `Card`. Use the primitive.
- **Inline formatters** — `new Intl.NumberFormat(...)` in the card body. Move to `util.ts`.
- **Untyped props** — `any` for the entity. Import from `types.ts`.
- **Hardcoded labels** — should come from the dictionary slice for translatable text.

## Relationship to siblings

- `card.tsx` is rendered by `content.tsx`, `client.tsx`, or `detail.tsx`.
- It imports from `types.ts` (entity shape), `util.ts` (formatters), and `config.ts` (status options).
- It does **not** import from `actions.ts`, `queries.ts`, or `form.tsx`.
- It does **not** call server actions. Action handlers live one level up.

## Naming

- Filename: `card.tsx`. Always.
- Function: `<Feature>Card` for the default; suffix-coded variants per the table above.
- Named exports.
