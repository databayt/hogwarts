## Overview

The Invoice module manages the full billing lifecycle for schools -- from draft creation through payment tracking. It supports two creation flows (modal form and multi-step wizard), bulk generation from fee structures, email delivery via Resend, and a dashboard with analytics. All operations enforce multi-tenant isolation via `getTenantContext()` and role-based access control via `checkCurrentUserPermission()`.

## Server Actions

All server actions live in `actions.ts` and follow the mandatory pattern:

```typescript
async function requireAuthAndTenant() {
  const session = await auth()
  if (!session?.user?.id) return { success: false, error: "Not authenticated" }
  const { schoolId } = await getTenantContext()
  if (!schoolId) return { success: false, error: "Missing school context" }
  return { userId: session.user.id, schoolId }
}
```

### Transaction Pattern

Invoice creation wraps address + invoice + item creation in a single `db.$transaction` to prevent orphan records:

```typescript
const invoice = await db.$transaction(async (tx) => {
  const fromAddr = await tx.userAddress.create({ data: fromData })
  const toAddr = await tx.userAddress.create({ data: toData })
  const inv = await tx.userInvoice.create({
    data: { ...invoiceData, fromId: fromAddr.id, toId: toAddr.id },
  })
  await tx.userInvoiceItem.createMany({ data: itemsData })
  return inv
})
```

**Wizard actions** (`wizard/actions.ts`) follow the same `getTenantContext()` pattern:

### Email Delivery

Invoices are sent via Resend using an inline-styled React template.

**Template** (`send-invoice-email.tsx`):

- Greeting: "Hello \{firstName\},"
- Body: "Your invoice is ready for review."
- Detail box: Invoice No, Due Date, Total Amount
- CTA button: "View Invoice" linking to `invoiceURL`
- Footer: "If you have any questions..."

**Current limitations:**

- No school branding (no logo, no school name in header)
- No itemized breakdown (only shows total)
- No signature
- Hardcoded English text
- Sender: `"Invoice App <onboarding@resend.dev>"` (placeholder, not school-specific)
- Template is plain React JSX, not React Email primitives

**Action:** `sendInvoiceEmail(id, email)` requires `invoice:export` RBAC permission.

## Service Areas

Invoices are consumed across 6 areas of the platform:

### 1. Admin/Accountant Invoice Management

Full CRUD via `/finance/invoice/` routes. Admin and Accountant roles have direct access to create, edit, send, and manage invoices through the DataTable interface with filtering, sorting, and pagination.

### 2. Student Fee Invoices (via Admission Enrollment)

When `confirmEnrollment()` runs in the admission module:

```
FeeStructure â†’ FeeAssignment (PENDING) â†’ confirmEnrollment()
    â†’ createInvoiceFromEnrollment() â†’ UserInvoice (ENR- prefix)
```

- Queries `FeeAssignment` records for the enrolled student in the campaign's academic year
- Maps `FeeStructure.name` to line item name, `FeeAssignment.finalAmount` to price
- Creates invoice with `ENR-` prefix, status `UNPAID`, due date +30 days
- **Non-fatal**: enrollment succeeds even if invoice creation fails
- **No FK**: connection between `FeeAssignment` and `UserInvoice` is runtime-only

### 3. Student Dashboard

`InvoiceHistorySection` (at `src/components/school-dashboard/dashboard/invoice-history-section.tsx`) renders a "Fee Payments" table. Uses `getInvoicesByRole("STUDENT")` which fetches invoices where `userId` matches the authenticated student.

### 4. Guardian Dashboard

Same `InvoiceHistorySection` component with "Fee Payments" title. Uses `getInvoicesByRole("GUARDIAN")` which fetches invoices for the guardian's children.

### 5. Accountant Dashboard

Dedicated `accountant.tsx` dashboard with:

- MetricCards: totalInvoices, paidInvoices, unpaidInvoices
- ProgressCard: "Invoices Paid" with payment rate ring
- `InvoiceHistorySection` with "Recent Transactions" title
- Queries `db.userInvoice.count()` directly for metrics

### 6. Finance Hub

`finance/content.tsx` (the finance landing page) queries `db.userInvoice.count()` for a stat grid showing "Unpaid Invoices" count and provides a quick-link to the invoice module.

## Data Model

### Prisma Models (5 tables)

**`UserInvoice`** -- Primary invoice record:

- Fields: `id`, `invoice_no`, `invoice_date` (DateTime), `due_date` (DateTime), `currency` (default "USD")
- Addresses: `fromAddressId` (unique), `toAddressId` (unique) -- one-to-one with `UserInvoiceAddress`
- Financials: `sub_total` (Decimal 10,2), `discount` (Decimal 10,2 optional), `tax_percentage` (Decimal 5,2 optional), `total` (Decimal 10,2)
- Metadata: `notes` (Text), `status` (InvoiceStatus, default UNPAID), `wizardStep` (null = complete)
- Relations: `userId` -> User, `schoolId` -> School, `journalEntryId` -> JournalEntry (accounting)
- Indexes: `@@unique([schoolId, invoice_no])`, `@@index([userId])`, `@@index([schoolId])`, `@@index([status])`

**`UserInvoiceItem`** -- Line items:

- Fields: `id`, `item_name`, `quantity` (Int), `price` (Decimal 10,2), `total` (Decimal 10,2), `invoiceId`, `schoolId`

**`UserInvoiceAddress`** -- From/To addresses:

- Fields: `id`, `name`, `email?`, `address1`, `address2?`, `address3?`, `schoolId`

**`UserInvoiceSettings`** -- Per-user settings:

- Fields: `id`, `invoiceLogo?`, `userId` (unique), `schoolId`
- Relations: `signature` -> UserInvoiceSignature

**`UserInvoiceSignature`** -- Signature storage:

- Fields: `id`, `name?`, `image?`, `settingsId` (unique), `schoolId`

### Fee-to-Invoice Data Flow

```
FeeStructure (template per class)
    â†“ assigned to student
FeeAssignment (PENDING status, finalAmount)
    â†“ confirmEnrollment() reads fee assignments
createInvoiceFromEnrollment()
    â†“ maps FeeStructure.name â†’ item_name, FeeAssignment.finalAmount â†’ price
UserInvoice + UserInvoiceItems (ENR- prefix, UNPAID)
```

There is no direct FK between `FeeAssignment` and `UserInvoice`. The connection is made at runtime during enrollment.

### Important Distinction

- **`UserInvoice`** -- School-level invoices (this module). Created by admins, accountants, or automatically via enrollment.
- **`Invoice`** (Stripe) -- Platform billing invoices for school subscriptions. Separate model mapped to `@@map("invoices")`. The `Receipt` model links to Stripe `Invoice`, NOT to `UserInvoice`.
