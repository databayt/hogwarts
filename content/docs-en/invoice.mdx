The invoice module manages billing from draft to payment. Two creation flows (3-step modal + multi-page wizard with draft persistence), bulk generation from fee structures, email delivery via Resend, and a dashboard with analytics. All operations enforce `getTenantContext()` + `checkCurrentUserPermission()`.

## Server actions

All in `actions.ts`, gated by `requireAuthAndTenant()`:

```typescript
async function requireAuthAndTenant() {
  const session = await auth()
  if (!session?.user?.id) return { success: false, error: "Not authenticated" }
  const { schoolId } = await getTenantContext()
  if (!schoolId) return { success: false, error: "Missing school context" }
  return { userId: session.user.id, schoolId }
}
```

## Creation flows

### Modal (3-step)

```
Step 1 — Basic info: from (pre-filled), to (client name, email, address)
Step 2 — Client & items: invoice number (auto: I{YY}{NNN}), dates, currency, line items
Step 3 — Review: summary, subtotal / discount / tax / total, notes
```

Per-step validation via `form.trigger(STEP_FIELDS[step])` against `InvoiceSchemaZod`.

### Wizard (multi-page)

```
/add/[id]/details  →  /add/[id]/items  →  Complete
```

Drafts saved to DB with `wizardStep` field. Wizard actions (`wizard/actions.ts`):

- `createDraftInvoice` — creates draft with `wizardStep`
- `getInvoiceForWizard` — loads draft (tenant-scoped)
- `completeInvoiceWizard` — validates and finalizes
- `updateInvoiceWizardStep` — track progress
- `deleteDraftInvoice` — draft-only with cascading cleanup

## School branding

`settings/content.tsx` provides two accordions:

1. **Invoice Logo** — `FileUploader` to `finance/invoice-logos/`, max 5 MB.
2. **Invoice Signature** — name + image to `finance/signatures/`, max 2 MB.

Storage:

```
UserInvoiceSettings { id, userId (unique), schoolId, invoiceLogo?, signature → UserInvoiceSignature }
UserInvoiceSignature { id, settingsId (unique), schoolId, name?, image? }
```

Branding is stored but **not consumed**: PDF expects `schoolLogo` / `schoolName` / `schoolAddress`; email expects no branding fields. Currently orphaned data.

## Service areas

Six consumers across the platform:

1. **Admin / Accountant management** — full CRUD via `/finance/invoice/`
2. **Student fee invoices** — auto-generated by `confirmEnrollment()`:

   ```
   FeeStructure → FeeAssignment (PENDING) → confirmEnrollment()
     → createInvoiceFromEnrollment() → UserInvoice (ENR- prefix)
   ```

   Maps `FeeStructure.name → item_name`, `FeeAssignment.finalAmount → price`. Status `UNPAID`, due 30 days. Non-fatal — enrollment succeeds even if invoice fails. No FK between `FeeAssignment` and `UserInvoice`; runtime-only connection.

3. **Student dashboard** — `InvoiceHistorySection` ("Fee Payments") via `getInvoicesByRole("STUDENT")`
4. **Guardian dashboard** — same component, fetches children's invoices via `getInvoicesByRole("GUARDIAN")`
5. **Accountant dashboard** — `accountant.tsx` with metrics, `ProgressCard` ("Invoices Paid"), `InvoiceHistorySection`
6. **Finance hub** — `finance/content.tsx` shows "Unpaid Invoices" stat + quick-link

## Schema

5 Prisma models:

- **`UserInvoice`** — primary record. Fields: `id`, `invoice_no`, `invoice_date`, `due_date`, `currency` (default `USD`), `sub_total` / `discount` / `tax_percentage` / `total` (Decimal), `notes`, `status` (`InvoiceStatus`, default `UNPAID`), `wizardStep` (null = complete). Relations: `userId`, `schoolId`, `journalEntryId`. Addresses: one-to-one `fromAddressId` + `toAddressId`. Indexes: `@@unique([schoolId, invoice_no])`, `[userId]`, `[schoolId]`, `[status]`.
- **`UserInvoiceItem`** — `id`, `item_name`, `quantity`, `price` (Decimal), `total` (Decimal), `invoiceId`, `schoolId`.
- **`UserInvoiceAddress`** — `id`, `name`, `email?`, `address1`, `address2?`, `address3?`, `schoolId`.
- **`UserInvoiceSettings`** — per-user settings (see Branding).
- **`UserInvoiceSignature`** — signature storage (see Branding).

```typescript
enum InvoiceStatus {
  PAID
  UNPAID    // default
  OVERDUE
  CANCELLED
}
```

Distinction: **`UserInvoice`** = school-level (this module). **`Invoice`** = Stripe platform billing (`@@map("invoices")`); the `Receipt` model links to `Invoice`, not `UserInvoice`.

## Configuration

Currency options: `USD`, `EUR`, `GBP`, `INR`, `CAD`, `AUD`. Wizard adds `SDG`, `SAR`, `AED`.

```typescript
STEP_FIELDS = {
  1: ["from", "to"],

  3: ["sub_total", "discount", "tax_percentage", "total", "notes"],
```

Number formats:

- Modal — `I{YY}{NNN}` (e.g., `I26042`), client-generated
- Auto-number — `INV-{NNNNNN}` (e.g., `INV-000042`), server sequential
- Enrollment — `INV-{NNNNNN}` same auto-number pattern
