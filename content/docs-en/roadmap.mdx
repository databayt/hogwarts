## Not on the roadmap (this year)

A few things look adjacent but are explicitly deferred. Listing them so contributors do not start speculative work.

- **Catalog split refactor.** Splitting `catalog-setup.ts` into per-curriculum configs is parked until a non-SD school onboards. Single DB table is fine for now.
- **Mobile API beyond current routes.** Existing routes (auth + transportation geofence-boarding) are enough. No broader mobile-first API surface this year.
- **Marketplace plans.** No instructor / content marketplace until cross-school catalog adoption signals real demand.
- **Hostel and cafeteria modules.** Dictionary mentions only; no schema, no UI, no roadmap entry.
- **Native iOS / Android apps.** Mobile web works today. React Native track only opens after PMF is clear.

## How decisions are made

- **Requirements:** [PRD](/docs/prd) defines scope, NFRs, and constraints. New work proves it fits these before it gets added here.
- **Shipped state:** [MVP](/docs/mvp) is the source of truth for what works today. If it conflicts with this doc, MVP wins for "what is" and roadmap wins for "what is next".
- **Commercial direction:** [Business model](/docs/business-model) shapes which initiatives get prioritised — finance and admission ship first because they tie to revenue.
- **Architecture:** [Architecture](/docs/architecture) is the canonical design doc. Initiatives that violate it (e.g. weakening multi-tenant isolation, bilingual columns, breaking the mirror pattern) are rejected.
- **Issues over chat.** Anything on this page is also tracked in GitHub issues. If a roadmap item has no issue, it is a draft and should not be picked up yet.
