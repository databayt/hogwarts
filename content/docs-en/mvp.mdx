## Maturity badges

| Badge | Meaning |
| ----- | ------- |
| `Built` | Feature is shipped end-to-end with backend, UI, i18n, and tests |
| `Built+Polish` | Feature is shipped but has known polish or migration work in flight |
| `In Progress` | Backend or UI partial; tracked actively |
| `Schema Only` | Prisma model exists, no UI or actions yet |
| `Vaporware` | String mentions in dictionaries but zero code |

## Epic 01 — Infrastructure

**Status:** `Built+Polish`

The multi-tenant foundation. Each school is a subdomain (`school.databayt.org`), resolved at the edge, isolated by `schoolId`. See [Architecture](/docs/architecture) and [Multi-tenancy](/docs/multi-tenancy).

- Multi-tenant Prisma schema with `schoolId` scoping on every tenant-owned model (302 models, 73 schema files).
- Subdomain routing via Next.js 16 edge `proxy.ts`.
- Custom-domain mapping via Upstash Redis (`custom-domain:${host}`).
- `getTenantContext()` resolves `schoolId` from the `x-subdomain` header for every server action.
- Vercel + Neon (PostgreSQL serverless) + AWS S3 + CloudFront (`cdn.databayt.org`).
- Sentry error monitoring (gracefully no-op without DSN).
- 11 daily cron jobs for housekeeping (notifications, reminders, fee overdue, cleanups).
- CI/CD: 245 dedicated test files, Playwright suites for auth / multi-tenant / RBAC / smoke.
- Edge bundle hygiene: `outputFileTracingExcludes` keeps serverless functions under Vercel's 300 MB limit.
- `experimental.serverActions.bodySizeLimit: "10mb"` for direct file uploads.

## Epic 02 — Authentication & Users

**Status:** `Built`

NextAuth v5 with email + password, OAuth (Google, Facebook), 2FA, full invitation flow, audit logging. See [Authentication](/docs/authentication).

- NextAuth v5 (beta.30) with JWT strategy and a multi-tenant Prisma adapter.
- Email + password (bcrypt cost 12).
- OAuth providers: Google, Facebook.
- Two-factor authentication via `TwoFactorToken` + `TwoFactorConfirmation` (`src/components/auth/verification/2f-token.ts`, `2f-confirmation.ts`).
- 8 roles: `DEVELOPER`, `ADMIN`, `TEACHER`, `STUDENT`, `GUARDIAN`, `ACCOUNTANT`, `STAFF`, `USER`.
- CASL abilities (`@casl/ability` + `@casl/prisma`) for fine-grained authorization.
- Route-level RBAC via role gates.
- Password reset flow with email verification.
- Email verification on signup.
- Session management with auto-renewal.
- User profile pages per role (admin / teacher / student / parent / staff variants).
- User invitation system (`src/lib/invitation-utils.ts`, `src/app/[lang]/accept-invite/page.tsx`, daily `/api/cron/expire-invitations` job).
- Audit logging for sensitive operations (`AuditLog` model, `src/lib/audit-log.ts`).
- Brute-force protection via `LoginAttempt` and rate limits.
- Compliance + legal consent models (`LegalConsent`, `LegalDocument`, `ComplianceLog`).

Schools customise branding, academic structure, and policies during an 18-step onboarding wizard. See [Onboarding](/docs/onboarding).

- 18-step onboarding wizard: `about-school`, `branding`, `capacity`, `description`, `discount`, `finish-setup`, `import`, `join`, `legal`, `location`, `price`, `schedule`, `stand-out`, `subdomain`, `title`, `visibility`, `congratulations`, `overview`.
- School profile (`School` model) with academic year, term, period.

- Timezone configuration.
- Domain request flow for custom domains.

## Epic 04 — People

**Status:** `Built`

CRUD for the four people types — students, teachers, parents, staff — with multi-step wizards and rich profiles. See [Students](/docs/students), [Teachers](/docs/teachers), [Listings](/docs/listings).

- **Students** — 12-step wizard (personal, contact, fees, enrollment, location, health, academic, previous-education, guardian, attachments, photo). Profile page with 8 tabs. CSV bulk import.
- **Teachers** — 9-step wizard (information, contact, employment, expertise, qualifications, experience, location, photo, attachments). Departments, schedule, performance.
- **Parents / Guardians** — multi-step add (information, contact). Link-child action wires guardians to students.
- **Staff** — full CRUD with `StaffMember`, `StaffAccessCard`, `StaffSalaryStructure`, `StaffSalarySlip`, `StaffTimesheetEntry`.
- Status transitions: `enrolled`, `transferred`, `graduated`.
- Bulk actions, individual + bulk CSV export.
- Photo uploads to S3 + CloudFront.

## Epic 05 — Admission

**Status:** `Built`

Public-facing admission portal with a 9-step application form, OTP status tracking, AI document review, merit ranking, multi-gateway fee payment, and one-click enrollment conversion. See [Admission](/docs/admission).

- **Public 9-step application** at `/[lang]/s/[subdomain]/application/...`: personal → contact → location → academic → fees → attachments → offer → payment → success.
- Auto-save with `ApplicationSession` resume.
- **OTP-verified status tracker** — applicants check status without an account.
- **Document upload** with attachments step.
- **AI document review** (`src/components/school-dashboard/admission/ai/`):
  - Anthropic-based document classification (`classify.ts`).
  - Completeness scoring (`completeness.ts`).
  - Merit engine for entrance / interview / merit / rank scoring (`merit-engine.ts`).
  - Bank-receipt extraction for offline-payment validation.
- **Tour booking** with `TourBooking` model and admin time-slot management.
- **Acceptance / offer flow** — `Application.admissionOffered`, `offerDate`, `offerExpiryDate`, `offerAccepted`.
- **Automatic enrollment conversion** — accepting an offer triggers `enrollStudentInGradeClasses` (`src/lib/enrollment-sync.ts`), creating Student + User + Guardian + StudentYearLevel + AcademicGrade + FeeAssignments + UserInvoices + StudentDocuments in one transaction.
- **Merit scoring + ranking** with admin merit table.
- **Campaigns** (`AdmissionCampaign`) with date-range calendar picker.
- **Multi-gateway fee payment** — see Epic 09.
- Admission analytics, settings, bulk placement, interview scheduling.
- Notifications (email + WhatsApp + push) at every status transition.
- 5 dedicated test files: `actions.test.ts`, `enrollment-notifications.test.ts`, `enrollment-pipeline.test.ts`, `offer-expiry.test.ts`, `validation.test.ts`.

- 33+ Prisma models split across `attendance.prisma` (16), `attendance-enhanced.prisma` (10), `geo-attendance.prisma` (3): `AccessCard`, `AttendanceDevice`, `AttendanceEvent`, `AttendancePolicy`, `AttendanceReport`, `AttendanceSession`, `BiometricTemplate`, `BluetoothBeacon`, `KioskSession`, `KioskLog`, `HallPass`, `QRCodeSession`, `MasterAttendance`, `GeoAttendanceEvent`, `GeoFence`, `LocationTrace`, plus competition / badge / streak / reward / intervention models for gamification.

- Trend analytics + monthly summaries + class / student reports.

## Epic 09 — Finance & Billing

**Status:** `Built`

Comprehensive finance module — double-entry bookkeeping, 6 payment gateways, Plaid + Dwolla banking, payroll, invoicing, scholarships, fines. Significantly larger than typical MVP scope. See [Fees](/docs/fees) and [Invoice](/docs/invoice).

- 36 Prisma models across 7 finance schema files.
- **Double-entry bookkeeping** — `ChartOfAccount`, `JournalEntry`, `LedgerEntry`, `AccountBalance`, `FiscalYear`.
- **6 payment gateways** — `src/lib/payment/providers/`: Stripe, bankak, tap, bank-transfer, cash, mobile-money. Pluggable provider interface in `provider.ts`.
- **Banking** — Plaid (`react-plaid-link`, `@plaid/plaid`) + Dwolla. `BankAccount`, `BankReconciliation`, `Transaction`, `Transfer`, `PlaidItem`, `DwollaCustomer`.
- **Fees** — `FeeStructure`, `FeeAssignment` with installment timeline, `Payment`, `Refund`, `Scholarship`, `ScholarshipApplication`, `Fine`.
- **Invoicing** — `UserInvoice` with line items, addresses, signature, and configurable settings (`UserInvoiceSettings`).
- **Receipts** — PDF receipt generation with `@react-pdf/renderer`.
- **Payroll** — `PayrollRun`, `SalaryStructure`, `SalarySlip`, `SalaryAllowance`, `SalaryDeduction`, `TimesheetEntry`, `TimesheetPeriod`.
- **Budget** — `Budget`, `BudgetAllocation`, `Expense`, `ExpenseCategory`, `Wallet`, `WalletTransaction`.
- **Subscriptions (school-level)** — `Subscription`, `Invoice`, `BillingHistory`, `BillingPaymentMethod`, `BillingPreferences`, `SubscriptionTier`, `UsageMetrics`, `Discount`, `AppliedDiscount`, `CreditNote`.
- **Reports** — `FinancialReport` plus dashboards.
- **Permissions** — `FinancePermission` for finance-staff scoping.

## Epic 14 — Catalog (global content)

**Status:** `Built+Polish`

Platform-wide educational content library — global subjects, chapters, lessons across multiple curricula, sourced and contributed. See [Catalog](/docs/catalog) and the ClickView trio: [elementary](/docs/clickview-elementary), [middle](/docs/clickview-middle), [high](/docs/clickview-high).

- **Community contribution workflow** with `Proposal` model (proposed → reviewed → approved → published).

- ClickView asset wave `Built+Polish` — see Epic 11.
