A snapshot of what's actually built today. For requirements see [PRD](/docs/prd); for direction see [Roadmap](/docs/roadmap). The MENA-10 pilot cohort runs on what is `Built` and `Built+Polish` below — see [Pilot](/docs/pilot) for the offer terms.

## Maturity badges

| Badge          | Meaning                                          |
| -------------- | ------------------------------------------------ |
| `Built`        | Shipped end-to-end with backend, UI, i18n, tests |
| `Built+Polish` | Shipped with known polish or migration in flight |
| `In Progress`  | Backend or UI partial; tracked actively          |
| `Schema Only`  | Prisma model exists, no UI yet                   |
| `Vaporware`    | Dictionary strings only, zero code               |

## 01 — Infrastructure · `Built+Polish`

The multi-tenant foundation. See [Architecture](/docs/architecture) and [Multi-tenancy](/docs/multi-tenancy).

- Hundreds of Prisma models across many schema files; `schoolId` scoping on every tenant model.
- Subdomain routing via Next.js 16 edge `proxy.ts`.
- Custom-domain mapping via Upstash Redis.
- Vercel + Neon (PostgreSQL serverless) + AWS S3 + CloudFront (`cdn.databayt.org`).
- Sentry monitoring; gracefully no-op without DSN.
- Daily cron jobs for notifications, reminders, fee overdue, cleanups.
- Playwright suites for auth, multi-tenant, RBAC, smoke.
- Bundle hygiene: `outputFileTracingExcludes` keeps serverless functions under Vercel's 300 MB limit.

## 02 — Authentication and users · `Built`

NextAuth v5 with email + OAuth + 2FA + invitations + audit. See [Authentication](/docs/authentication).

- NextAuth v5 (beta.30), JWT strategy, multi-tenant Prisma adapter.
- Email + password (bcrypt cost 12); OAuth Google, Facebook.
- 2FA via `TwoFactorToken` + `TwoFactorConfirmation`.
- 8 roles: `DEVELOPER`, `ADMIN`, `TEACHER`, `STUDENT`, `GUARDIAN`, `ACCOUNTANT`, `STAFF`, `USER`.
- CASL abilities (`@casl/ability` + `@casl/prisma`) for fine-grained authz.
- Password reset, email verification, session auto-renewal.
- Per-role profile pages (admin / teacher / student / parent / staff).
- Invitations: `src/lib/invitation-utils.ts`, `accept-invite/page.tsx`, daily `expire-invitations` cron.
- `AuditLog` model + brute-force protection via `LoginAttempt`.
- Compliance models: `LegalConsent`, `LegalDocument`, `ComplianceLog`.

## 03 — Configuration · `Built`

15-step onboarding wizard creates schools end-to-end. See [Onboarding](/docs/onboarding).

- Steps: about-school · title · description · location · stand-out · capacity · branding · import · finish-setup · join · visibility · price · discount · legal · subdomain · congratulations.
- `School` model with year, term, period; `YearLevel` / `AcademicLevel` / `AcademicGrade`.
- `StreamType`: `GENERAL`, `SCIENCE`, `ARTS`, `COMMERCE`, `TECHNICAL`, `RELIGIOUS`.
- Configuration editors at `school/configuration/` for branding, capacity, discount, domain, hero, legal, location, modules, name format, plan, price, schedule, title, visibility.
- Logo upload, color picker, theme (`UserTheme`), timezone.
- Custom-domain request flow.

## 04 — People · `Built`

CRUD for the four people types. See [Students](/docs/students), [Teachers](/docs/teachers), [Listings](/docs/listings).

| Entity              | Wizard steps | Highlights                                                                                                                                      |
| ------------------- | ------------ | ----------------------------------------------------------------------------------------------------------------------------------------------- |
| Students            | 12           | Personal, contact, fees, enrollment, location, health, academic, prior education, guardian, attachments, photo. 8-tab profile. CSV bulk import. |
| Teachers            | 9            | Information, contact, employment, expertise, qualifications, experience, location, photo, attachments.                                          |
| Parents / Guardians | Multi-step   | Information, contact. Link-child action.                                                                                                        |
| Staff               | Full CRUD    | `StaffMember`, `StaffAccessCard`, `StaffSalaryStructure`, `StaffSalarySlip`, `StaffTimesheetEntry`.                                             |

Status transitions: `enrolled`, `transferred`, `graduated`. Bulk export, photo uploads to S3 + CloudFront.

## 05 — Admission · `Built`

Public-facing portal with AI document review and automatic enrollment conversion. See [Admission](/docs/admission).

- 9-step public application: personal → contact → location → academic → fees → attachments → offer → payment → success.
- Auto-save with `ApplicationSession` resume.
- OTP-verified status tracker — applicants check status without an account.
- AI in `school-dashboard/admission/ai/`: `classify.ts`, `completeness.ts`, `merit-engine.ts`, bank-receipt extraction.
- Tour booking (`TourBooking`) with admin time-slot management.
- Acceptance flow: `admissionOffered`, `offerDate`, `offerExpiryDate`, `offerAccepted`.
- One-transaction enrollment conversion via `enrollStudentInGradeClasses`.
- Merit ranking; `AdmissionCampaign` with date-range picker.
- Multi-gateway fee payment — see Epic 09.
- Notifications (email + WhatsApp + push) on every status transition.

## 06 — Attendance · `Built+Polish`

Daily attendance with multiple capture modes. Section-centric migration in flight. See [Attendance](/docs/attendance).

- 11 `AttendanceMethod` modes: `MANUAL`, `GEOFENCE`, `QR_CODE`, `BARCODE`, `RFID`, `FINGERPRINT`, `FACE_RECOGNITION`, `NFC`, `BLUETOOTH`, `BULK_UPLOAD`, `KIOSK`.
- Many routes including manual, kiosk, bulk-upload, QR, barcode, geofence, hall-pass, intentions, excuses, letters (auto-gen), interventions, gamification, early-warning, AI, analytics, reports.
- Models split across `attendance.prisma`, `attendance-enhanced.prisma`, `geo-attendance.prisma`.
- Status: present / absent / late / excused; parent absence notifications + late-arrival tracking.
- Trend analytics, monthly summaries, class / student reports.
- **Section-centric migration `In Progress`** — see `.bmad/planning/architecture-section-timetable-attendance.md`.

## 07 — Exams and grading · `Built`

End-to-end assessment lifecycle. See [Exams](/docs/exams) and [Exam wizard](/docs/exam-wizard).

- Many models across `exam.prisma`, `school-exam.prisma`, `school-qbank.prisma`, `quiz.prisma`, `quiz-game.prisma`, `quick-assessments.prisma`, `grading-scheme.prisma`, `school-assignment.prisma`, `progress-reports.prisma`.
- Exams scheduled by type: midterm, final, quiz, mock.
- Question bank with AI generation, source-material chunking, analytics, review workflow.
- Paper generation with cover / header / footer / question-type templates.
- Marking with rubrics + criteria + grade boundaries + overrides.
- Report cards, grades, transcripts (PDF + CSV).
- Certificates with verify-by-code at `/certificate/[shareToken]` and `/verify/[code]`.
- Promotion workflow: `PromotionBatch`, `PromotionCandidate`, `PromotionPolicy`, `Transcript`.

## 08 — Timetable · `Built`

Visual builder with conflict detection. See [Timetable](/docs/timetable).

- Drag-and-drop period scheduling with break management.
- Room allocation with `RoomConstraint`.
- Conflict detection across teachers, rooms, students.
- Per-teacher / per-class / per-room views.
- Substitutions (`SubstitutionRecord`); teacher constraints + unavailable blocks.
- Generation engine with conflict resolution; templates (`TimetableTemplate`, `TemplateApplication`).
- Schedule exceptions, analytics, PDF / print export.

## 09 — Finance and billing · `Built`

Comprehensive finance — much larger than typical MVP. See [Fees](/docs/fees) and [Invoice](/docs/invoice).

- Many models across multiple finance schema files.
- **Double-entry bookkeeping** — `ChartOfAccount`, `JournalEntry`, `LedgerEntry`, `AccountBalance`, `FiscalYear`.
- **6 payment gateways** in `src/lib/payment/providers/`: Stripe, bankak, tap, bank-transfer, cash, mobile-money.
- **Banking** — Plaid + Dwolla; `BankAccount`, `BankReconciliation`, `Transaction`, `Transfer`, `PlaidItem`, `DwollaCustomer`.
- **Fees** — `FeeStructure`, `FeeAssignment` with installment timeline, `Payment`, `Refund`, `Scholarship`, `ScholarshipApplication`, `Fine`.
- **Invoicing** — `UserInvoice` with line items, addresses, signature, settings.
- **Receipts** — PDF via `@react-pdf/renderer`.
- **Payroll** — `PayrollRun`, `SalaryStructure`, `SalarySlip`, `SalaryAllowance`, `SalaryDeduction`, `TimesheetEntry`, `TimesheetPeriod`.
- **Budget** — `Budget`, `BudgetAllocation`, `Expense`, `ExpenseCategory`, `Wallet`, `WalletTransaction`.
- **School-level subscriptions** — `Subscription`, `Invoice`, `BillingHistory`, `BillingPaymentMethod`, `BillingPreferences`, `SubscriptionTier`, `UsageMetrics`, `Discount`, `AppliedDiscount`, `CreditNote`.
- **Reports + permissions** — `FinancialReport`, `FinancePermission`.

## 10 — Communication and messaging · `Built`

Announcements, real-time messages, notifications, WhatsApp. See [Messages](/docs/messages).

- **Announcements** (`announcement.prisma`): scope (school / class / role), priority, scheduled publishing, expiry, pinned / featured, read receipts, templates.
- **Messages** (`messages.prisma`): `Conversation`, `ConversationInvite`, `ConversationParticipant`, `Message`, `MessageAttachment`, `MessageDraft`, `MessageReaction`, `MessageReadReceipt`, `PinnedMessage`, `StarredMessage`, `TypingIndicator`. Realtime via custom Socket.io server (`server.js`).
- **Notifications** (`notifications.prisma`): in-app + email + push.
- **WhatsApp** (`whatsapp.prisma`) — full Evolution API bridge with retry + exponential backoff.
- React Email templates in `src/lib/email-templates/`.
- Daily cron jobs deliver scheduled comms.

## 11 — Stream / LMS · `Built+Polish`

School-scoped LMS. US cover-asset wave partial. See [US curriculum](/docs/catalog#us-curriculum).

- Models in `stream.prisma`: `StreamCourse`, `StreamCategory`, `StreamChapter`, `StreamLesson`, `StreamEnrollment`, `StreamLessonProgress`, `StreamCertificate`, `StreamAttachment`.
- Admin course CRUD + categories; Tiptap rich-text editor.
- Video player with progress; student enrollment + completion certificates.
- Stripe payment flow (cancel / success); per-school storage quota (`videoStorageUsedBytes` / `videoStorageQuotaBytes`).
- **source CDN partial** — a small fraction of lessons downloaded; rest fall back to source CDN. Tracked in [Roadmap](/docs/roadmap).

## 12 — Library · `Built`

Catalog, contribute, borrow records, school admin. See [Library](/docs/library).

- Components in `src/components/library/`: admin, book-detail, book-list, catalog, contribute, my-profile.
- Models: `Book`, `Textbook`, `SchoolBook`, `BorrowRecord`.
- Browse + search + checkout for students; admin CRUD + circulation reports.
- Per-school via `SchoolBook` while leveraging the global `Book` / `Textbook` catalog.

## 13 — Transport · `Built`

Drivers, vehicles, routes, geofence boarding, parent notifications. **Production-ready as of 2026-05.**

- 9 Prisma models (`transportation.prisma`): `Vehicle`, `Driver`, `Route`, `RouteStop`, `RouteAssignment`, `Trip`, `TripBoarding`, `TransportationSettings`, `SchoolApiToken`.
- 5 surfaces: writable Settings, `/transportation/me` (STUDENT/GUARDIAN), `/transportation/fees` (ACCOUNTANT), POST `/api/transportation/geofence-boarding` (Bearer-token webhook), plus the admin/staff console (drivers, vehicles, routes, trips, assignments, reports).
- dnd-kit drag-and-drop stop ordering.
- Geofence-linked boarding with parent notifications on board / disembark.
- Mapbox + Leaflet for route visualisation.
- Trip reports: utilisation, hours, trip stats, fee preview.
- Demo seed: 5 vehicles, 5 drivers, 5 routes × 4 stops, 147 assignments, 92 trips, 1092 boardings.
- 67/67 tests green.
- Block guide at `src/components/school-dashboard/transportation/CLAUDE.md`.

## 14 — Catalog · `Built+Polish`

Platform-wide educational content library. See [Catalog](/docs/catalog).

- Many models across `catalog.prisma`, `curriculum.prisma`, `chapter.prisma`, `lesson.prisma`, `material.prisma`, `book.prisma`, `textbook.prisma`, `video.prisma`, `image.prisma`, `document.prisma`, `assignment.prisma`, `qbank.prisma`, `quiz.prisma`, `bridge.prisma`, plus `proposal.prisma`, `enrollment.prisma`.
- **Bridge pattern** (`bridge.prisma`) — schools subscribe to a shared catalog and apply `ContentOverride`, `BookSelection`, `SubjectSelection`, `InstructorPreference`.
- 4 curricula on disk: SD (Sudan, 12 grades, bilingual subjects), US, IN, FR.
- Operator dashboard at `/[lang]/(saas-dashboard)/(catalog)/catalog/...`.
- Community contribution workflow with `Proposal` (proposed → reviewed → approved → published).
- US-curriculum source library: [overview](/docs/catalog#us-curriculum) — 62 subjects, 240 grade-specific catalog records, three tiers (Elementary/Middle/High).
