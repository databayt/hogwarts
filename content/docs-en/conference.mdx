## Decisions

- **One block, one provider abstraction.** External is the floor every school gets; LiveKit is the
  opt-in premium ceiling, gated by `isLiveKitConfigured()`. LiveKit's SFU lifecycle stays out of the
  `providers/` adapter layer (it's not link-based).
- **Rename is DB-transparent.** `@@map`/`@map` keep every table/column/enum-type name.
- **Webhook + crons stay API routes** тАФ external/scheduler-invoked; a server action can't receive them.
- **Pending (non-code):** LiveKit infra provisioning (runbook ready), native OAuth credentials, the
  in-room kick button (LiveKit prebuilt has no slot).
