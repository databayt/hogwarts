One self-contained block at `src/components/school-dashboard/conference/`, mirrored 1:1 to the
`/conference` route. It runs three meeting back-ends behind a single UI: the **external
pasted-link** provider is live everywhere; the **LiveKit SFU** (in-app rooms + recording) is fully
coded but dormant until infra is provisioned; the **native Meet / Zoom / Teams** adapters create
real meetings via each vendor's API but ship dark until OAuth credentials land.

Routes mirror this block (see [Routes](#routes)). The 4 Prisma models + 1 recurring-link model live

`__tests__/` folder inside the block.

## Components

Two permission/validation layers coexist on purpose: the **rich sessions layer**
(`authorization.ts` ¬∑ `permissions.ts` ¬∑ `validation.ts`) is the strict runtime gate for the LiveKit
room flow (join / token / start / end), while the **list layer** (`list-*.ts`) is the CRUD gate for
the dashboard table and external-link sessions.

## Data flow

```
index (/conference)            content.tsx (server) ‚Üí table.tsx (client) ‚Üí form.tsx (modal create)
   ‚îî‚îÄ row Join / detail
detail (/conference/[id])      detail.tsx ‚Üí Join (live/scheduled) | Recordings (ended)
   ‚îî‚îÄ Join
room  (/conference/[id]/room)  joinLiveClass ‚Üí eligibility ‚Üí 5-min JWT ‚Üí room.tsx (LiveKit prebuilt)
                                              ‚Üë refreshLiveClassToken every ~60s re-runs eligibility
recordings (/[id]/recordings)  recordings.tsx ‚Üí recording-player.tsx (signed S3 URL, re-signed/4min)
```

- **Session state machine** (`actions/sessions.ts`): `scheduled ‚Üí live ‚Üí ended`, plus `cancelled`
  and `failed`. `startLiveClass` provisions the SFU room (checks `conferenceMaxConcurrent`);
  `endLiveClass` is idempotent. The **webhook is the authoritative writer** for `live`
  (`room_started`) and `ended` (`room_finished`).
- **Eligibility** is resolved lazily per join (`actions/tokens.ts`): TEACHER‚ÜíHOST (own session) /
  CO_HOST, STUDENT‚ÜíPARTICIPANT (section match), GUARDIAN‚ÜíOBSERVER (child in section), ADMIN‚ÜíCO_HOST,
  DEVELOPER‚ÜíHOST. Kicked users (`status="removed"`) are blocked at the next token refresh.
- **Recordings** (`pending ‚Üí processing ‚Üí ready ‚Üí expired`): LiveKit Egress ‚Üí S3 `me-central-1`;
  per-school retention enforced by the daily cron.
- **Notifications** (`actions/notifications.ts`): all 5 `live_class_*` events resolve their audience
  (teacher + section roster + guardians) and dispatch through the shared hub
  (`dispatchNotificationsToAudience`), so they reach the **in-app bell and email**, honour per-user
  channel preferences, and carry an absolute action URL. Both the in-app room path (`sessions.ts`) and
  the external pasted-link path (`list-actions.ts` create / reschedule / cancel) fan out.
- **Attendance** (`actions/attendance-sync.ts`, opt-in via `School.conferenceAttendanceSync`): when a
  session ends (webhook `room_finished` or the stale-session cron), `syncConferenceAttendance` marks
  each section student PRESENT/LATE from their in-room presence and ABSENT for roster non-joiners,
  writing `Attendance` rows with method `VIRTUAL`. Requires a `sectionId` + `timetableId`; **LiveKit
  only** ‚Äî external links carry no presence data. Idempotent (section unique key, revive-on-update).

The room route lives under a separate **`(live-room)`** group with a bare full-viewport layout (no
sidebar / dashboard chrome) so the SFU stream owns the screen; it is `force-dynamic` for OOM-safe
builds. API (external / scheduler-invoked ‚Üí stay routes, not server actions): `api/webhooks/livekit`
(HMAC), `api/cron/live-class-reminders` (every 5 min), `api/cron/end-stale-live-classes` (every 15 min
‚Äî closes sessions stuck `live` past their end + runs the attendance sync), `api/cron/expire-live-recordings` (daily).

### Access control ‚Äî private vs school-wide

`visibility` decides who may see and join a session, on top of the absolute tenant boundary:

- **`section`** (default) ‚Äî the session's section roster joins as PARTICIPANT, their guardians as
  OBSERVER; a session with no section is host-only (a private meeting).
- **`school`** ‚Äî every member of the school: students PARTICIPANT, guardians OBSERVER,
  staff/accountant PARTICIPANT. For assemblies, PD webinars, parent town halls. Scheduling one
  notifies the whole school through the notification hub.

There is deliberately **no cross-school or anonymous tier** ‚Äî eligibility, list scoping, detail
reads, and recordings access all enforce the same rule (`resolveParticipantRole`,
`canAccessSession`, `buildLiveClassWhere`). Meeting and reference URLs are scheme-locked to
http(s) ‚Äî `javascript:`/`data:` URIs are rejected at validation.

### References ‚Äî lesson, quiz, assignment, links

One `catalogLessonId` FK links a session to a catalog lesson; the session page then surfaces the
lesson's **videos, attachments, materials, and practice-question count** automatically.
Quizzes/exams (`SchoolExam` ‚Äî a quiz is `examType: QUIZ`, not a separate model), assignments, and
ad-hoc links attach as `ConferenceResource` rows ‚Äî tenant-verified before write, replaced as a set
on update. The wizard's References step loads its pickers per chosen subject
(`getLiveClassReferenceOptions`, staff-gated).

### The create wizard

The dashboard form is a 5-step compact wizard (`form.tsx` + `form-steps.tsx`) on the house
stepped-modal pattern ‚Äî **Basics ‚Üí Schedule ‚Üí Meeting ‚Üí References ‚Üí Access**, a handful of fields
per step. The Meeting step picks the back-end: **In-app room** (LiveKit ‚Äî default whenever
`isLiveKitConfigured()`, with recording + max-participants knobs) or **External link** (pasted
Meet/Zoom/Teams URL + optional weekly-recurring default). Provider is immutable after creation;
Join is provider-aware everywhere (table menu, detail page, and the room route hands external
sessions to their vendor URL).

Each native adapter creates a real meeting through the vendor API (`createMeeting`), gated by

persist as `provider="external"` + `meetingProvider="<id>"` ‚Üí **no enum migration** to ship.

## Decisions

- **One block, one provider abstraction.** External is the floor every school gets; LiveKit is the
  opt-in premium ceiling, gated by `isLiveKitConfigured()`. LiveKit's SFU lifecycle stays out of the
  `providers/` adapter layer (it's room-based, not link-based).
- **Rename is DB-transparent.** `@@map`/`@map` keep every table / column / enum-type name.
- **Webhook + crons stay API routes** ‚Äî external / scheduler-invoked; a server action can't receive them.
- **Token TTL = 5 min, client-refreshed.** LiveKit JWTs are stateless, so access revocation takes
  effect at the next refresh boundary (‚â§5 min), not instantly; kicks are enforced via the
  `status="removed"` gate at join.
- **Pending (non-code):** LiveKit infra provisioning (runbook ready), native OAuth credentials.
