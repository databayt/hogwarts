One self-contained block at `src/components/school-dashboard/conference/`, mirrored 1:1 to the
`/conference` route. It runs three meeting back-ends behind a single UI: the **external
pasted-link** provider is live everywhere; the **LiveKit SFU** (in-app rooms + recording) is fully
coded but dormant until infra is provisioned; the **native Meet / Zoom / Teams** adapters create
real meetings via each vendor's API but ship dark until OAuth credentials land.

Routes mirror this block (see [Routes](#routes)). The 4 Prisma models + 1 recurring-link model live

`__tests__/` folder inside the block.

## Components

Two permission/validation layers coexist on purpose: the **rich sessions layer**
(`authorization.ts` · `permissions.ts` · `validation.ts`) is the strict runtime gate for the LiveKit
room flow (join / token / start / end), while the **list layer** (`list-*.ts`) is the CRUD gate for
the dashboard table and external-link sessions.

## Data flow

```
index (/conference)            content.tsx (server) → table.tsx (client) → form.tsx (modal create)
   └─ row Join / detail
detail (/conference/[id])      detail.tsx → Join (live/scheduled) | Recordings (ended)
   └─ Join
room  (/conference/[id]/room)  joinLiveClass → eligibility → 5-min JWT → room.tsx (LiveKit prebuilt)
                                              ↑ refreshLiveClassToken every ~60s re-runs eligibility
recordings (/[id]/recordings)  recordings.tsx → recording-player.tsx (signed S3 URL, re-signed/4min)
```

- **Session state machine** (`actions/sessions.ts`): `scheduled → live → ended`, plus `cancelled`
  and `failed`. `startLiveClass` provisions the SFU room (checks `conferenceMaxConcurrent`);
  `endLiveClass` is idempotent. The **webhook is the authoritative writer** for `live`
  (`room_started`) and `ended` (`room_finished`).
- **Eligibility** is resolved lazily per join (`actions/tokens.ts`): TEACHER→HOST (own session) /
  CO_HOST, STUDENT→PARTICIPANT (section match), GUARDIAN→OBSERVER (child in section), ADMIN→CO_HOST,
  DEVELOPER→HOST. Kicked users (`status="removed"`) are blocked at the next token refresh.
- **Recordings** (`pending → processing → ready → expired`): LiveKit Egress → S3 `me-central-1`;
  per-school retention enforced by the daily cron.
- **Notifications** (`actions/notifications.ts`): all 5 `live_class_*` events resolve their audience
  (teacher + section roster + guardians) and dispatch through the shared hub
  (`dispatchNotificationsToAudience`), so they reach the **in-app bell and email**, honour per-user
  channel preferences, and carry an absolute action URL. Both the in-app room path (`sessions.ts`) and
  the external pasted-link path (`list-actions.ts` create / reschedule / cancel) fan out.
- **Attendance** (`actions/attendance-sync.ts`, opt-in via `School.conferenceAttendanceSync`): when a
  session ends (webhook `room_finished` or the stale-session cron), `syncConferenceAttendance` marks
  each section student PRESENT/LATE from their in-room presence and ABSENT for roster non-joiners,
  writing `Attendance` rows with method `VIRTUAL`. Requires a `sectionId` + `timetableId`; **LiveKit
  only** — external links carry no presence data. Idempotent (section unique key, revive-on-update).

## Routes

| Route                         | Layout           | Renders                                              | Roles                                         |
| ----------------------------- | ---------------- | ---------------------------------------------------- | --------------------------------------------- |
| `/conference`                 | school-dashboard | DataTable index                                      | all 7 school roles                            |
| `/conference/[id]`            | school-dashboard | `detail.tsx`                                         | all 7 school roles                            |
| `/conference/[id]/recordings` | school-dashboard | `recordings.tsx`                                     | all except ACCOUNTANT (academic, not finance) |
| `/conference/schedule`        | school-dashboard | `schedule-form.tsx`                                  | DEVELOPER · ADMIN · TEACHER                   |
| `/conference/settings`        | school-dashboard | `settings-form.tsx` + `section-recording-policy.tsx` | DEVELOPER · ADMIN                             |
| `/conference/network-test`    | school-dashboard | `network-test.tsx`                                   | DEVELOPER · ADMIN (env-gated)                 |
| `/conference/[id]/room`       | **(live-room)**  | `room.tsx`                                           | any authenticated session participant         |
| `/live-classes/*`             | —                | redirect → `/conference`                             | legacy compatibility shim (pre-rename links)  |

The room route lives under a separate **`(live-room)`** group with a bare full-viewport layout (no
sidebar / dashboard chrome) so the SFU stream owns the screen; it is `force-dynamic` for OOM-safe
builds. API (external / scheduler-invoked → stay routes, not server actions): `api/webhooks/livekit`
(HMAC), `api/cron/live-class-reminders` (every 5 min), `api/cron/end-stale-live-classes` (every 15 min
— closes sessions stuck `live` past their end + runs the attendance sync), `api/cron/expire-live-recordings` (daily).

## Schema — `prisma/models/conference.prisma`

`LiveClass*` → `Conference*`; every DB table / column / enum-type name preserved via `@@map`/`@map`
→ **zero-change migration** (byte-identical `migrate diff` vs baseline).

| Model                   | Purpose                                                                           |
| ----------------------- | --------------------------------------------------------------------------------- |
| `Conference`            | A session. `provider` (`livekit`\|`external`), `meetingUrl?`, `meetingProvider?`. |
| `ConferenceParticipant` | One row per invited user + connection telemetry (RTT, TCP-fallback flag).         |
| `ConferenceRecording`   | Egress metadata + S3 location + retention `expiresAt`.                            |
| `ConferenceEvent`       | Webhook audit log + `eventId @unique` idempotency.                                |
| `ConferenceLink`        | Set-once recurring link `[schoolId, subjectId, sectionId, termId]`.               |

Enums `ConferenceProvider` · `ConferenceStatus` · `ConferenceRecordingStatus` ·
`ConferenceParticipantRole` · `ConferenceParticipantStatus`; 4 `School.conference*` config columns
(`conferenceRetentionDays` 90 · `conferenceMaxConcurrent` 50 · `conferenceMaxDuration` 120 ·
`conferenceRecordingDefault` true).

## Providers

| Provider        | Status                         | Missing                                          |
| --------------- | ------------------------------ | ------------------------------------------------ |
| External link   | ✅ Live                        | — (universal zero-infra default)                 |
| LiveKit SFU     | 🟡 Coded, dormant              | 6 infra gates → `conference/RUNBOOK.md`          |
| Google Meet     | 🟡 `createMeeting` wired, dark | Google Cloud OAuth creds (`GOOGLE_MEET_*`)       |
| Zoom            | 🟡 `createMeeting` wired, dark | Server-to-Server OAuth creds (`ZOOM_*`)          |
| Microsoft Teams | 🟡 `createMeeting` wired, dark | Azure AD creds (`AZURE_*`, Graph OnlineMeetings) |

Each native adapter creates a real meeting through the vendor API (`createMeeting`), gated by
`isConfigured()`; `getRecording`/`getAttendance` still throw `ProviderNotImplementedError`. Natives
persist as `provider="external"` + `meetingProvider="<id>"` → **no enum migration** to ship.

## Decisions

- **One block, one provider abstraction.** External is the floor every school gets; LiveKit is the
  opt-in premium ceiling, gated by `isLiveKitConfigured()`. LiveKit's SFU lifecycle stays out of the
  `providers/` adapter layer (it's room-based, not link-based).
- **Rename is DB-transparent.** `@@map`/`@map` keep every table / column / enum-type name.
- **Webhook + crons stay API routes** — external / scheduler-invoked; a server action can't receive them.
- **Token TTL = 5 min, client-refreshed.** LiveKit JWTs are stateless, so access revocation takes
  effect at the next refresh boundary (≤5 min), not instantly; kicks are enforced via the
  `status="removed"` gate at join.
- **Pending (non-code):** LiveKit infra provisioning (runbook ready), native OAuth credentials.
