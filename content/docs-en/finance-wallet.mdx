---
title: Wallet
description: Cashless stored-value wallets for students and staff — top-ups, balances, and spending. Wallet CRUD and top-ups work, but top-ups don't post to the ledger, there is no surfaced refund workflow, and PIN/QR payments are not built.
---

Wallet gives each user a stored-value balance for cashless school transactions — top up, hold a balance, and (eventually) pay fees from it. The UI lives in `content.tsx`; there are no dedicated table/columns/form components yet.

## Flow

```
Create wallet  →  top up (payment method)  →  hold balance  →  spend (e.g. pay fees)
```

## Server actions

| Action         | Purpose                                  |
| -------------- | ---------------------------------------- |
| `createWallet` | Create a wallet for a user               |
| `getWallets`   | List wallets (tenant-scoped)             |
| `topupWallet`  | Add funds to a wallet                    |
| `refundWallet` | Reverse funds (no surfaced workflow yet) |

## Ledger / accounting

**Does not post to the ledger.** `postWalletTopup` exists in the engine but has zero callers, so top-ups do not create the `DR Cash / CR Unearned Revenue` entries the README describes. Reversing an overpaid top-up requires a manual journal entry in [Accounts](/docs/finance-accounts).

## Schema

`prisma/models/finance-budgets.prisma`: `Wallet`, `WalletTransaction` — both `schoolId`-scoped. No `lang` field.
