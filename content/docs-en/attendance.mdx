## Overview

The Attendance module provides a comprehensive system for tracking student attendance. It supports multiple tracking methods (manual, QR code, barcode, bulk upload, geofence), parent engagement features, early warning systems for chronic absenteeism, policy enforcement, and flexible reporting with CSV/Excel/PDF export.

## Role-Based Access Control

All attendance server actions enforce RBAC via `authorization.ts`. Teachers are scoped to their assigned classes only.

### Teacher Scoping

Teachers see **only their assigned classes** (via `Class.teacherId` + `ClassTeacher` bridge table). This applies to:

- Class selection dropdown (`getClassesForSelection()`)
- Dashboard stats (`getTodaysDashboard()`)
- Analytics queries (`getClassComparisonStats()`, `getStudentsAtRisk()`)
- Follow-up students (`getFollowUpStudents()`)
- Unmarked classes (`getUnmarkedClasses()`)
- Reports and CSV export

The helper `getTeacherClassIds()` in `actions/helpers.ts` combines primary teacher assignments and co-teacher roles from the `ClassTeacher` bridge table.

### Student/Guardian Scoping

- **Students** see only their own attendance via `getStudentOwnAttendance()`
- **Guardians** see only their linked children's attendance via `getGuardianChildrenAttendance()` and `getParentAttendanceSummary()`

## Grade-Based Filtering

Several actions accept an optional `gradeId` parameter to filter classes by academic grade:

- `getClassesForSelection({ gradeId })` -- filter class dropdown by grade
- `getClassComparisonStats({ gradeId })` -- compare classes within a grade
- `getStudentsAtRisk({ gradeId })` -- at-risk students in a grade

The filter works through `Class.gradeId` → `AcademicGrade`, using the `getClassIdsByGrade()` helper in `actions/helpers.ts`. When combined with teacher scoping, the grade filter intersects with the teacher's class list.

## Quick Start

Navigate to **Attendance** in the school dashboard sidebar.

### Overview Dashboard (Staff)

The staff overview page shows today's snapshot:

- **Quick Stats** -- total students, marked today, present/absent/late counts, attendance rate
- **Unmarked Classes** -- classes that haven't been marked yet today
- **Follow-Up Needed** -- students requiring attention (chronic absence, new absences)
- **Recent Activity** -- latest attendance events

### Overview (Student/Guardian)

Students and guardians see their own attendance summary with percentage, calendar view, and status breakdown.

## Taking Attendance

### Manual Attendance

The primary attendance marking method. Navigate to **Attendance > Manual**.

**Toolbar:** Flat inline toolbar (PlatformToolbar pattern) with class selector, date picker, bulk status buttons, and Save -- all on one row.

**Smart Class Selection:** The system calls `getCurrentPeriod()` to auto-select the class matching the current timetable period. A badge shows the auto-selected period name and subject.

**Locale-Aware Names:** Student names stored in the school's preferred language are translated on-demand via `getDisplayText()` when the user's locale differs. Translations are cached in the database.

**Keyboard Shortcuts:**

- `P` -- Mark all present
- `A` -- Mark all absent
- `L` -- Mark all late
- `Ctrl/Cmd+S` -- Save attendance

**Steps:**

1. Select a class (auto-selected from timetable if available)
2. Select the date (defaults to today)
3. Use the status dropdown per student or bulk buttons (All Present / All Absent / All Late)
4. Click **Save Attendance** -- absent students' guardians are automatically notified

**Auto-Excuse:** If a student has an approved `AbsenceIntention` covering the date, marking them absent automatically converts the status to `EXCUSED`.

### Period-by-Period Tracking

For secondary schools with timetable-based attendance:

- **Period Selector** -- lists all periods for the class and day, shows which are already marked, highlights current
- **Auto-Detection** -- `getCurrentPeriod()` identifies the active period from the school timetable
- **Period Analytics** -- view attendance patterns by period and subject

### QR Code Sessions

Navigate to **Attendance > QR Code**.

1. Select a class
2. Set validity period (default: 30 minutes)
3. Click **Generate** to create a unique QR code
4. Students scan to check in -- enrollment validated, duplicates prevented

### Barcode Scanning

Navigate to **Attendance > Barcode** (accessible via Bulk method picker).

1. Select a class
2. Scan student ID barcodes using a scanner or device camera
3. System looks up `StudentIdentifier` and marks attendance

### Bulk Upload

Navigate to **Attendance > Bulk Upload**.

1. Download the CSV template
2. Fill in records: `studentId,date,status,checkInTime,checkOutTime,notes`
3. Upload and review the import summary

### Geofence Tracking

Navigate to `/attendance/geo`.

- Students see a location tracker
- Admins/teachers see geofence management with live map, form, and zone list
- Location-based check-in/check-out

### Master Gate Attendance

School gate entry/exit tracking via server actions:

- `recordMasterAttendance()` -- record gate entry/exit
- `getMasterAttendanceForDay()` -- day's gate records
- `getPrefillFromMaster()` -- pre-fill class attendance from gate data

## Parent Engagement

### Absence Notifications

When a student is marked absent, `triggerAbsenceNotification()` fires:

- **In-app notification** -- visible in parent portal with high priority
- **SMS** -- via Twilio (if configured and guardian has phone number)
- **Email** -- via notification system

Notifications include student name, class, date, and use the school's `preferredLanguage`.

### Parent Excuse Submission

Parents access the **Excuses** tab to submit and track excuses.

**Excuse Reasons:**

- Medical, Family Emergency, Religious Observance, School Activity, Transportation Issue, Weather, Other

**Workflow:**

1. Parent sees unexcused absences
2. Clicks "Submit Excuse" -- provides reason, description, attachments
3. Staff reviews in the Excuses tab
4. **Approved** → attendance updated to `EXCUSED` / **Rejected** → remains unexcused

### Student/Guardian View

Students and guardians see:

- Attendance percentage with trend indicator
- Calendar view of attendance history
- Status breakdown (present, absent, late, excused)
- Excuse submission for unexcused absences

## Early Warning System

### Chronic Absenteeism Detection

**Risk Levels (US DoE Guidelines):**

| Risk Level         | Attendance Rate | Absence Rate |
| ------------------ | --------------- | ------------ |
| Satisfactory       | ≥95%            | Less than 5% |
| At-Risk            | 90-94.9%        | 5-9.9%       |
| Moderately Chronic | 80-89.9%        | 10-19.9%     |
| Severely Chronic   | Less than 80%   | ≥20%         |

Dashboard features:

- Students grouped by risk level
- Trend analysis (improving/declining)
- Class-level and school-level views

### Intervention Tracking

Dedicated page at `/attendance/interventions` with stats, filters, search, and pagination.

**14 Intervention Types:**
Parent phone call, Parent email, Parent meeting, Home visit, Counselor referral, Social worker referral, Administrator meeting, Attendance contract, Truancy referral, Community resource, Academic support, Mentorship assignment, Incentive program, Other

**5 Statuses:** Scheduled → In Progress → Completed / Cancelled / Escalated

**Workflow:**

1. Create intervention -- select type, set priority (1-4), assign staff
2. Schedule -- set date and follow-up dates
3. Track progress -- update status and outcome
4. Escalate if needed -- creates new intervention linked to original

### Policy Engine

Automated policy enforcement via `actions/policy.ts`:

- `evaluatePolicies()` -- check student against absence thresholds
- `getPolicyTriggers()` -- get triggered policies for a student
- `createPolicyExemption()` -- exempt student from specific policy
- `dismissPolicyTrigger()` -- dismiss a triggered alert

Server actions for scheduled compliance reporting:

## Settings

Settings page at `/attendance/settings` (Admin/Developer only).

> **Note:** The settings UI is rendered but the Save button is not connected to any server action. Settings changes are not persisted.

Displayed settings sections:

- **General** -- default check-in time, late threshold, methods, period mode
- **QR Code** -- session duration, location verification, single use
- **Notifications** -- email/SMS toggle, notification delay
- **Early Warning** -- at-risk threshold (90%), chronic threshold (80%), alert recipients
- **Security** -- advanced security options
- **Advanced** -- additional configuration

## Known Issues

### Fixed

- **RBAC gaps** -- All 67 server actions now enforce role checks via `authorization.ts`
- **Teacher class scoping** -- Teachers see only their assigned classes (primary + co-teacher)
- **Grade filtering** -- `getClassesForSelection()`, `getClassComparisonStats()`, `getStudentsAtRisk()` accept optional `gradeId`
- **Missing `deletedAt` filter** -- `getStudentOwnAttendance()`, `getGuardianChildrenAttendance()`, `getRecentAttendance()`, `getClassComparisonStats()`, `getTodaysDashboard()`, `getFollowUpStudents()` now exclude soft-deleted records
- **Attendance rate formula** -- `getStudentOwnAttendance()` fixed from `present / total` to `(present + late) / total`
- **Guardian attendance take limit** -- increased from 90 to 500 records with `schoolId` + `deletedAt` filter on nested query
- **Null-safe subject name** -- `getStudentOwnAttendance()` uses `r.class.subject?.subjectName` to avoid crash when subject is null

### Remaining

1. **Settings page not wired** -- UI renders but Save button has no action. All controls use `defaultValue` with no state management.
2. **Teacher dashboard quick-link** -- points to `/attendance/mark` which doesn't exist (should be `/manual`). See `src/components/school-dashboard/dashboard/config.ts`.
3. **Bulk method picker** -- "Live Tracking" card has empty `href`.
4. **Components without routes** -- AI predictions, gamification, hall-pass, kiosk, compliance letters, and absence intentions have full component implementations but no route pages exposing them.

## Best Practices

### For Teachers

1. Mark attendance at the start of each class/period
2. Let smart selection auto-pick your current class
3. Use keyboard shortcut `P` then adjust exceptions
4. Use `Ctrl/Cmd+S` to save quickly
5. Review weekly reports for patterns
6. Respond to excuse submissions promptly

### For Administrators

1. Monitor the overview dashboard daily for unmarked classes
2. Review the At Risk tab in analytics weekly
3. Follow up on students below 80% attendance
4. Track interventions to completion in `/interventions`
5. Export monthly reports for records
6. Configure notification settings when settings page is wired

### For Parents

1. Check attendance in the overview tab
2. Submit excuses promptly with documentation via the Excuses tab
3. Respond to absence notifications
4. Work with school on intervention plans
