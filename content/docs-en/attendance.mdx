The attendance module tracks student attendance across multiple capture methods, with role-based access, parent notifications, US DoE risk levels, intervention tracking, and CSV / Excel / PDF export. Teachers see only their assigned classes; students and guardians see only their own records.

## Routes

All under `src/app/[lang]/s/[subdomain]/(school-dashboard)/attendance/`. Tabs: `/`, `/manual`, `/qr-code`, `/records`, `/excuses`, `/analytics`, `/reports`, `/settings`. Off-tab pages (URL only): `/barcode`, `/bulk-upload`, `/bulk`, `/interventions`, `/geo`, `/analysis`, `/recent`. `/parent/attendance` redirects to `/attendance/records`. See the Capture methods table below for per-route purpose; see RBAC for per-role access.

## Manual attendance

Flat inline toolbar (`PlatformToolbar`): class selector, date picker, bulk status buttons, Save. `getCurrentPeriod()` auto-selects the class matching the current timetable period; names translate on demand via `getDisplayText()`. Keyboard: `P` / `A` / `L` for present / absent / late, `Cmd/Ctrl+S` save. Absent-student guardians are auto-notified on save; an approved `AbsenceIntention` converts ABSENT → EXCUSED.

## Parent engagement

`triggerAbsenceNotification()` fires on absent: in-app (high priority), SMS via Twilio (if configured), email. Uses school `preferredLanguage`.

Excuse reasons: Medical, Family Emergency, Religious, School Activity, Transportation, Weather, Other. Approved → `EXCUSED`; rejected → unexcused.

Student / guardian view: percentage + trend, calendar history, status breakdown, excuse submission for unexcused absences.

## Early warning

US DoE risk levels:

| Risk               | Attendance rate | Absence rate |
| ------------------ | --------------- | ------------ |
| Satisfactory       | ≥95%            | `<5%`        |
| At-risk            | 90 – 94.9%      | 5 – 9.9%     |
| Moderately chronic | 80 – 89.9%      | 10 – 19.9%   |
| Severely chronic   | `<80%`          | ≥20%         |

Intervention tracking at `/attendance/interventions` — 14 intervention types (parent contact, home visit, counsellor / social worker / administrator referral, attendance contract, truancy referral, community resource, academic support, mentorship, incentive, other) with statuses Scheduled → In Progress → Completed / Cancelled / Escalated. Policy engine in `actions/policy.ts` evaluates triggers and manages exemptions.

## Analytics & reports

7-tab analytics at `/analytics`: Summary, Calendar, Trends, Patterns, Methods, Classes, At Risk.

Reports at `/reports`: filter by class, student, status, date range, period. Export: CSV, Excel (5 sheets — summary, detailed, by status, by class, by date), PDF (branded).

Compliance: `getComplianceDashboard()`, `getComplianceReport()`, `getScheduledReports()`.

## Server actions

Actions are split across per-concern modules under `src/components/.../attendance/actions/`.

| Module        | File               | Surface                                                                           |
| ------------- | ------------------ | --------------------------------------------------------------------------------- |
| Core          | `core.ts`          | Mark, list, quick-mark, check-out, bulk delete + restore                          |
| Analytics     | `analytics.ts`     | Stats, trends, method usage, calendar, class comparison, at-risk, recent activity |
| Periods       | `periods.ts`       | Per-period mark, current-period detection, period analytics                       |
| QR            | `qr.ts`            | Session generate, scan, active sessions                                           |
| Identifiers   | `identifiers.ts`   | Student barcode / card identifier CRUD + lookup                                   |
| Excuses       | `excuses.ts`       | Submit, review, list pending / unexcused                                          |
| Interventions | `interventions.ts` | Create, update, escalate, list, stats, assignees                                  |
| Bulk          | `bulk.ts`          | CSV / Excel upload, report builder, CSV export, recent uploads                    |
| Dashboard     | `dashboard.ts`     | Today's overview, by-risk, early warning, unmarked classes, parent summary        |
| Policy        | `policy.ts`        | Evaluate policies, triggers, exemptions, dismissals                               |

## Schema

```prisma
model Attendance {
  id           String           @id @default(cuid())
  schoolId     String
  studentId    String
  classId      String
  date         DateTime         @db.Date
  status       AttendanceStatus
  method       AttendanceMethod @default(MANUAL)
  checkInTime  DateTime?
  checkOutTime DateTime?
  notes        String?
  markedBy     String?
  markedAt     DateTime         @default(now())
  deletedAt    DateTime?

  periodId    String?
  timetableId String?
  periodName  String?

  @@unique([schoolId, studentId, classId, date, periodId])
  @@index([schoolId, date, periodId])
}
```

Supporting models: `AttendanceExcuse`, `AttendanceIntervention`, `QRCodeSession`, `StudentIdentifier`, `AbsenceIntention`, `PolicyTrigger`, `PolicyExemption`, `MasterAttendance`, `HallPass`, `KioskLog`, `AttendanceReward`, `StudentBadge`, `AttendanceStreak`.
