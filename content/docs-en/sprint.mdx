The [MVP](/docs/mvp) doc records what is shipped; the [Roadmap](/docs/roadmap) records the quarter direction. This doc is narrower and time-boxed: the **single sprint that ends with launch**. Anchor date: 2026-06-20, HEAD `febb7da3b`.

The thesis is simple. After the June deploys — transportation Phase 2–5, the conference notification/attendance integration, the public features + community hubs, exam-and-grade automation, the stream/LMS optimize pass, the saas-dashboard de-fabrication, timetable production-readiness, and demo-seed auto-provision — almost every epic is **code-complete**. What stands between here and launch is no longer feature-building. It is four things: making the last few correctness fixes, confirming the database, turning on the ops and credentials, and running a QA pass that actually proves it. This sprint does exactly that, for all three launch audiences at once: the **King Fahad** pilot (Sudan, Sept 2026), the **public SaaS** at `ed.databayt.org`, and the **Aldar** enterprise pilot (UAE).

For requirements see [PRD](/docs/prd). For the pilot context see [Pilot](/docs/pilot).

Three buckets. Bucket A is small and surgical, bucket B is mostly confirmation, bucket C is where the real lead time sits.

### B — Database (audit + apply, Neon-branch-first)

## The 4-week sprint

Five tracks run in parallel across all four weeks: **A** Code, **B** Data, **C** Ops/Infra, **D** QA/Proof, **E** Polish. Each week has a theme and an exit gate.

| Week | Theme                | Exit gate                                                                                      |
| ---- | -------------------- | ---------------------------------------------------------------------------------------------- |
| 1    | Reachable & Truthful | Every shipped feature has a URL, CI runs e2e, prod has error visibility, DB confirmed          |
| 2    | Money & Messaging    | Every financial event posts a balanced ledger entry; real-time + WhatsApp live for all tenants |
| 3    | Conference & Proof   | LiveKit live (network-test scheduled), e2e green in CI, Arabic RTL clean                       |
| 4    | Harden & Launch      | Launch gate green for all three audiences                                                      |

**Week 1 — Reachable & Truthful**

- A · Fix the Playwright CI command (`tests` → `test`) so e2e runs · make `STRIPE_API_KEY` optional in `/api/health` · add the `manage/` + `paper/` exam index pages and confirm sidebar surfacing.
- B · Audit Neon prod vs Prisma; apply any additive drift on a Neon branch, then promote.
- C · Set Sentry + Upstash env in Vercel (immediate observability + correct rate limiting).

**Week 2 — Money & Messaging**

- A · ✅ Wired `postFeeAssignment` (assignFee + bulkAssignFees) — the accrual fix that stops the receivable going negative. Investigation found the other three blocked _beyond_ wiring (no invoice/expense payment-recording flow; salary rule unbalanced) and **fixed** a pre-existing **`toCents` ×100 ledger-inflation P0** — rules now store whole units, `posting-rules.test.ts` green (15/15). The debit=credit invariant test already existed, and the `finance/content.tsx` + `salary/content.tsx` 1/100 display bug is fixed. ✅ payroll progressive tax (replaces flat 15% in payroll + salary, unit-tested). The ÷100 ledger migration is a **verified no-op safety net** — a read-only prod check found zero inflated rows (all 200 entries are seeded whole-units), so the toCents fix deploys safely alone; the script is now self-verifying/fees-scoped, run post-deploy only as a sweep. ✅ invoice PDF + branded RTL email (logo into the PDF; email rebuilt on react-email — RTL, branding, itemized, signature). ✅ invoice + expense payment flows (`markInvoicePaid` + `markExpensePaid`). ✅ salary poster — rebalanced the salary posting rule + wired `postSalaryPayment` in `processPayments` → **6 of 6 posters wired; the double-entry ledger is fully wired for every finance money event.** **Week-2 finance code is complete** — and an adversarial multi-agent review of the whole finance diff found + fixed **12 money-path defects** (multi-tenant `schoolId` on updates, concurrent double-post races via conditional updates + a `JournalEntry` `@@unique` idempotency constraint, salary tax base, invoice partial-payment, an `expenses/approve` permission gate, 4 dashboards' ÷100 display, RTL discount sign). What remains is ops (Track C: Oracle VM, Tap/FCM env, ADEK), an expense-list UI to surface `markExpensePaid`, and **two additive deploy-pending migrations** (the toCents ÷100 sweep — a verified no-op — and the JournalEntry `@@unique`).
- C · Stand up Oracle VM + Evolution API (6 env vars; per-school QR for King Fahad + Albayan) · set Tap + FCM env · **file the ADEK inquiry now** (longest external lead time).

**Week 3 — Conference & Proof**

- C · LiveKit 6-gate provisioning; schedule gate-6 in-school network test to a King Fahad / Aldar site visit.
- D · Real e2e pass now that CI runs: triage the `test.skip` backlog, un-skip what runs headless, add a named section-grid attendance case, give the assertion-less specs real `expect()`s · i18n RTL Arabic pass on the swept surfaces + parent-portal dictionary-key migration.
- E · Exams schema cleanup (bilingual → `lang`, reconcile `Result`/`ExamResult`, harden the `ExamCertificate` cascade chain).

**Week 4 — Harden & Launch**

- E · Drain the highest-value SaaS-dashboard backlog (delete the dead Kanban/Products route pages, add the invoice state-machine guard, close catalog TOCTOU windows, i18n sweep) · exams RBAC + calculator test coverage + AI cost cap.
- D · Per-pilot smoke on King Fahad + Albayan · load + rate-limit verification (now Redis-backed) · final `secure` pass · wire the hardcoded-string ratchet into the PR gate.
- Run the launch gate.

## Launch gate

The sprint ends when this is green. Items that are audience-specific are tagged.

- [ ] All bucket-A code merged to `main`
- [ ] Neon prod schema confirmed against Prisma (no additive drift)
- [ ] Sentry + Upstash + FCM env set; Sentry receiving events
- [ ] `playwright test` green in CI (no longer a no-op)
- [ ] Finance ledger correct: `toCents` ×100 bug resolved (`posting-rules.test.ts` green) + wireable posters wired (`postFeeAssignment`/`postFeePayment`/`postWalletTopup` done; invoice/expense/salary pending their flows + rule fix)
- [ ] King Fahad + Albayan tenants smoke-clean end-to-end
- [ ] Oracle VM live; WhatsApp QR scanned per pilot school
- [ ] Tap keys set — _UAE / Aldar_
- [ ] LiveKit network-test passes inside school WiFi — _Aldar_
- [ ] ADEK eSIS status acceptable (DRY*RUN is fine for King Fahad / SaaS; live submission only gates Aldar compliance go-live) — \_Aldar*
