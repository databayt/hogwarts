---
title: Integration Flow
description: The 8-layer dependency chain from global catalog to a working timetable, with what each layer creates and depends on.
---

Every school progresses through 8 layers to reach a working timetable. Each layer depends on the ones above. Most are auto-provisioned during onboarding via `provisionSchoolDefaults()`; a few need manual admin steps.

## The 8 layers

| Layer | Name                    | What it creates                                                                   | Depends on |
| ----- | ----------------------- | --------------------------------------------------------------------------------- | ---------- |
| 1     | Global catalog          | `Subject` (~240 grade-specific) → chapters → lessons → books                      | —          |
| 2     | School onboarding       | School profile + provisioning trigger                                             | 1          |
| 3     | Academic structure      | `AcademicLevel`, `AcademicGrade`, `AcademicStream`                                | 2          |
| 4     | Subject selection       | `SubjectSelection` bridge rows                                                    | 1, 3       |
| 5a    | Teacher assignment      | `Class.teacherId`, `ClassTeacher`, `TeacherSubjectExpertise`, `TeacherDepartment` | 4          |
| 5b    | Student enrollment      | `Student`, `StudentYearLevel`, `StudentClass`, `StudentBatch`                     | 3, 4       |
| 6     | Classroom configuration | `ClassroomType`, `Classroom`, `Section`, `Class`                                  | 3–5        |
| 7     | Timetable               | `Timetable` slots                                                                 | All above  |
| 8     | Exams + reports         | `SchoolExam`, `Result`, report cards, certificates, promotion                     | 5, 7       |

## Layer 1 — Global catalog

The shared backbone. All curriculum content lives here once; schools reference it via bridge tables.

```
Subject (~240 grade-specific US K-12 + more curricula)
  └─ Chapter (topic groups per grade-subject)
       └─ Lesson
```

Subjects are grade-specific, not level-based. `"Math Grade 1"` (slug `us-g1-math`), `"Math Grade 2"` (`us-g2-math`), etc., replacing `"Elementary Math"` with `levels: [ELEMENTARY]`.

Teacher-created content (videos, questions, materials) is saved as `PENDING` + `PRIVATE`, reviewed by SaaS admin, then `APPROVED` (indexed with visibility tier) or `REJECTED`.

## Layer 2 — School onboarding

Onboarding fields drive school provisioning.

| Onboarding step | Saves into                                                 |
| --------------- | ---------------------------------------------------------- |
| Description     | `School.schoolType` (`public`, `british`, `ib`…)           |
| Location        | `School.country`, `School.city`, `School.state`            |
| Capacity        | `School.sectionsPerGrade`, `School.studentsPerSection`     |
| Schedule        | `School.curriculum` + `Period`, `Term`, `SchoolWeekConfig` |

### Capacity inputs and computed limits

| Form field           | Saved as                    | Meaning                       |
| -------------------- | --------------------------- | ----------------------------- |
| Teachers             | `School.maxTeachers`        | Total teacher headcount       |
| Sections per Grade   | `School.sectionsPerGrade`   | Sections (A, B, C…) per grade |
| Students per Section | `School.studentsPerSection` | Target class size             |

| Computed      | Formula                           | Meaning              |
| ------------- | --------------------------------- | -------------------- |
| `maxClasses`  | `grades × sectionsPerGrade`       | `Class` record limit |
| `maxStudents` | `maxClasses × studentsPerSection` | Plan student limit   |

`maxClasses` limits course sections, not physical `Classroom` records. Classroom + Section records are auto-created by `autoProvisionSections()`.

### `completeOnboarding()`

Runs when the user accepts terms on the legal step. Sets `isPublished: true` and `onboardingCompletedAt`. Triggers seven non-blocking provisioning steps via `provisionSchoolDefaults()` (run in `after()`):

1. `setupDefaultsForSchool()` — YearLevels, 6 universal Departments, ScoreRanges (A+ → F).
2. `setupCatalogForSchool()` — Layer 3 + 4.
3. `applyTimetableStructureForNewSchool()` — school year, terms, periods, week config.
4. Default `ClassroomType` upsert — so the Configure tab works immediately.
5. `autoProvisionSections()` — N classrooms + N sections per grade (`School.sectionsPerGrade`, default 2). Naming: `"Grade N-A"`, `"Grade N-B"`.
6. `generateUniqueJoinCode()` — internal join page code.
7. `syncStudentGrades()` — link students with raw `yearLevel` strings to new `AcademicGrade` records.

Each step has its own `.catch()`. Failures don't abort subsequent steps. Errors are not surfaced to the user.

`congratulations/actions.ts` has `publishSchool()` as an admin-only fallback for manual re-provisioning.

### What `applyTimetableStructureForNewSchool()` creates

| Record             | Details                                                            |
| ------------------ | ------------------------------------------------------------------ |
| `SchoolYear`       | Default year (Sep 1 – Jun 30) if none exists                       |
| `Term` ×2          | Term 1 (Sep 1 – Jan 31, `isActive: true`), Term 2 (Feb 1 – Jun 30) |
| `Period` ×N        | All periods from the structure (class, break, lunch)               |
| `SchoolWeekConfig` | Working days + `defaultLunchAfterPeriod`                           |

### Timetable structures

| Slug             | Name                     | Periods | Days    | Country | Default |
| ---------------- | ------------------------ | ------- | ------- | ------- | ------- |
| `sd-gov-default` | Sudan Government         | 8 × 45m | Sun–Thu | SD      | Yes     |
| `sd-private`     | Sudan Private            | 7 × 50m | Sun–Thu | SD      | No      |
| `sd-british`     | British Curriculum       | 6 × 55m | Mon–Fri | SD      | No      |
| `sd-ib`          | IB / American            | 6 × 50m | Mon–Fri | SD      | No      |
| `sd-half-day`    | Half Day / KG            | 5 × 40m | Sun–Thu | SD      | No      |
| `gulf-standard`  | Gulf Government Standard | 7 × 45m | Sun–Thu | GULF    | Yes     |
| `gulf-private`   | Gulf Private / Intl      | 7 × 50m | Sun–Thu | GULF    | No      |
| `mena-standard`  | MENA Standard            | 7 × 45m | Mon–Fri | MENA    | Yes     |
| `us-standard`    | US Standard              | 7 × 50m | Mon–Fri | US      | Yes     |
| `intl-default`   | International Default    | 6 × 50m | Mon–Fri | `*`     | No      |

`getRecommendedStructures(country, schoolType, schoolLevel)` ranks structures.

| Signal              | Score |
| ------------------- | ----- |
| Country exact match | +40   |
| Region match        | +25   |
| Wildcard `*`        | +5    |
| `schoolType` match  | +20   |
| `schoolLevel` match | +10   |

Score ≥ 30 → recommended. Score ≥ 50 → auto-select. Region map: SA/AE/QA/KW/BH/OM → GULF; EG/JO/LB/TN/MA/DZ → MENA.

## Layer 3 — Academic structure

`setupCatalogForSchool()` creates the academic hierarchy in a single transaction.

| `schoolLevel`   | Levels        | Grades |
| --------------- | ------------- | ------ |
| `"primary"`     | ELEMENTARY    | 1–6    |
| `"secondary"`   | MIDDLE + HIGH | 7–12   |
| `"both"` / null | All three     | 1–12   |

| Model            | Scope  | Key fields                                      |
| ---------------- | ------ | ----------------------------------------------- |
| `AcademicLevel`  | School | `level`, `levelOrder`, `startGrade`, `endGrade` |
| `AcademicGrade`  | School | `gradeNumber`, `levelId`, `yearLevelId`         |
| `AcademicStream` | School | `streamType` (SCIENCE, ARTS), `gradeId`         |

## Layer 4 — Subject selection

Bridge layer between global catalog and school grades. Created in the same transaction as Layer 3.

Auto-assignment first matches catalog `subject.grades` arrays; falls back to level-based mapping if empty. ~99% auto-assigned.

### Progressive fallback

| Step | Filter                              | Example                         |
| ---- | ----------------------------------- | ------------------------------- |
| 1    | `country + curriculum + schoolType` | SD + national + private         |
| 2    | `country + curriculum`              | SD + national                   |
| 3    | `country="*" + curriculum`          | IB worldwide, British worldwide |
| 4    | US + us-k12                         | Baseline fallback               |

Curriculum inferred from `country + schoolType` if not set: international → `us-k12`, known countries → `national`, unknown → `us-k12`.

### Customization

- Browse the full catalog and add subjects from other systems.
- Hide subjects per grade by toggling `isActive`.
- Hide chapters or lessons via `ContentOverride`.
- Rename via `customName` on the selection record.
- Set `weeklyPeriods` for timetable planning.

Schools can request new subjects: school CRUD request → SaaS review → approved → added to global catalog.

## Layer 5a — Teacher assignment

| Role              | Where                     | Description                               |
| ----------------- | ------------------------- | ----------------------------------------- |
| Subject Teacher   | `Class.teacherId`         | Primary teacher of a section              |
| Co-Teacher        | `ClassTeacher.role`       | `PRIMARY`, `CO_TEACHER`, `ASSISTANT`      |
| Subject Expert    | `TeacherSubjectExpertise` | Qualification metadata (`expertiseLevel`) |
| Department Member | `TeacherDepartment`       | `isPrimary` flag                          |
| Department Head   | `TeacherDepartment`       | `isDepartmentHead: true` (one per dept)   |
| Homeroom Teacher  | `Class`                   | Pastoral, not subject-tied                |

| Model                     | Unique constraint                     |
| ------------------------- | ------------------------------------- |
| `ClassTeacher`            | `[schoolId, classId, teacherId]`      |
| `TeacherSubjectExpertise` | `[schoolId, teacherId, subjectId]`    |
| `TeacherDepartment`       | `[schoolId, teacherId, departmentId]` |

### Cascade validation on delete

`deleteTeacher` blocks if `Class.teacherId` or `Timetable` references exist; `ClassTeacher` rows are auto-cleaned in a transaction.

`deleteSubject` blocks if `TeacherSubjectExpertise`, `Class.subjectId`, `Exam.subjectId`, or `QuestionBank.subjectId` references exist.

## Layer 5b — Student enrollment

5-step admission wizard: Attachments, Personal, Location, Academic, Fees (guardian folded into Personal tabs; contact removed — email back-filled at submit). Submission happens on the Fees step.

Academic step fields:

| Field              | Type       | Purpose                             |
| ------------------ | ---------- | ----------------------------------- |
| `applyingForClass` | `String`   | Target grade ("Grade 10")           |
| `preferredStream`  | `String?`  | "science", "arts"                   |
| `dateOfBirth`      | `DateTime` | Used for DOB-based grade suggestion |

Status flow: `DRAFT → SUBMITTED → UNDER_REVIEW → SHORTLISTED → ENTRANCE_SCHEDULED → INTERVIEW_SCHEDULED → SELECTED → WAITLISTED / REJECTED / ADMITTED / WITHDRAWN`.

After `ADMITTED`: `Student → StudentYearLevel + StudentClass + StudentBatch`. Through `Class`, students inherit subjects, teachers, classrooms, and timetable.

### Section placement

Two UIs in `admission/`:

- `placement-dialog.tsx` — per-student. Shows available sections with capacity badges.
- `bulk-placement.tsx` — all `ADMITTED` without placement. Batch-assign to a target section.

Both call `placeStudentInSection(applicationId, sectionId)`, which validates `maxCapacity`, updates `Student.sectionId`, auto-creates `StudentClass` for all classes in that grade, and triggers `syncStudentClassToEnrollment()` for LMS `Enrollment` records.

### Grade number extraction

`extractGradeNumber()` in `src/lib/grade-utils.ts` parses free-text grades.

| Format          | Example                 | Result |
| --------------- | ----------------------- | ------ |
| Direct          | `"5"`                   | 5      |
| English         | `"Grade 5"`, `"Year 5"` | 5      |
| Ordinal         | `"5th grade"`           | 5      |
| Arabic ordinal  | `"الصف الخامس"`         | 5      |
| English ordinal | `"fifth grade"`         | 5      |
| Trailing        | `"Section 5"`           | 5      |

## Layer 6 — Classroom configuration

| Record          | Auto?  | How                                        |
| --------------- | ------ | ------------------------------------------ |
| `ClassroomType` | Yes    | `completeOnboarding()` upserts "Classroom" |
| `Classroom`     | Yes    | `autoProvisionSections()` — N per grade    |
| `Section`       | Yes    | `autoProvisionSections()` — N per grade    |
| `Class`         | **No** | Admin runs "Generate Classes" in Configure |
| `StudentClass`  | **No** | Admin runs "Enroll Students" in Configure  |

### Configure tab workflow

1. **Generate Sections** (optional) — add more beyond auto-provisioned.
2. **Generate Classes** — 1 `Class` per subject × grade × term.
3. **Enroll Students** — `StudentClass` for every student × class per grade.

`Class` shape:

```
Class
  ├─ name: "Math - Grade 7 A"
  ├─ gradeId     → AcademicGrade
  ├─ subjectId   → Subject
  ├─ teacherId   → Teacher
  ├─ classroomId → Classroom
  ├─ termId      → Term
  ├─ minCapacity (enrollment floor)
  └─ maxCapacity (enrollment ceiling)
```

## Layer 7 — Timetable

```
Timetable {
  schoolId, termId, dayOfWeek (0=Sun..6=Sat),
  periodId, classId, teacherId, classroomId,
  weekOffset (0=current, 1=next)
}
```

Three unique constraints prevent double-booking.

```
@@unique([schoolId, termId, dayOfWeek, periodId, classId, weekOffset])
@@unique([schoolId, termId, dayOfWeek, periodId, teacherId, weekOffset])
@@unique([schoolId, termId, dayOfWeek, periodId, classroomId, weekOffset])
```

## Layer 8 — Exams + reports

```
Exam → ExamSession → StudentAnswer → MarkingResult → ExamResult
QuestionBank → GeneratedExam → GeneratedPaper
Result (assignments / homework grades)

ExamResult + Result
  → Report Card (per student per term)
       ├─ Certificate PDF
       ├─ Transcript (multi-year)
       ├─ Notifications (grade_posted, report_ready, certificate_issued, promotion_decided)
       └─ Promotion / Retention → Student.academicGradeId
```

Certificates use the same 5-layer composition as exam papers: Atom → Section → Variant → Composition → Document. 7 slots: Header, Title, Recipient, Body, Scores, Signatures, Footer. 4 regional presets (SD, SA, US, MENA). Stored in `ExamCertificateConfig.compositionConfig`.

Promotion: admin triggers evaluation → `PromotionBatch` → per student GPA + failed subjects + attendance → `PromotionCandidate` → admin reviews/overrides → batch approved → execute (update `Student.academicGradeId`, create `StudentYearLevel`) → notifications.

## Failure modes

| Missing            | Blocks                                   |
| ------------------ | ---------------------------------------- |
| Catalog            | Subject selection, teacher assignment    |
| Onboarding         | Academic structure                       |
| Academic structure | Subject selection, class sections        |
| Terms              | Timetable (`termId` required)            |
| Teachers           | `Class` records (`teacherId` required)   |
| Class records      | Timetable slots (`classId` required)     |
| `StudentClass`     | Students don't see timetable             |
| Classrooms         | `Class` records (`classroomId` required) |
| Exams / Results    | Report cards, transcripts                |
| Report cards       | Promotion (GPA required)                 |

## Readiness check

```ts
const [
  levels,
  grades,
  selections,
  teachers,
  classrooms,
  periods,
  terms,
  weekConfig,
] = await Promise.all([
  db.academicLevel.count({ where: { schoolId } }),
  db.academicGrade.count({ where: { schoolId } }),
  db.subjectSelection.count({ where: { schoolId, isActive: true } }),
  db.teacher.count({ where: { schoolId } }),
  db.classroom.count({ where: { schoolId } }),
  db.period.count({ where: { schoolId } }),
  db.term.count({ where: { schoolId } }),
  db.schoolWeekConfig.findFirst({ where: { schoolId } }),
])

const ready =
  levels > 0 &&
  grades > 0 &&
  selections > 0 &&
  teachers > 0 &&
  classrooms > 0 &&
  periods > 0 &&
  terms > 0 &&
  weekConfig !== null
```
