Banking manages a school's bank accounts, links them via Plaid, moves money via Dwolla, and is meant to reconcile book entries against bank statements. The reconciliation UI is currently a placeholder.

## Flow

```
Link a bank (Plaid Link)  →  list accounts + balances  →  initiate transfer (Dwolla)
  →  view transaction history  →  reconcile against statements (stubbed)
```

## Server actions

Split across `banking/actions/` (`bank.actions.ts`, `transaction.actions.ts`, `transfer.actions.ts`) plus `my-banks/actions.ts` and `payment-transfer/actions.ts`. All `schoolId`-scoped; RBAC via `banking/lib/permissions.ts`.

| Action                                                         | Purpose                                   |
| -------------------------------------------------------------- | ----------------------------------------- |
| `createBankAccount` / `getAccount` / `getAccounts`             | Bank account CRUD + reads                 |
| `removeBank`                                                   | Unlink a bank                             |
| `createTransfer` / `getTransferHistory` / `getRecentTransfers` | Dwolla transfers + history                |
| `getTransactionsByBankId` / `getTransactionsByUserId`          | Transaction reads                         |
| `getBankInfo` / `prefetchAccounts`                             | Plaid account info + prefetch             |
| `syncBankData` / `syncTransactions`                            | Sync from provider (Plaid sync is a stub) |

## Ledger / accounting

Not a ledger money-mover. Banking tracks external bank rails and is _intended_ to reconcile against payroll disbursements, expense payments, and fee deposits — but the reconciliation panel is stubbed, and banking does not itself post journal entries (the README's `lib/accounting/` claim is aspirational).

## Schema

`prisma/models/finance-banking.prisma`: `BankAccount`, `Transaction`, `Transfer`, `PlaidItem`, `DwollaCustomer`, `BankReconciliation` — all `schoolId`-scoped. No `lang` field.

## Localization & currency

The banking dashboard renders money in the school's own currency, not a fixed one.
`lib/utils.ts → formatAmount(amount, locale, currency)` delegates to
`@/lib/payment/currency`, and `currency` is `School.currency`, read once in
`dashboard/content.tsx` and passed down. A Sudanese tenant therefore sees
`٢٬٠١٠٬٠٠٠٫٠٠ ج.س.` on `/ar` and `SDG 2,010,000.00` on `/en`.

Transaction categories are stored as raw slugs (`tuition`, `salary`, `utilities`, …) and
translated for display through `finance.bankingTransactions.categories.<slug>`. When a new
category slug reaches the data, add the key in both `dictionaries/en/finance.json` and
`dictionaries/ar/finance.json` — the UI falls back to printing the bare slug otherwise.

The greeting reads `User.username` from the database. The `User` model has no `name`
column and the NextAuth session carries no display name, so anything relying on
`session.user.name` renders "Guest" for every user.
