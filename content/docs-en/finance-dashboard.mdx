The finance dashboard aggregates the block into a role-scoped overview: admins and accountants see full KPIs and charts; teachers, students, and guardians see scoped slices (own salary, fee status). It is the landing page at `/finance/dashboard`.

## Flow

```
Load (role-scoped)  →  KPI cards (real totals)  +  trend charts (mock)
  +  smart alerts  +  quick actions  +  recent transactions
```

## Server actions

| Action                   | Purpose                                              |
| ------------------------ | ---------------------------------------------------- |
| `getDashboardStats`      | KPI totals + trend series (totals real, trends mock) |
| `getFinancialAlerts`     | Smart alerts (overdue, low balance, etc.)            |
| `getQuickActionsForRole` | Role-appropriate action buttons                      |
| `getRecentTransactions`  | Recent transactions feed                             |

## Ledger / accounting

Not a money-mover — the dashboard reads and aggregates across the invoice, fees, expenses, payroll, banking, budget, and wallet models for display only.

## Schema

No model of its own; composes reads across the other finance models.
