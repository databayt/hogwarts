## Overview

Setting up a school on the platform follows a layered process. Each layer builds on the previous one. Some layers are fully automatic, some require admin action.

```
┌─────────────────────────────────────────────────────┐
│  1. Onboarding          15-step wizard              │  ← Admin
│  2. Auto-Provisioning   Academic structure + schedule│  ← System
│  3. People              Teachers + students          │  ← Admin (CSV)
│  4. Grade Assignment    Students → grades            │  ← Admin
│  5. Classes             Subject offerings            │  ← Admin
│  6. Enrollment          Students → classes           │  ← Admin
│  7. Teacher Expertise   Teachers → subjects          │  ← Admin
│  8. Timetable           Weekly schedule              │  ← Algorithm
│  9. Attendance           Daily operation             │  ← Teachers
└─────────────────────────────────────────────────────┘
```

### What's Automatic vs Manual

| Layer                      | What                             | Auto?      | Admin Effort             |
| -------------------------- | -------------------------------- | ---------- | ------------------------ |
| Academic structure         | Levels, grades, streams          | Auto       | None                     |
| Subject selections         | Catalog → school bridge          | Auto       | None                     |
| Schedule structure         | Periods, terms, working days     | Auto       | None                     |
| Classrooms + sections      | Physical rooms + student groups  | Auto       | None                     |
| Departments + score ranges | Org structure                    | Auto       | None                     |
| Teachers                   | User accounts                    | CSV import | Upload file              |
| Students                   | User accounts + guardians        | CSV import | Upload file              |
| Student grade assignment   | Student → AcademicGrade          | **Manual** | Per student or re-import |
| Class generation           | Subject + teacher + term + room  | **Manual** | Click button             |
| Student enrollment         | Student × class records          | **Manual** | Click button             |
| Teacher expertise          | Teacher → subject qualifications | **Manual** | Per teacher              |
| Timetable                  | Weekly slot placement            | Semi-auto  | Generate + review        |
| Attendance                 | Daily status tracking            | Manual     | Daily operation          |

## 1. Onboarding

A 15-step wizard collects the school profile. Only some steps feed into the operational setup:

### Steps That Drive Setup

| Step           | Collects                                                    | Drives                                                |
| -------------- | ----------------------------------------------------------- | ----------------------------------------------------- |
| `about-school` | School name                                                 | Identity                                              |
| `description`  | `schoolType`, `schoolLevel`                                 | Which grades to create, catalog filtering             |
| `location`     | `country`                                                   | Which curriculum, timetable structure recommendations |
| `capacity`     | `sectionsPerGrade`, `studentsPerSection`, `teachers`        | How many sections per grade, plan limits              |
| `schedule`     | Timetable structure (e.g., `us-standard`, `sd-gov-default`) | Periods, working days, term dates                     |
| `import`       | CSV files                                                   | Teacher + student records                             |
| `legal`        | Terms acceptance                                            | Triggers provisioning                                 |
| `subdomain`    | Unique subdomain                                            | `{subdomain}.databayt.org` routing                    |

### Other Steps

`title`, `stand-out`, `branding`, `finish-setup`, `join`, `visibility`, `price`, `discount` — these configure the school's public profile, branding, and billing. They don't affect the operational setup.

### The Trigger

Accepting terms on the `legal` step calls `completeOnboarding()`, which:

1. Sets `isPublished: true` and `onboardingCompletedAt`
2. Triggers four async provisioning jobs (Layer 2)

### Customizing Onboarding Choices

| Choice                                         | Impact                                                    | Can Change Later?        |
| ---------------------------------------------- | --------------------------------------------------------- | ------------------------ |
| `schoolLevel` ("primary", "secondary", "both") | Determines which grades are created (1-6, 7-12, or 1-12)  | Requires re-provisioning |
| `country`                                      | Determines catalog subjects and timetable recommendations | School > Settings        |
| `schoolType`                                   | Filters catalog subjects (public, private, british, ib)   | School > Settings        |
| `sectionsPerGrade`                             | Number of sections A, B, C... per grade                   | Classrooms > Configure   |
| `studentsPerSection`                           | Capacity per section (default 30)                         | Edit individual sections |
| Timetable structure                            | Period count, duration, working days                      | Timetable > Settings     |

## 2. Auto-Provisioning

Runs automatically after onboarding. The admin does nothing — just wait 2-3 seconds and refresh.

### What Gets Created

```
School
  │
  ├── Academic Structure
  │     ├── AcademicLevel ×3      Elementary, Middle, High
  │     ├── AcademicGrade ×12     Grade 1 through Grade 12
  │     └── AcademicStream ×6     Science + Arts for grades 10-12
  │
  ├── Curriculum Bridge
  │     └── SchoolSubjectSelection ×N   ~15-30 subjects per grade from catalog
  │
  ├── Schedule
  │     ├── SchoolYear ×1         Sep 1 – Jun 30
  │     ├── Term ×2               Term 1 (Sep-Jan), Term 2 (Feb-Jun)
  │     ├── Period ×7-8           Based on structure selection
  │     └── SchoolWeekConfig ×1   Working days (Sun-Thu or Mon-Fri)
  │
  ├── Physical Space
  │     ├── ClassroomType ×1      Default "Classroom" type
  │     ├── Classroom ×N          grades × sectionsPerGrade rooms
  │     └── Section ×N            grades × sectionsPerGrade student groups
  │
  └── Organization
        ├── YearLevel ×N          Filtered by schoolLevel
        ├── Department ×6         Sciences, Mathematics, Languages, etc.
        └── ScoreRange ×8         A+ through F
```

### How to Customize Each

#### Academic Grades

Auto-provisioned based on `schoolLevel`:

| schoolLevel     | Grades Created |
| --------------- | -------------- |
| `"primary"`     | 1-6            |
| `"secondary"`   | 7-12           |
| `"both"` / null | 1-12           |

**Customization**: Currently no UI to add/remove individual grades. The full set for the selected level is created.

#### Subject Selections

The system reads the global catalog and auto-assigns subjects to grades based on the school's `country` and `schoolType`. A 4-step progressive fallback ensures every school gets content:

1. Exact match: country + curriculum + schoolType
2. Country + curriculum (ignore schoolType)
3. Wildcard subjects (e.g., IB worldwide)
4. US K-12 baseline fallback

**Customization**:

| Action                            | Where                                    |
| --------------------------------- | ---------------------------------------- |
| Add subjects from other curricula | Subjects > Catalog tab                   |
| Hide a subject for a grade        | Toggle `isActive` on the selection       |
| Rename a subject                  | Set `customName` on the selection        |
| Set periods per week              | Edit `weeklyPeriods` (affects timetable) |
| Hide chapters/lessons             | `SchoolContentOverride` records          |

#### Timetable Structure

Based on the structure selected during the `schedule` step. Available structures:

| Structure        | Periods | Duration | Days    | Region        |
| ---------------- | ------- | -------- | ------- | ------------- |
| `sd-gov-default` | 8       | 45min    | Sun-Thu | Sudan         |
| `sd-private`     | 7       | 50min    | Sun-Thu | Sudan         |
| `us-standard`    | 7       | 50min    | Mon-Fri | US            |
| `gulf-standard`  | 7       | 45min    | Sun-Thu | Gulf          |
| `intl-default`   | 6       | 50min    | Mon-Fri | International |

**Customization**:

| Action                    | Where                |
| ------------------------- | -------------------- |
| Add/remove/rename periods | Timetable > Settings |
| Change period duration    | Timetable > Settings |
| Toggle working days       | Timetable > Settings |
| Adjust term dates         | School > Settings    |
| Activate/deactivate terms | School > Settings    |

#### Classrooms + Sections

Auto-created with naming pattern `"Grade {N}-{letter}"` (e.g., "Grade 7-A", "Grade 7-B"). Each section gets a paired classroom.

**Customization**:

| Action                                   | Where                      |
| ---------------------------------------- | -------------------------- |
| Add more sections per grade              | Classrooms > Configure tab |
| Create room types (lab, library, sports) | Classrooms > Settings      |
| Change room capacity                     | Edit individual classroom  |
| Change section capacity                  | Edit individual section    |
| Assign homeroom teacher                  | Edit section               |
| Rename rooms                             | Edit individual classroom  |

## 3. People Import

Teachers and students can be imported via CSV during onboarding or after.

### Teacher CSV Format

```csv
name,email,employeeId,department,subjects
Ahmed Hassan,ahmed.h@school.local,TCH001,Sciences,"Physics,Chemistry"
Fatima Ali,fatima.a@school.local,TCH002,Mathematics,"Math,Algebra"
```

**What it creates**: User account (role: TEACHER) + Teacher record. The `department` and `subjects` columns are stored as text metadata.

**What it does NOT create**: `TeacherSubjectExpertise` records, `TeacherDepartment` links. The `subjects` column is informational only — it does not formally qualify the teacher to teach those subjects in the system.

### Student CSV Format

```csv
name,email,studentId,yearLevel,gender,dateOfBirth,guardianName,guardianEmail,guardianPhone
Ali Mohammed,ali.m@school.local,STD001,Grade 7,male,2012-03-15,Khalid Mohammed,khalid@parent.local,+249912345001
```

**What it creates**: User account (role: STUDENT) + Student record + Guardian record + Guardian link.

**What it does NOT create**: `academicGradeId` link. The `yearLevel` column is stored as text but not mapped to an `AcademicGrade` record.

### Import Timing Matters

| When                                | Issue                                        | Result                                                   |
| ----------------------------------- | -------------------------------------------- | -------------------------------------------------------- |
| **During onboarding** (import step) | School FK not committed + no `AcademicGrade` | Import **fails** with FK constraint violation            |
| **After onboarding** (Bulk Import)  | `AcademicGrade` records exist                | Students created **with grade** (if `yearLevel` matches) |

**Recommendation**: Skip the import step during onboarding and import via School > Settings > Bulk Import after the school is fully provisioned.

### Post-Import: Adding More People

| Method             | Path                                     |
| ------------------ | ---------------------------------------- |
| Bulk CSV import    | School > Settings > Bulk Import          |
| Individual teacher | Teachers > New Teacher                   |
| Individual student | Students > New Student                   |
| Admission pipeline | Admission > Applications (6-step wizard) |

## 4. Student Grade Assignment

Students need an `academicGradeId` before they can be enrolled in classes or appear in grade-specific views.

### Methods

| Method                      | Path                            | Best For                                     |
| --------------------------- | ------------------------------- | -------------------------------------------- |
| **Re-import CSV**           | School > Bulk Import > Students | Many students, grades in CSV                 |
| **Edit individual student** | Students > click student > Edit | One-off fixes                                |
| **Admission placement**     | Admission > Placement           | Students entering through admission pipeline |

### Verification

Check Students listing — students without a grade will show as "incomplete" or have no grade badge.

## 5. Class Generation

A **Class** is a subject offering: Subject + Teacher + Term + Room + Grade. This is different from a Classroom (physical room) or Section (student group).

### Terminology

| Term          | What It Is                   | Example                                             |
| ------------- | ---------------------------- | --------------------------------------------------- |
| **Classroom** | Physical room                | "Room 7-A" (capacity 30)                            |
| **Section**   | Student group within a grade | "Grade 7-A" (students Ali, Sara, Musa...)           |
| **Class**     | Subject offering for a term  | "Math taught by Ahmed in Room 7-A, Term 2, Grade 7" |

### Prerequisites

- Active term (auto-provisioned)
- Subject selections from catalog (auto-provisioned)
- At least 1 teacher
- At least 1 classroom per grade (auto-provisioned)

### How to Generate

**Path**: Classrooms > Configure tab

1. Verify sections exist per grade (check "Existing" column)
2. Select the active term from the dropdown
3. Click **"Generate Classes for All Grades"**

This reads `SchoolSubjectSelection` records and creates:

- A school-scoped `Subject` record per catalog subject
- A `Class` record linking subject + teacher + term + room + grade

### Customizing Classes

| Action                  | Where                 |
| ----------------------- | --------------------- |
| Change assigned teacher | Edit individual class |
| Change assigned room    | Edit individual class |
| Add co-teachers         | Class > Teachers tab  |
| Set min/max enrollment  | Edit individual class |
| Create class manually   | Classes > New Class   |

## 6. Student Enrollment

Enrollment creates `StudentClass` records — one per student per class in their grade.

### How to Enroll

**Path**: Classrooms > Configure tab > **"Enroll Students for All Grades"**

This creates the Cartesian product: every student in Grade 7 gets enrolled in every Grade 7 class.

### Prerequisites

- Students must have `academicGradeId` assigned (Layer 4)
- Classes must exist for those grades (Layer 5)

### After Enrollment

Students appear in class rosters, attendance sheets, and grade books.

### Customizing Enrollment

| Action                             | Where                 |
| ---------------------------------- | --------------------- |
| Unenroll student from a class      | Class > Students tab  |
| Enroll student in additional class | Class > Students tab  |
| View student's classes             | Student > Classes tab |

## 7. Teacher Expertise

The timetable algorithm needs to know which teachers can teach which subjects. This is stored in `TeacherSubjectExpertise` records.

### Why This Matters

Without expertise records, the timetable algorithm **places 0 slots**. The algorithm's `placeClassGreedy()` function reads `preferredTeacherIds` from expertise records. No qualified teachers = class skipped entirely.

### How to Set Up

**Path**: Teachers > select teacher > Wizard > Expertise tab

The expertise form lets you add subject qualifications with an expertise level (Primary, Secondary, or Certified). Each teacher can have multiple expertise records.

| Method             | How                                                     |
| ------------------ | ------------------------------------------------------- |
| **Teacher wizard** | Teachers > select teacher > Wizard > Expertise tab      |
| **Bulk via CSV**   | Re-import teachers with subjects column (creates links) |

### Required Fields

| Field            | Type   | Description                            |
| ---------------- | ------ | -------------------------------------- |
| `schoolId`       | String | Tenant isolation                       |
| `teacherId`      | String | FK to Teacher                          |
| `subjectId`      | String | FK to Subject (school-scoped)          |
| `expertiseLevel` | Enum   | `PRIMARY`, `SECONDARY`, or `CERTIFIED` |

### Recommendation

Map each teacher to their primary subject(s) before generating the timetable. A teacher can have multiple expertise records (e.g., Ahmed teaches both Physics and Chemistry).

## 8. Timetable Generation

The timetable ties everything together: classes + teachers + rooms + periods + days.

### Prerequisites

All of these must exist before generation:

| Prerequisite              | Source            | Auto?     |
| ------------------------- | ----------------- | --------- |
| Terms                     | Auto-provisioning | Yes       |
| Periods                   | Auto-provisioning | Yes       |
| Working days (WeekConfig) | Auto-provisioning | Yes       |
| Teachers                  | CSV import        | Semi-auto |
| Subjects (school-scoped)  | Class generation  | Manual    |
| Classes                   | Class generation  | Manual    |
| Classrooms                | Auto-provisioning | Yes       |
| Teacher expertise         | Manual config     | **No**    |

### How to Generate

**Path**: Timetable > Generate tab

1. Select active term
2. Configure constraints (max consecutive periods, teacher preferences)
3. Click **"Generate"**
4. Review the preview grid
5. Click **"Apply"** to save

### Algorithm

Three-phase process:

1. **Greedy Assignment** — places classes one by one, checking availability
2. **Constraint Satisfaction** — resolves conflicts, applies soft constraints
3. **Optimization** — swaps slots to improve distribution

### Customizing the Timetable

| Action                 | Where                                      |
| ---------------------- | ------------------------------------------ |
| Manual slot placement  | Timetable > Admin grid (drag and drop)     |
| Room constraints       | Classrooms > Room > Constraints            |
| Teacher preferences    | Timetable > Settings > Teacher constraints |
| Block specific periods | Timetable > Settings > Reserved periods    |
| Re-generate            | Timetable > Generate (replaces existing)   |

### Views

| View         | Shows                         | Who Uses It        |
| ------------ | ----------------------------- | ------------------ |
| Admin grid   | All classes × all periods     | School admin       |
| Class view   | One class's weekly schedule   | Section teachers   |
| Teacher view | One teacher's weekly schedule | Individual teacher |
| Room view    | One room's weekly occupancy   | Facility manager   |

### Known Limitations

- **Lab subjects**: The algorithm sets `requiresLab: true` for subjects with "lab" in the name. Without a lab-type room, these classes get 0 slots. **Fix**: Create a classroom with type "lab" before generating.
- **Hours per week**: Defaults to 3 periods per subject per week. Customize via `weeklyPeriods` on `SchoolSubjectSelection`.

## 9. Attendance

Daily operation once the school is running.

### Prerequisites

| Prerequisite                 | Required?                          |
| ---------------------------- | ---------------------------------- |
| Classes                      | Yes                                |
| Students enrolled in classes | Yes                                |
| Timetable slots              | Optional (enhances auto-selection) |

### Manual Attendance

**Path**: Attendance > Manual tab

1. **Select class** — dropdown auto-selects based on current timetable slot
2. **Select date** — defaults to today
3. **Mark status** — Present (default), Absent, or Late per student
4. **Bulk actions** — "All Present", "All Absent", "All Late" buttons
5. **Save** — only deviations from Present are stored

### Storage Design

Efficient: only **Absent** and **Late** records are written to the database. The absence of a record means "Present." This keeps the table small even for large schools.

### Other Attendance Methods

| Method    | Path                  | How It Works                   |
| --------- | --------------------- | ------------------------------ |
| Manual    | /attendance/manual    | Teacher marks each student     |
| QR Code   | /attendance/qr-code   | Students scan QR to check in   |
| Excuses   | /attendance/excuses   | Parents submit absence excuses |
| Analytics | /attendance/analytics | Dashboard with trends          |
| Reports   | /attendance/reports   | CSV/PDF export                 |

### Customizing Attendance

| Action                   | Where                 |
| ------------------------ | --------------------- |
| Attendance policies      | Attendance > Settings |
| Excuse categories        | Attendance > Settings |
| Notification rules       | Attendance > Settings |
| Early warning thresholds | Attendance > Settings |

## Setup Checklist

Quick reference for getting a school from zero to operational:

### Automatic (verify only)

- [ ] Academic grades appear in Grades listing
- [ ] Catalog subjects appear in Subjects listing
- [ ] Classrooms + sections appear in Classrooms listing
- [ ] Periods and working days appear in Timetable > Settings
- [ ] Terms appear in School > Settings
- [ ] Departments and score ranges exist

### Manual Steps (required)

- [ ] Import teachers (CSV or manual) — or verify import from onboarding
- [ ] Import students (CSV or manual) — or verify import from onboarding
- [ ] Assign grades to students (re-import or edit individually)
- [ ] Generate classes (Classrooms > Configure > Generate Classes)
- [ ] Enroll students (Classrooms > Configure > Enroll Students)
- [ ] Set teacher expertise (teacher → subject qualifications)
- [ ] Generate timetable (Timetable > Generate)

### Optional Customization

- [ ] Add room types (lab, library, sports) for specialized scheduling
- [ ] Adjust subject selections (add/remove/rename from catalog)
- [ ] Set weekly periods per subject for timetable weighting
- [ ] Configure attendance policies and notification rules
- [ ] Set up homeroom teachers per section
- [ ] Add co-teachers to classes

## Data Model

```
School (onboarding)
  │
  ├── AcademicLevel (auto)            Elementary, Middle, High
  │     └── AcademicGrade (auto)      Grade 1-12
  │           ├── Section (auto)      Student groups (7-A, 7-B)
  │           └── Classroom (auto)    Physical rooms
  │
  ├── SchoolSubjectSelection (auto)   Catalog → school subject bridge
  │
  ├── Term (auto)                     Academic terms
  ├── Period (auto)                   Daily time slots
  ├── SchoolWeekConfig (auto)         Working days
  │
  ├── Teacher (import)                From CSV or manual
  │     └── TeacherSubjectExpertise (manual)
  │
  ├── Student (import)                From CSV or manual
  │     └── academicGradeId (manual)  Grade assignment
  │
  ├── Subject (class generation)      School-scoped from catalog
  ├── Class (class generation)        Subject + Teacher + Term + Room
  │     └── StudentClass (enrollment) Student × Class
  │
  ├── Timetable (generation)          Day × Period × Class × Teacher × Room
  └── Attendance (daily)              Status deviations from Present
```

**Legend**: `(auto)` = auto-provisioned, `(import)` = CSV import, `(manual)` = requires admin action, `(generation)` = bulk action/algorithm, `(daily)` = ongoing operation.
