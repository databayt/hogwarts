The profile system gives every school user a GitHub-style profile: left sidebar (avatar, bio, stats) + right content (tabs, contribution graph, pinned items, activity timeline). One **optional catch-all route** handles both own-profile and view-by-ID.

## Routes

| Route                | Page                         | Behaviour                    |
| -------------------- | ---------------------------- | ---------------------------- |
| `/profile`           | `profile/[[...id]]/page.tsx` | Own profile (`isOwner=true`) |
| `/profile/<user-id>` | `profile/[[...id]]/page.tsx` | View by ID (`isOwner=false`) |

Path: `src/app/[lang]/s/[subdomain]/(school-dashboard)/profile/[[...id]]/`.

## Component architecture

Mirror pattern with all components in a single directory:

```
Route page.tsx
  → school-dashboard/profile/detail/content.tsx (ProfileDetailContent)
    → school-dashboard/profile/client.tsx (main layout)
      → graph.tsx | activity.tsx | pinned.tsx | sidebar.tsx
      → student.tsx | teacher.tsx | parent.tsx | staff.tsx
```

Directory `src/components/school-dashboard/profile/`:

```
actions.ts               # Server actions (CRUD, GitHub fields, contributions)
validation.ts            # Zod schemas
types.ts                 # Visual + system types
client.tsx               # Main layout (sidebar + tabs + content)
graph.tsx                # Contribution heatmap (real data via SWR)
activity.tsx             # Activity timeline below graph
pinned.tsx               # Pinned items grid
sidebar.tsx              # Avatar, bio, stats, links
form.tsx                 # Profile edit form
student.tsx | teacher.tsx | parent.tsx | staff.tsx   # Role dashboards
__tests__/actions.test.ts
detail/
  actions.ts             # getProfileById with permission filtering
  content.tsx            # ProfileDetailContent (entry from route)
  permissions.ts         # Permission resolution + data filtering
  types.ts               # ProfileData, PermissionLevel, ProfileContext
```

~60 dead / legacy files were deleted after verifying zero live imports — old `theme-provider.tsx`, header / sidebar shells, reference GitHub implementation, role dirs (each with content + directory + 6 tabs). The SaaS profile route (`(saas-dashboard)/profile/`) and its 7 files were also removed.

## Data model

### User as identity hub

```
User
├── Student?      (1:1 via userId)
├── Teacher?      (1:1 via userId)
├── Guardian?     (1:1 via userId)
└── StaffMember?  (1:1 via userId)
```

### GitHub-style fields on User

| Field           | Type      | Purpose                         |
| --------------- | --------- | ------------------------------- |
| `bio`           | `String?` | Short biography (max 500 chars) |
| `website`       | `String?` | Personal website URL            |
| `timezone`      | `String?` | User timezone                   |
| `statusEmoji`   | `String?` | Status emoji (max 10 chars)     |
| `statusMessage` | `String?` | Status message (max 100 chars)  |
| `pronouns`      | `String?` | User pronouns (max 50 chars)    |
| `socialLinks`   | `Json?`   | GitHub, Twitter, LinkedIn URLs  |

### UserActivity — 16 types

| Category       | Types                                                                           |
| -------------- | ------------------------------------------------------------------------------- |
| Academic       | `ASSIGNMENT_SUBMITTED`, `GRADE_RECEIVED`, `COURSE_ENROLLED`, `COURSE_COMPLETED` |
| Achievement    | `ACHIEVEMENT_EARNED`, `CERTIFICATE_EARNED`                                      |
| Library        | `LIBRARY_CHECKOUT`, `LIBRARY_RETURN`                                            |
| Events         | `EVENT_ATTENDED`, `CLUB_JOINED`                                                 |
| Projects       | `PROJECT_CREATED`, `PROJECT_UPDATED`                                            |
| Administrative | `ATTENDANCE_MARKED`, `EXAM_COMPLETED`, `PROFILE_UPDATED`, `OTHER`               |

### PinnedItem — up to 6, 10 types

`COURSE`, `SUBJECT`, `PROJECT`, `ACHIEVEMENT`, `CERTIFICATE`, `CLASS`, `CHILD`, `DEPARTMENT`, `PUBLICATION`, `TASK`. Each has `title`, `description`, `metadata` (JSON), `isPublic`, `order`.

## Permissions

5 levels: `OWNER > ADMIN > STAFF > RELATED > PUBLIC`.

```typescript
// detail/permissions.ts
function getPermissionLevel(context: ProfileContext): PermissionLevel {
  if (!viewerId)                          → "PUBLIC"
  if (viewerId === profileUserId)         → "OWNER"
  if (role is DEVELOPER or ADMIN)         → "ADMIN"
  if (role is TEACHER, STAFF, ACCOUNTANT) → "STAFF"
  if (same schoolId)                      → "RELATED"
  default                                 → "PUBLIC"
}
```

| Level   | Sees                                                                                                          |
| ------- | ------------------------------------------------------------------------------------------------------------- |
| OWNER   | Everything (full profile, private pinned items, settings)                                                     |
| ADMIN   | Everything (admin purposes)                                                                                   |
| STAFF   | Name, email, photo, school data, year levels, classes, guardians, medical info, phone numbers, qualifications |
| RELATED | Name, photo, student ID, enrollment date, year levels, teacher email, classes                                 |
| PUBLIC  | Name and photo only                                                                                           |

`filterProfileData()` applies per-level filtering. Each role model (student, teacher, guardian) has its own filter that strips sensitive fields based on viewer permission.

## Server actions

### Main (`actions.ts`)

| Action                                    | Purpose                                                       |
| ----------------------------------------- | ------------------------------------------------------------- |
| `getStudentProfile(id?)`                  | Student with classes, year levels, guardians                  |
| `getTeacherProfile(id?)`                  | Teacher with departments, classes, students                   |
| `getParentProfile(id?)`                   | Guardian with children and their classes                      |
| `getStaffProfile(id?)`                    | Staff user by role                                            |
| `updateProfile(input)`                    | Display name, avatar, locale                                  |
| `updateProfileBio(input)`                 | Bio (max 500 chars)                                           |
| `updateProfileSettings(input)`            | Theme, language, notifications                                |
| `uploadProfileAvatar(formData)`           | Avatar upload (placeholder)                                   |
| `getProfileBasicData(userId, lang?)`      | Flat data for sidebar with translation                        |
| `updateGitHubProfile(input)`              | Bio, website, timezone, status, pronouns, social links        |
| `getPinnedItems(userId?)`                 | Pinned items (public-only for other users)                    |
| `updatePinnedItems(items[])`              | Replace all pinned items (max 6)                              |
| `getContributionData(userId?, year?)`     | Heatmap from UserActivity + Attendance + AssignmentSubmission |
| `getRecentActivity(userId?, limit?)`      | Recent UserActivity entries                                   |
| `logUserActivity(input)`                  | Create UserActivity row                                       |
| `getUserProfileWithGitHubFields(userId?)` | Full user with GitHub fields, pinned items, activities        |

### Detail (`detail/actions.ts`)

| Action                   | Purpose                                                      |
| ------------------------ | ------------------------------------------------------------ |
| `getProfileById(userId)` | Fetch profile with all relations, apply permission filtering |
| `canViewProfile(userId)` | Boolean — same school or DEVELOPER role                      |

```typescript
export async function getProfileBasicData(userId: string, lang?: string) {
  const session = await auth()
  if (!session?.user?.id) return { success: false, error: "Not authenticated" }

  const { schoolId } = await getTenantContext()
  if (!schoolId) return { success: false, error: "School context not found" }

  const user = await db.user.findFirst({
    where: { id: userId, schoolId },
    select: {
      /* role-specific fields */
    },
  })

  return { success: true, data }
}
```

## Achievements

Sidebar displays GitHub-style PNG badges from `/github/{badge-name}.png`. Click opens a popover (badge title, description, earned date, context).

```typescript
interface Achievement {
  id: string
  title: string
  description: string
  icon: string // Maps to /github/{icon}.png
  earnedAt?: Date | string
  level?: "bronze" | "silver" | "gold" | "platinum"
  context?: string // e.g. "Term 1, 2025-2026 · Grade 10 Mathematics"
}
```

Badge names: `starstruck`, `galaxy-brain`, `pull-shark`, `yolo`, `pair-extraordinaire`, `quickdraw`, `public-sponsor`.

| Role    | Achievements (badge)                                                                                                                                                                                                |
| ------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Student | Student of the Year (`starstruck`), First / Second of Class (`galaxy-brain`), Best Football Player (`yolo`), Perfect Attendance (`quickdraw`), Science Fair Winner (`galaxy-brain`), Math Olympiad (`galaxy-brain`) |
| Teacher | Teacher of the Year (`starstruck`), Best Curriculum Design (`pull-shark`), Student Favorite (`starstruck`), Department Leader (`pull-shark`), Perfect Attendance (`quickdraw`), Innovation Award (`starstruck`)     |
| Parent  | Most Engaged Parent (`pair-extraordinaire`), Community Champion (`pair-extraordinaire`), Volunteer of the Year (`public-sponsor`)                                                                                   |
| Staff   | Employee of the Year (`starstruck`), Best Team Player (`pair-extraordinaire`), Perfect Attendance (`quickdraw`), Service Excellence (`pull-shark`), Outstanding Support (`galaxy-brain`)                            |

`AchievementPopover` (in `sidebar.tsx`):

- **Trigger** — `size-16` (64 px) badge PNG via `<img>`, no hover scale
- **Content** — `w-[360px]`, `p-0`, `overflow-hidden`, `side="top"`
  - Header — per-badge gradient PNG from `achievementDetailBg`, badge centred at `size-[140px]`
  - Title `<h3>` `text-base font-bold`, description `text-sm text-muted-foreground`
  - `<hr className="mx-4 mt-3">`
  - History — `OcticonTrophy` (muted) + `"Unlocked on {date}"` + bullet + context
