The catalog hierarchy is three global tables. All are scope-free (no `schoolId`); schools reference them via bridge tables.

## Schema

```
Subject (337 rows)            тЖТ table catalog_subjects
  тФЬтФАтФА id, slug (unique), name, lang
  тФЬтФАтФА country, curriculum          тЖР multi-curriculum routing
  тФЬтФАтФА curriculumId (FK)            тЖР optional FK into Curriculum
  тФЬтФАтФА concept                      тЖР cross-curriculum image key
  тФЬтФАтФА thumbnail (@map thumbnailKey)тЖР S3/CDN path (unified image system)
  тФЬтФАтФА grades[], levels[]
  тФЬтФАтФА department, color, status
  тФЬтФАтФА totalChapters, totalLessons  тЖР denormalized counts
  тФФтФАтФА Chapter (1,156 rows)         тЖТ table catalog_chapters
        тФЬтФАтФА id, slug, name, lang
        тФЬтФАтФА subjectId (FK)
        тФЬтФАтФА sequenceOrder, grades[], levels[]
        тФЬтФАтФА totalLessons, totalContent
        тФФтФАтФА Lesson (5,042 rows)    тЖТ table catalog_lessons
              тФЬтФАтФА id, slug, name, lang
              тФЬтФАтФА chapterId (FK)
              тФФтФАтФА sequenceOrder, grades[], levels[]
```

Key fields for multi-curriculum routing:

```prisma
model Subject {                                    // table catalog_subjects
  country      String        @default("SD")        // ISO 3166-1 alpha-2
  curriculum   String        @default("SD")         // canonical code: bare ISO (US, SD, GB) or BODY-PROGRAMME (IB-DP, CAIE-IGCSE, CBSE)
  curriculumId String?                             // optional FK тЖТ Curriculum
  concept      String?                             // shared image key
  thumbnail    String?       @map("thumbnailKey")  // S3 key for WebP variants (unified)
  grades       Int[]         @default([])          // [5] = grade 5 only
  levels       SchoolLevel[]                       // [ELEMENTARY]
  slug         String        @unique               // sd-g5-math, us-g5-math
}
```

`slug` is the primary upsert key. It encodes country + grade + subject; no compound unique on `(country, curriculum, grade, name)` is needed.

## Runtime flow

### School onboarding

```
School.country = "SD"
  тЖТ inferCurriculum("SD") тЖТ "SD"
  тЖТ findSubjects("SD", "SD")
       Step 1  exact   (country + curriculum + schoolType)
       Step 2  broad   (country + curriculum)            тЖР SD hits here (97 subjects)
       Step 3  worldwide (country = "*")
       Step 4  fallback (US + US)
  тЖТ filter by school.schoolLevel
       primary    тЖТ grades 1-6  (ELEMENTARY)
       secondary  тЖТ grades 7-12 (MIDDLE + HIGH)
       both       тЖТ grades 1-12
  тЖТ create SubjectSelection bridge rows
  тЖТ create YearLevel, Department, Stream, ScoreRange defaults
```

### Subject display

```
SubjectSelection.findMany({ schoolId })

       1. thumbnailKey  тЖТ CDN: catalog/concepts/math/thumbnail-sm.webp
       2. concept       тЖТ static SUBJECT_IMAGE_MAP["math"] fallback
```

## Seed architecture

Seeds are idempotent (upsert). These two produce the bulk of the catalog content:

| Seed                    | Source                   | Subjects | Chapters | Lessons | Slug                  | Country | Curriculum   | Lang |
| ----------------------- | ------------------------ | -------: | -------: | ------: | --------------------- | ------- | ------------ | ---- |
| `catalog/us.ts`         | `us-inventory.json`      |      240 |      750 |   3,993 | `us-g{N}-{subject}`   | US      | `US`         | en   |
| `catalog/sd.ts`         | `curriculum/sd/`         |       97 |      406 |   1,049 | `sd-g{N}-{subject}`   | SD      | `SD`         | ar   |
| `catalog/gb.ts`         | `curriculum/uk/`         |      184 |      363 |     730 | `gb-g{N}-{subject}`   | GB      | `GB`         | en   |
| `catalog/cbse.ts`       | `curriculum/in/`         |        7 |       58 |      58 | `cbse-g{N}-{subject}` | IN      | `CBSE`       | en   |
| `catalog/caie-igcse.ts` | `curriculum/caie-igcse/` |        8 |       76 |      76 | `caie-g{N}-{subject}` | \*      | `CAIE-IGCSE` | en   |
| `catalog/ib.ts`         | `curriculum/ib/`         |        9 |       49 |      49 | `ib-g{N}-{subject}`   | \*      | `IB-DP`      | en   |

The four deep tree sources (`gb`/`cbse`/`caie-igcse`/`ib`) all run through the shared
`syncCurriculumTree` engine (see [Unified tree engine](#unified-tree-engine)). The
**six subjects-only nationals** (SA, EG, AE, QA, KW, JO) each live in their own file
(`sa.ts`тАж`jo.ts`) sharing `seedSubjectsOnly`; GB and IB graduated to deep tree
sources. `catalog/sd-base.ts` remains a smaller Sudan base set (~32 subjects). The
default `catalog` seed runs `us` + the full `sd` + `registry`; every other source is
an explicit run.

A `catalog/concepts.ts` seed runs after the catalog seeds:

1. Reads `public/subjects/concepts/*.jpg` (24 images).
2. Sharp тЖТ 4 WebP variants тЖТ S3 upload.
3. Backfills `concept` field on subjects via `NAME_TO_CONCEPT`.
4. Sets `thumbnailKey` on subjects that don't have one.

| ----------------------------------- | -------------------------------------------------------- |

## Unified tree engine

The duplicated per-seed logic (upsert, denormalized count update, chapter/lesson
loop, concept assignment) is now factored into one shipped helper,
`syncCurriculumTree`, in `prisma/seeds/catalog/engine.ts`. A new curriculum
is a ~20-line source file that hands it a config:

```ts
// prisma/seeds/catalog/gb.ts
import { syncCurriculumTree } from "./engine"

export async function seedGbCurriculum(prisma: PrismaClient) {
  return syncCurriculumTree(prisma, {
    dir: "uk", // curriculum/uk/
    country: "GB",
    code: "GB", // canonical curriculum code
    slugPrefix: "gb", // gb-g{N}-{subject}
    name: "British (England)",
    lang: "en",
    organization: "UK Department for Education",
    gradeRange: [1, 12],
    sortBase: 20000,
  })
}
```

`syncCurriculumTree` reads any `curriculum/<dir>/g{N}/<subject>/structure.json`
tree, upserts `Subject`/`Chapter`/`Lesson` with the canonical `code`/`country`,
attaches the textbook PDF S3 key when `textbook.pdf` is present, and rotates the
shared concept thumbnail/banner/cover across chapters. It is idempotent (subject
upsert by slug; chapters/lessons replaced). Subjects carry only the curriculum
**code** тАФ `registry.ts` owns the `Curriculum` row and backfills `curriculumId`.
`gb.ts`, `cbse.ts`, `caie-igcse.ts`, and `ib.ts` are the four live callers.

### On-disk contract тАФ `structure.json`

The engine (`curriculum/_build_tools/engine.py`) writes, and
`syncCurriculumTree` reads, this tree shape:

```
curriculum/<dir>/
  g{N}/                         # grade
    <subject>/                  # e.g. math, science
      structure.json            # { subject, chapters: [{ name, slug, lessons: [...] }] }
      textbook.pdf              # optional тАФ merged free official PDF
```

All future curricula write to this shape. Legacy formats
(`us-inventory.json`, `sudan-curriculum.json`) keep their bespoke seeds
(`us.ts`, `sd.ts`) тАФ only net-new curricula go through the tree engine.

## Improvement areas

### Hoist the concept maps

`SUBJECT_CONCEPT` / rotation `POOL` / `DEPARTMENT` maps are duplicated across
four files (`concepts.ts`, `us.ts`, `sd.ts`, `engine.ts`). Extract a single
`concepts-data.ts` and import everywhere.

### Slug convention (unified)

All curricula use the same pattern: `{cc}-g{N}-{subject}`. Unified March 2026; the US K-12 seed migrates legacy `us-{subject}-grade-{N}` slugs automatically.

### Image layer unification (complete)

Both curricula now share the unified image system: `thumbnailKey` тЖТ concept static fallback. The legacy `imageKey` field has been dropped from `Subject`. The 240 US per-subject keys collapsed into 23 shared concept keys, matching the SD topology. See [/docs/concept](/docs/concept) for the canonical concept system.

### TOC extraction quality

Many Sudan PDFs (especially grades 6-8 and 10-12 secondary) produced garbled or empty `pdftotext` output. Subjects with chapters but 0 lessons:

| Grade range | With lessons | With 0 lessons |
| ----------- | -----------: | -------------: |
| 1-5         |     31 (89%) |        4 (11%) |
| 6-9         |      8 (40%) |       12 (60%) |
| 10-12       |     14 (29%) |       34 (71%) |

Priorities: OCR pass with Tesseract Arabic on scanned PDFs (grade 6) тЖТ manual entry for 10-12 тАФ community contribution via the content layer.

### Denormalized count update performance

Today: ~97├Ч3 + 406├Ч2 тЙИ 1,103 queries. Sudan seed took 822s, mostly on this loop. Replace with two bulk SQL statements:

```sql
UPDATE catalog_chapters SET
  "totalLessons" = sub.count,
  "totalContent" = sub.count
FROM (SELECT "chapterId", COUNT(*) AS count FROM catalog_lessons GROUP BY "chapterId") sub
WHERE catalog_chapters.id = sub."chapterId";

UPDATE catalog_subjects SET
  "totalChapters" = ch.count,
  "totalLessons"  = ls.count,
  "totalContent"  = ls.count
FROM
  (SELECT "subjectId", COUNT(*) AS count FROM catalog_chapters GROUP BY "subjectId") ch,
  (SELECT c."subjectId", COUNT(*) AS count FROM catalog_lessons l
   JOIN catalog_chapters c ON l."chapterId" = c.id GROUP BY c."subjectId") ls
WHERE catalog_subjects.id = ch."subjectId"
  AND catalog_subjects.id = ls."subjectId";
```

Expected: 822s тЖТ ~5s.
