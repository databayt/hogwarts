All changes go through Pull Requests → Code Review → CI → Merge. The `main` branch is protected: no force pushes, no deletes, no merging without approval.

## Branch protection

| Required                              | Blocked                |
| ------------------------------------- | ---------------------- |
| Pull request for all changes          | Force pushes           |
| 1 approval from a `CODEOWNERS` member | Deleting `main`        |
| All CI checks passing                 | Merge without approval |
| Conversation resolution               |                        |

## Pull request flow

```bash
# 1. fork via GitHub UI, then clone your fork
git clone git@github.com:YOUR_USERNAME/hogwarts.git
cd hogwarts
git remote add upstream git@github.com:abdout/hogwarts.git

# 2. branch from latest main
git checkout main
git pull upstream main
git checkout -b feature/your-feature-name

# 3. commit with a conventional message
git add .
git commit -m "feat(platform): add attendance export to CSV"

# 4. push and open the PR via GitHub
git push origin feature/your-feature-name
```

Branch prefixes:

| Prefix      | Use           |
| ----------- | ------------- |
| `feature/`  | New features  |
| `fix/`      | Bug fixes     |
| `docs/`     | Documentation |
| `refactor/` | Refactoring   |

Commit message format: `type(scope): description`. Types: `feat`, `fix`, `docs`, `style`, `refactor`, `test`, `chore`, `ci`. Scope is the folder name (`platform`, `auth`, `marketing`, `onboarding`, `ui`, `atom`, `template`, `docs`).

## CI checks

Every PR runs `pr-check.yml`:

| Job         | Purpose             | Must pass |
| ----------- | ------------------- | --------- |
| `typecheck` | `pnpm tsc --noEmit` | yes       |
| `lint`      | ESLint              | yes       |
| `test`      | Vitest unit tests   | yes       |
| `build`     | Production build    | yes       |

Security: `codeql.yml` scans on every PR + weekly. Dependencies: `dependabot.yml` opens grouped PRs every Monday.

## Auto-labeling

PRs are auto-labeled by file paths:

| Label            | Files                                |
| ---------------- | ------------------------------------ |
| `area: platform` | `src/components/school-dashboard/**` |
| `area: auth`     | `src/auth*`, `auth.ts`               |
| `area: database` | `prisma/**`                          |
| `area: docs`     | `docs/**`, `content/**`, `*.md`      |
| `area: ui`       | `src/components/ui/**`               |
| `type: ci`       | `.github/**`                         |
| `type: tests`    | `*.test.*`, `tests/**`               |

Stale: issues / PRs marked stale after 30 days of inactivity, closed after 7 more. Exempt labels: `pinned`, `security`, `help wanted`, `good first issue`.

## CODEOWNERS

The `.github/CODEOWNERS` file defines who must review which paths:

```
* @abdout
/src/components/school-dashboard/ @abdout
/prisma/ @abdout
/src/auth* @abdout
/.github/ @abdout
```

Modifying files in those paths auto-requests the owner.

## Issue templates

When opening an issue, pick the template. Bug reports include description, repro steps, expected vs actual, screenshots, browser / OS, affected area. Feature requests include problem, proposed solution, alternatives, priority, willingness to contribute. Doc issues include location, what's wrong, suggested improvement.

## Security

**Don't** open public issues for vulnerabilities. Email `security@databayt.org` with type, affected files, repro, impact. Response within 48 hours. See `SECURITY.md` for the full policy.

When contributing:

- Always include `schoolId` in database queries (multi-tenant safety).
- Use Zod schemas for input validation.
- Never commit secrets or credentials.
- Keep dependencies up to date.

## Pre-PR checklist

- `pnpm tsc --noEmit` clean
- `pnpm lint` clean
- `pnpm test` green
- Tested locally
- Docs updated if needed
- PR template filled out

## Essential git commands

```bash
# stay synced with upstream
git checkout main
git fetch upstream
git merge upstream/main
git push origin main

# rebase your feature branch
git checkout feature/your-feature
git rebase main

# undo last commit (keep changes)
git reset --soft HEAD~1

# delete local + remote branch
git branch -d feature/your-feature
git push origin --delete feature/your-feature
```
