## Overview

Teachers enter the system through **five entry points**, each now unified under a single validation schema and wizard infrastructure. Whether an admin creates a teacher via the wizard, imports from CSV, or a teacher joins through internal onboarding — the data flows through the same pipeline and incomplete records are clearly marked for completion.

```
                        TEACHER ENTRY POINTS
  ┌──────────────────────────────────────────────────────────────┐
  │                                                              │
  │  1. Admin Wizard         → 7 steps, 35+ fields, draft flow  │
  │  2. CSV Bulk Import      → ~9 fields, marked incomplete     │
  │  3. Internal Onboarding  → ~15 fields, marked incomplete    │
  │  4. Profile Self-Edit    → Contact, qualifications, exp     │
  │  5. Admin Detail Edit    → Link to wizard at any step       │
  │                                                              │
  └──────────────────────────────────────────────────────────────┘
```

## Wizard Flow

The primary creation path is a **7-step wizard** that creates a draft record on entry and progressively fills it in. Each step saves independently, so the teacher can be resumed at any point.

A **Skip & Create** fast path lets admins complete the wizard early — once the required fields (name + email) are filled, a "Skip & Create" button appears that saves the current step, marks the wizard complete, and redirects to the teacher list.

### Steps

| Step              | Route Segment     | Fields                                                                                        | Required?               |
| ----------------- | ----------------- | --------------------------------------------------------------------------------------------- | ----------------------- |
| 1. Photo          | `/photo`          | profilePhotoUrl                                                                               | None                    |
| 2. Information    | `/information`    | givenName, surname, gender, birthDate                                                         | givenName, surname      |
| 3. Contact        | `/contact`        | emailAddress, phoneNumbers[]                                                                  | emailAddress            |
| 4. Employment     | `/employment`     | employeeId, joiningDate, employmentStatus, employmentType, contractStartDate, contractEndDate | None (defaults applied) |
| 5. Qualifications | `/qualifications` | qualifications[] (type, name, institution, major, dates, license)                             | None                    |
| 6. Experience     | `/experience`     | experiences[] (institution, position, dates, isCurrent)                                       | None                    |
| 7. Expertise      | `/expertise`      | subjectExpertise[] (subjectId, expertiseLevel)                                                | None                    |

### Route Structure

```
src/app/[lang]/s/[subdomain]/(school-dashboard)/(listings)/
  teachers/
    add/
      page.tsx              # Creates draft, redirects to step 1
      [id]/
        photo/page.tsx
        information/page.tsx
        contact/page.tsx
        employment/page.tsx
        qualifications/page.tsx
        experience/page.tsx
        expertise/page.tsx
```

### Draft Lifecycle

```
1. Admin clicks "Add Teacher"
2. page.tsx creates draft: { givenName: "", emailAddress: "draft-xxx@draft.internal", wizardStep: "photo" }
3. Each step saves fields + advances wizardStep
4. Two completion paths:
   a. FAST: "Skip & Create" on contact step (or later) → saves + completes + redirects
   b. FULL: Click through all 7 steps → final step completes wizard
5. Both paths set wizardStep: null (complete)
6. Complete teachers appear in the main list (wizardStep: null filter)
7. Abandoned drafts can be deleted via deleteDraftTeacher()
```

## Validation (Single Source of Truth)

The root `validation.ts` composes from wizard step schemas. This ensures the wizard forms, server actions, CSV import, and onboarding all validate against the same rules.

```typescript
// teachers/validation.ts — composed from wizard step schemas
import { contactSchema } from "./wizard/contact/validation"
import { experienceItemSchema } from "./wizard/experience/validation"
import { expertiseItemSchema } from "./wizard/expertise/validation"
import { informationSchema } from "./wizard/information/validation"
import { qualificationSchema } from "./wizard/qualifications/validation"

export const teacherCreateSchema = informationSchema
  .merge(contactSchema)
  .extend({
    // Employment fields (can't merge refined schemas)
    employeeId: z.string().optional(),
    joiningDate: z.coerce.date().optional(),
    employmentStatus: z
      .enum(["ACTIVE", "ON_LEAVE", "TERMINATED", "RETIRED"])
      .default("ACTIVE"),
    employmentType: z
      .enum(["FULL_TIME", "PART_TIME", "CONTRACT", "SUBSTITUTE"])
      .default("FULL_TIME"),
    contractStartDate: z.coerce.date().optional(),
    contractEndDate: z.coerce.date().optional(),
    // Nested arrays
    qualifications: z.array(qualificationSchema).optional().default([]),
    experiences: z.array(experienceItemSchema).optional().default([]),
    subjectExpertise: z.array(expertiseItemSchema).optional().default([]),
  })
  .refine(/* contract dates + birth/joining cross-validation */)
```

### Re-exported Schemas

Individual schemas are re-exported for consumers:

| Export                   | Source                             | Used By                |
| ------------------------ | ---------------------------------- | ---------------------- |
| `phoneNumberSchema`      | `wizard/contact/validation`        | CSV import, onboarding |

## Entry Point Details

### 1. Admin Wizard (Primary)

The full 7-step wizard. Creates a draft record, fills fields progressively, and marks complete either via **Skip & Create** (after required fields) or by completing all steps.

**wizardStep lifecycle**: `"photo"` -> `"information"` -> `"contact"` -> ... -> `null` (complete)

**Skip & Create**: Once givenName + surname (information) and emailAddress (contact) are filled, the "Skip & Create" button appears on the contact step and all subsequent steps. Clicking it saves the current step, calls `completeTeacherWizard()`, and redirects to `/teachers`.

### 2. CSV Bulk Import

```typescript
// csv-import.ts — importTeachers()
await tx.teacher.createMany({
  data: chunk.map((r) => ({
    schoolId,
    givenName: parts[0],
    surname: parts.slice(1).join(" "),
    emailAddress: r.validated.email,
    wizardStep: "employment", // Marked incomplete — CSV covers name + contact only
  })),
})
```

CSV-imported teachers appear in the "Incomplete" tab. Admins click to open the wizard at the `employment` step to fill in remaining details.

### 3. Internal Onboarding

When a teacher joins through the school's join page, the onboarding flow creates:

- Teacher record with personal + contact fields + `wizardStep: "employment"`
- TeacherPhoneNumber (if phone provided)
- TeacherQualification (if qualification provided)
- TeacherSubjectExpertise (for selected subjects)

### 4. Profile Self-Edit

Teachers can edit their own:

- **Contact** information (email, phone numbers)
- **Qualifications** (degrees, certifications, licenses)
- **Experience** (teaching history)

They **cannot** edit:

- Personal information (name, gender, birth date) — admin only
- Employment details (status, type, contract dates) — admin only
- Subject expertise assignments — admin only

Self-edit uses the same wizard step forms rendered in a dialog on the profile page.

### 5. Admin Detail Edit

The teacher detail view has an "Edit" button that links to `/{lang}/teachers/add/{teacherId}/information` — opening the wizard pre-populated with existing data.

## Incomplete Records

### How Records Become Incomplete

| Source               | wizardStep Set To | Fields Covered                              |
| -------------------- | ----------------- | ------------------------------------------- |
| Wizard (in progress) | Current step name | All steps up to current                     |
| CSV Import           | `"employment"`    | Name, email, phone, department              |
| Internal Onboarding  | `"employment"`    | Name, email, phone, qualification, subjects |

### How Admins Find Them

- Main teacher list filters `wizardStep: null` by default (shows only complete records)
- Toggle **"Show Incomplete"** or use the status filter to see drafts
- Incomplete records show an **amber "Incomplete" badge** on the name column
- The action dropdown shows **"Complete Profile"** instead of "Edit", linking directly to the wizard at the incomplete step
- Admins can also use **Skip & Create** when completing an incomplete record — once required fields are present, skip remaining optional steps

### Query Parameter

```typescript
// To include incomplete records in queries:
buildTeacherWhere(schoolId, { includeDrafts: true })
```
