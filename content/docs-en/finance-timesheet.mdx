title: Timesheet
description: Staff time tracking via pay periods and per-day entries (hours, overtime, leave) with an open ŌåÆ closed ŌåÆ approved workflow. Period and entry CRUD work; there is no clock-in UI, no overtime detection, and the manual-entry form is a placeholder.
---

Timesheet manages pay periods and the time entries within them. Approved timesheets are _intended_ to feed [payroll](/docs/finance-payroll)'s day/hour counts ŌĆö but that link is not wired today, so payroll hardcodes 22 days regardless.

## Flow

```
1. Create period (OPEN)                              ŌĆö createTimesheet
2. Add entries (hours / overtime / leave per day)    ŌĆö addTimesheetEntry  (db.$transaction)
3. Submit period (ŌåÆ CLOSED)                          ŌĆö submitTimesheet
4. Approve / reject                                  ŌĆö approveTimesheet
```

## Server actions

RBAC is enforced in `content.tsx`. These actions resolve the tenant from `session.user.schoolId` directly rather than `getTenantContext()` (an inconsistency with payroll/salary ŌĆö flagged for cleanup). `addTimesheetEntry` wraps its write in `db.$transaction`.

| Action              | Purpose                                         |
| ------------------- | ----------------------------------------------- |
| `createTimesheet`   | Create period (OPEN)                            |
| `addTimesheetEntry` | Create entry + reload period in one transaction |
| `submitTimesheet`   | Period ŌåÆ CLOSED                                 |
| `approveTimesheet`  | Set approve/reject status + `closedBy`          |
| `getTimesheets`     | List periods + entries                          |

## Concepts

| Concept        | Values                                                                       |
| -------------- | ---------------------------------------------------------------------------- |
| `PeriodStatus` | OPEN, CLOSED, LOCKED                                                         |
| `EntryStatus`  | DRAFT, SUBMITTED, APPROVED, REJECTED                                         |
| Config rules   | `PERIOD_DURATION_DAYS` 14, `ROUNDING_INTERVAL` 0.25, `REQUIRE_APPROVAL` true |

## Ledger / accounting

No journal posting ŌĆö timesheet is upstream of payroll and has no money event of its own.
