Hogwarts is a multi-tenant K-12 school management platform serving administrators, teachers, students, parents, accountants, and staff. This page captures the requirements; for implementation status see [MVP](/docs/mvp).

## Classification

- **Type:** SaaS B2B multi-tenant web application.
- **Domain:** EdTech — K-12 school management.
- **Complexity:** enterprise-scale (200+ functional requirements, regulated domain).

## Scope

A school can enroll students, manage classes, track attendance, grade assignments, collect fees, and communicate with parents — all on one platform. See [MVP](/docs/mvp) for the 13-epic breakdown.

### Beyond MVP

Growth-phase features expanding capabilities and reach:

- AI predictive analytics.
- Digital classroom and LMS integration.
- Library management.
- Native mobile apps (React Native).
- French, Spanish, Hindi, Mandarin locales.
- Regional compliance certifications.

### Vision

- AI teaching assistant.
- Personalized learning paths.
- Marketplace and plugin ecosystem.
- Blockchain credentials.
- VR / AR learning experiences.
- District-level management.
- Global teacher collaboration network.

## Stack

Multi-tenant subdomain architecture; each school is `schoolId`-scoped. Stateless serverless on Vercel with 8-role RBAC. Stack: Next.js 16, React 19, TypeScript, Prisma, PostgreSQL, NextAuth v5, Tailwind CSS 4, shadcn/ui.

```text
Production:   school.databayt.org → /[lang]/s/school/...
Preview:      tenant---branch.vercel.app → /[lang]/s/tenant/...
Development:  subdomain.localhost:3000 → /[lang]/s/subdomain/...
Custom:       www.theirschool.com (CNAME) → Hogwarts platform
```

| ---------- | ------------------------------------------------------------ |
| DEVELOPER  | All schools, platform configuration, monitoring              |

| TEACHER    | Manage assigned classes, grade assignments, mark attendance  |

## API and integrations

### REST API v1

- OpenAPI 3.0 spec with Swagger UI.
- API key auth and tiered rate limiting (100 req/hr → unlimited).
- Webhooks: `student.enrolled`, `payment.received`, `grade.updated`.
- HMAC-SHA256 signing; retry with exponential backoff.

### SSO (Enterprise)

- SAML 2.0 — Okta, Azure AD, OneLogin.
- OAuth 2.0 — Google Workspace, Microsoft 365.
- SCIM for user provisioning.

### Third-party

Google Workspace, Microsoft 365, Zoom, Stripe, Twilio, Resend, QuickBooks / Xero, Canvas / Blackboard / Moodle (LTI).

## Non-functional requirements

### Performance

| Metric                 | Target             |
| ---------------------- | ------------------ |
| Initial page load (3G) | &lt;2s             |
| Subsequent navigation  | &lt;500ms          |
| API response (P95)     | &lt;500ms          |
| Database queries (P95) | &lt;100ms          |
| TTFB                   | &lt;200ms via edge |

### Security

- AES-256 at rest, TLS 1.3 in transit.
- Bcrypt (cost 12), JWT (24h), 2FA, rate limiting.
- RBAC on every route; multi-tenant isolation enforced server-side.
- Zod server-side validation; SQL injection and XSS prevention.
- Audit logging for all modifications, 7-year retention.

### Scalability

- Prisma connection pooling on Neon Serverless PostgreSQL.
- Vercel serverless, stateless, edge caching.
- S3-compatible storage (Cloudflare R2) with CDN delivery.

### Accessibility

- WCAG 2.1 AA compliance.
- Keyboard navigation and screen-reader support.
- Color contrast minimum 4.5:1.
- Automated axe-core checks in CI/CD.

### Compliance

| Regulation | Scope                               |
| ---------- | ----------------------------------- |
| FERPA      | US student privacy                  |
| GDPR       | EU data protection                  |
| COPPA      | Parental consent for users under 13 |
| WCAG 2.1   | AA accessibility                    |

- 1-week sprints with weekly standups.
- Code review (minimum 1 approver).
- 80%+ coverage on critical paths.
