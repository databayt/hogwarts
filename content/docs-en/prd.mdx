### Tenancy model

Subdomain-based multi-tenancy resolved at the edge.

```text
Production:   school.databayt.org             →  /[lang]/s/school/...
Pilot tenant: kingfahad.databayt.org          →  /[lang]/s/kingfahad/...
Preview:      tenant---branch.vercel.app      →  /[lang]/s/tenant/...
Development:  subdomain.localhost:3000        →  /[lang]/s/subdomain/...
Custom:       www.theirschool.com (CNAME)     →  Hogwarts platform
```

Resolution flow:

1. Edge `proxy.ts` extracts the subdomain from the host header.
2. Custom domains are mapped to canonical subdomains via Upstash Redis lookup (`custom-domain:${host}`).
3. Internal rewrite to `/[lang]/s/[subdomain]/...`; the `/s/[subdomain]` prefix is **internal only** — never used in `redirect()` / `Link href` / `router.push`.
4. `x-subdomain` header is set for downstream handlers.
5. `getTenantContext()` reads the header and resolves `schoolId` for use in every Prisma query.

### Single-language content storage

Hogwarts does not use bilingual columns (`titleEn`/`titleAr`). All user-generated content is stored in the school's chosen language with a `lang` field. Reads in the other language go through Google Translate, with results cached in `TranslationCache` for repeat hits. UI strings live in dictionaries (`src/components/internationalization/dictionaries/`); `getDisplayText()` (`@/lib/content-display`) is the canonical accessor.

## Personas

Eight roles, all defined in `prisma/models/auth.prisma:UserRole`. Authorization is enforced by CASL abilities (`@/lib/abilities`) and route-level role gates.

| Role | Who | UI surface | Auth helper |
| ---- | --- | ---------- | ----------- |
| `DEVELOPER` | Platform staff (no `schoolId`) | `/[lang]/(saas-dashboard)/*` — tenants, observability, catalog ops | `requireDeveloper()` |
| `ADMIN` | Principal, head of school | Full per-tenant dashboard at `/[lang]/s/[subdomain]/*` | `requireRole("ADMIN")` |
| `TEACHER` | Classroom teacher | `school-dashboard/*` (assigned classes, grades, attendance) | role gate |
| `STUDENT` | Enrolled student | `school-dashboard/*` (own grades, assignments, attendance) | role gate |
| `GUARDIAN` | Parent / legal guardian | Parent area + linked-student views | role gate |
| `ACCOUNTANT` | Finance staff | `school-dashboard/finance/*` (no academic modules) | role gate |
| `STAFF` | Operational staff (admin office, librarian, driver-coordinator) | Limited operational modules | role gate |
| `USER` | Default post-signup, pre-onboarding | `/onboarding/[id]/*` only | — |

## Functional scope

The 14-epic shipped state is enumerated in [MVP](/docs/mvp). The Next + Vision roadmap lives in [Roadmap](/docs/roadmap). Detail docs per epic live alongside in `content/docs-en/`. Block-level CLAUDE.md files (`src/components/<feature>/CLAUDE.md`) document the per-feature implementation contract.

A school can: enrol applicants → register students → manage classes / sections / subjects / timetable → mark attendance (11 capture modes) → grade exams + generate report cards → collect fees (6 payment gateways) → message parents (in-app + email + WhatsApp) → run an LMS → manage library + transport — all on one platform with `schoolId`-scoped isolation.

## Non-functional requirements

### Performance

| Metric | Target |
| ------ | ------ |
| Initial page load (3G) | &lt; 2s |
| Subsequent navigation | &lt; 500ms |
| API response (P95) | &lt; 500ms |
| Database queries (P95) | &lt; 100ms |
| TTFB | &lt; 200ms via edge |
| Lighthouse score | ≥ 90 |

### Security

- AES-256 at rest, TLS 1.3 in transit.
- Bcrypt (cost 12) for passwords; JWT sessions; 2FA via `TwoFactorToken` + `TwoFactorConfirmation`.
- Brute-force protection via `LoginAttempt` audit + rate limits.
- RBAC on every route; multi-tenant isolation enforced server-side.
- Zod validation server-side; SQL injection and XSS prevention.
- `AuditLog` retention with structured event types for all sensitive mutations.
- 11 daily cron jobs for housekeeping (notifications, cleanups, reminders, fee overdue).

### Scalability

- Prisma connection pooling on Neon serverless PostgreSQL.
- Vercel serverless, stateless functions; aggressive `outputFileTracingExcludes` keeps function bundles below the 300 MB limit.
- S3 + CloudFront (`cdn.databayt.org`) for asset delivery.
- Server-side `experimental.serverActions.bodySizeLimit: "10mb"` for file uploads.

### Accessibility

- WCAG 2.1 AA targeted.
- Keyboard navigation, screen-reader support, color contrast ≥ 4.5:1.
- Logical CSS properties (no `ml-*` / `mr-*`) so RTL works without overrides.
- Semantic HTML — no hardcoded `text-*` / `font-*` Tailwind classes.

### Compliance

| Regulation | Scope |
| ---------- | ----- |
| FERPA | US student privacy |
| GDPR | EU data protection |
| COPPA | Parental consent under 13 |
| WCAG 2.1 AA | Accessibility |

## Constraints

These rules apply to every contribution. They are enforced by hooks, agents, and review gates.

- **License — SSPL-1.0.** Self-host is free. Reselling Hogwarts as a hosted service requires a commercial license.
- **Multi-tenant safety.** Every Prisma query MUST include `schoolId` (or be on a global model that explicitly opts out). Isolation tests in `tests/multi-tenant/` guard regressions.
- **Single-language storage.** Never add bilingual columns. Use a `lang` field and rely on `TranslationCache` for cross-locale reads.
- **Logical CSS only.** Use `ms-*` / `me-*` / `ps-*` / `pe-*` (start / end) — never `ml-*` / `mr-*` / `pl-*` / `pr-*`. RTL must work without overrides.
- **No raw HTML primitives in components.** Use shadcn `ui/` primitives. No raw `<button>` / `<input>` / `<select>` in feature code.
- **Mirror pattern.** Routes import from `src/components/<feature>/content.tsx`. Per-feature canonical filenames: `content.tsx`, `actions.ts`, `queries.ts`, `authorization.ts`, `validation.ts`, `form.tsx`, `table.tsx`, `columns.tsx`.
- **One central `.env`.** Never `.env.local`, `.env.development`, etc.
- **Port 3000 in dev — always.**
- **`pnpm tsc --noEmit` before any build.** Vercel build has `typescript.ignoreBuildErrors: true` to dodge OOM, so type-check is a separate gate.
- **Internal rewrite safety.** `/s/[subdomain]` is an internal route only. UI must use the clean tenant-relative path.
- **All UI text via dictionary keys.** No hardcoded English or Arabic strings in components.

## API and integrations

### REST API v1

- OpenAPI 3.0 contract.
- API key auth with tiered rate limiting.
- Webhook events: `student.enrolled`, `payment.received`, `grade.updated`, plus the full `webhooks.prisma:ProcessedWebhookEvent` log.
- HMAC-SHA256 request signing; retry with exponential backoff.

### SSO (Enterprise)

- SAML 2.0 — Okta, Azure AD, OneLogin.
- OAuth 2.0 — Google Workspace, Microsoft 365.
- SCIM for user provisioning.

### Third-party connections (built or wired)

Stripe (payments + subscriptions), bankak / tap / mobile-money / bank-transfer / cash (additional payment gateways), Plaid + Dwolla (banking), Resend (email), Twilio (SMS), WhatsApp Evolution API (messaging), Google Translate (on-demand translation), AWS S3 + CloudFront (assets), Upstash Redis (custom-domain mapping + rate limits), Sentry (errors), Mapbox + Leaflet (transport routing), ClickView (catalog content CDN), Anthropic + OpenAI + Groq (AI features).

## Development approach

- 1-week sprint cadence.
- Conventional commits, signed.
- Code review with at least one approver before merge to `main`.
- 80%+ coverage target on critical paths.
- Performance budgets (Lighthouse ≥ 90).
- 20% of sprint capacity reserved for tech debt.

### Testing pyramid

| Layer | Share | Tool |
| ----- | ----- | ---- |
| Unit | 70% | Vitest |
| Integration | 20% | Vitest + Supertest |
| E2E | 10% | Playwright (auth, multi-tenant, RBAC, smoke) |

### Local DB workflow

Destructive Prisma operations (`db execute`, `db push --accept-data-loss`, `migrate reset`, `DROP TABLE`, `TRUNCATE`) are blocked by hooks. Use the **Branch-Before-Touch** protocol on Neon for any risky DB op. Never run `pnpm db:seed` directly — always `pnpm db:seed:single <name>`.

## How to contribute

1. Read [Get started](/docs/get-started) for the local setup.
2. Skim [Architecture](/docs/architecture) and [Pattern](/docs/pattern) for conventions.
3. Pick a Roadmap "Next" item or open a feature issue.
4. Follow the mirror pattern + the constraints above.
5. The Claude harness in `.claude/` (50 agents, 47 commands, 16 skills) is wired to enforce most of these rules automatically — `pnpm dev` plus the project's `CLAUDE.md` is enough to get going.

The OSS contribution model — including how time invested converts to recognition or revenue share — lives in [Shared economy](/docs/shared-economy).
