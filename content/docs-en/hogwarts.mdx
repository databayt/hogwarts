Hogwarts is an open-source, multi-tenant K-12 school platform. Every classroom in a school — admission, attendance, exams, fees, messaging, transport, library, LMS — fits inside one codebase, behind one subdomain per school, with one shared catalog underneath.

This page is the entry point. If you're new to the codebase, start here.

## What is it

A school operations platform built around three ideas:

1. **One subdomain per school.** Each tenant gets `school.databayt.org` (or a custom CNAME). Data is isolated by `schoolId` end-to-end.
2. **One shared catalog underneath.** Curricula, subjects, chapters, and lessons are global content. Schools subscribe and apply overrides — they don't duplicate.
3. **Arabic-first, RTL by default.** The default locale is Arabic; English is parity. The codebase enforces logical CSS so RTL works without overrides.

## Why open source

Hogwarts is built in public. The reasons are practical:

- **Schools deserve code they can audit.** Student data is sensitive; black-box vendors are a hazard.
- **Schools deserve software they can extend.** Curriculum and policy change; a closed platform is a cage.
- **Contributors deserve transparency.** Every contribution is publicly tracked; the [Shared economy](/docs/shared-economy) doc explains how that converts to recognition and value.

SSPL-1.0 keeps the code free for self-hosters and protects against parasitic SaaS resale. The pattern: **fork, fix, contribute back**.
