## Executive summary

Accelerators give us three things: capital, distribution, credibility. Two of three matter; one alone is not enough. We apply to programs where the network is asymmetrically useful for an Arabic-first, AI-integrated, MENA-anchored EdTech SaaS — and where the equity ask is < 10% (or zero).

## Selection criteria

Apply if **at least two of these are true**:

1. The program has a documented EdTech or B2B SaaS thesis.
2. The network reaches MENA buyers (schools, ministries, family offices, distributors).
3. Equity ask is < 10%, or program is zero-equity.
4. The program does not require relocation to a country we can't operate from.
5. Application is free (no pay-to-pitch).

If only one is true, the program goes to "watching" — not "applying."

## What we will NOT apply to

- **Pay-to-pitch programs** (any application fee > $0).
- **Equity > 10% at this stage**.
- **Programs requiring full relocation** to a country we can't operate from for visa or family reasons (e.g., Singapore-only programs).
- **Pivot-demanding accelerators** — if the program insists we abandon Arabic-first or open-source positioning, we walk.

## How we prepare per application

Each application has the same checklist; only the words change.

1. **Audience research** — read the program's recent portfolio. Identify the EdTech, MENA, or open-source angle that maps to their thesis.
2. **Adapt the deck** — port [Deck](/docs/deck) slides 7 (Traction) and 11 (Ask) to the program's filter (e.g., YC asks for traction in MRR terms; Mastercard asks for impact terms).
3. **Build the "why this program, why now"** paragraph — specific to the program, never copy-pasted across programs.
4. **Confirm references** — make sure pilot and demo are reachable from the program's IP region.
5. **Submit via T2** ([Outreach](/docs/outreach)) — cold program-manager email at the start, formal application via the portal.
6. **Follow-up cadence** — D+7, D+14 (T10, T11 in Outreach).
7. **Log status here** — update the table above and the [Data Room](/docs/data-room) once a status changes.
