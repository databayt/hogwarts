## Overview

Every step below is a concrete milestone where the platform delivers real, usable value to a school owner. They are ordered from first signup to full operation. Each builds on the previous.

| #   | Add Value          | Status | What It Is                                             | Prerequisites        |
| --- | ------------------ | ------ | ------------------------------------------------------ | -------------------- |
| 1   | School Online      | ~85%   | Live school URL + join code after onboarding           | Complete 15 steps    |
| 2   | Share & Invite     | ~50%   | Teachers and parents join via URL or join code         | School Online        |
| 3   | First Announcement | ~95%   | Post visible to all school members                     | School Online        |
| 4   | Team Onboard       | ~80%   | Add teachers manually or via CSV import                | School Online        |
| 5   | Students In        | ~85%   | Add students manually, CSV, or admission flow          | School Online        |
| 6   | Classes Formed     | ~85%   | Create classes linking teacher + subject + students    | Team + Students      |
| 7   | Timetable Running  | ~90%   | Generate weekly schedule for all classes               | Classes Formed       |
| 8   | Attendance Tracked | ~98%   | Mark daily attendance per class                        | Classes + Timetable  |
| 9   | Grades Recorded    | ~80%   | Create exams, enter marks, generate report cards       | Classes + Students   |
| 10  | Fees Collected     | ~75%   | Set up fee structure, record payments                  | Students In          |
| 11  | Parents Connected  | ~70%   | Parents linked to students, see grades and attendance  | Students + Guardians |
| 12  | School Automated   | ~50%   | Full cycle: attendance + grades + fees + communication | All above            |

## 1. School Online — ~85%

Complete [onboarding](/docs/onboarding) (15 steps) and the platform provisions a live subdomain.

### Progress

- All 15 onboarding steps with routes, forms, validation, and server actions
- School creation is atomic (`$transaction`)
- Subdomain provisioning works via middleware rewrite (`school.databayt.org`)
- `completeOnboarding()` marks school active and fires `provisionSchoolDefaults()`
- Full school dashboard available immediately at the subdomain

### Gaps

- Join code never auto-generated during onboarding — schools finish with `joinCode: null`
- No DNS provisioning (relies on wildcard DNS — works but no custom domain support)
- Maps API not integrated (location step collects data but no interactive map)
- Stripe not integrated (price/discount steps collect data only, no payment processing)
- Duplicate subdomain action files (`action.ts` + `actions.ts`) need cleanup

### Blockers

None critical for MVP — subdomain works via middleware rewrite.

### How to Test

1. Login with `user@databayt.org` / `1234`
2. Walk through all 15 onboarding steps
3. Verify congratulations page shows the school URL
4. Navigate to `yourschool.localhost:3000` and confirm the dashboard loads
5. After testing, reset: `pnpm db:reset-test-user`

### Dependencies

None — this is the entry point.

## 2. Share & Invite — ~50%

Once the school is online, members can join. Backend is solid but key UI pieces are missing.

### Progress

- Full membership management (~1089 lines in `actions.ts`): invite, approve, reject, suspend, role change, bulk ops, CSV export
- `InviteDialog` with role selection
- `MembershipRequest` model with status tracking

### Gaps

- No join code entry UI for users (backend exists, no frontend)

- No invitation token acceptance route (`/accept-invite?token=...`)

- No bulk invite UI (paste multiple emails)

### Blockers

- Join codes never generated during onboarding (from #1) — `joinCode` is `null`

### How to Test

3. Click "Invite Member" — dialog opens with role selection
4. Join code flow is not testable until code generation is implemented

### Dependencies

- **School Online (#1)** — school must exist with a subdomain

## Value Chain

Each add value builds on the previous. Here is the dependency graph:

```
School Online (1) .............. ~85%
├── Share & Invite (2) ......... ~50%  ⚠ missing join code UI
├── First Announcement (3) ..... ~95%  ✓ near-complete
├── Team Onboard (4) ........... ~80%  ⚠ no invite acceptance route
│   └── Classes Formed (6) ..... ~85%  ← Students In (5) ~85%
│       ├── Timetable Running (7) ~90%  ✓ MVP-ready
│       │   └── Attendance Tracked (8) ~98%  ✓ most complete
│       └── Grades Recorded (9) ~80%  ✗ report card bug
├── Students In (5) ............ ~85%  ⚠ guardian linking missing
│   ├── Fees Collected (10) .... ~75%  ⚠ no online payments
│   └── Parents Connected (11) . ~70%  ⚠ mock data + missing linking
└── School Automated (12) ...... ~50%  ✗ subsystems disconnected
```

## Critical Path to Fix

To reach full automation, these must be resolved in order:

1. **Auto-generate join codes** during onboarding (unblocks #2)
2. **Implement guardian linking** actions (unblocks #5, #11)
3. **Fix `termId` bug** in report card generation (unblocks #9, #12)
4. **Add invitation acceptance route** (`/accept-invite?token=...`) (unblocks #2, #4)
5. **Wire real data** to parent dashboard, replacing mock data (unblocks #11)
6. **Add event triggers** between subsystems (unblocks #12)

The fastest path to a fully operational school today: complete onboarding, import teachers and students via CSV, form classes, generate timetable. Four actions to reach step 7 — attendance and exams (minus report cards) work from there.
