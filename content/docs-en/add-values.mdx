## Overview

Every step below is a concrete milestone where the platform delivers real, usable value to a school owner. They are ordered from first signup to full operation. Each builds on the previous.

| #   | Add Value          | Status | What It Is                                             | Prerequisites        |
| --- | ------------------ | ------ | ------------------------------------------------------ | -------------------- |
| 1   | School Online      | ~93%   | Live school URL + join code after onboarding           | Complete 15 steps    |
| 2   | Share & Invite     | ~70%   | Teachers and parents join via URL or join code         | School Online        |
| 3   | First Announcement | ~97%   | Post visible to all school members                     | School Online        |
| 4   | Team Onboard       | ~88%   | Add teachers manually or via CSV import                | School Online        |
| 5   | Students In        | ~93%   | Add students manually, CSV, or admission flow          | School Online        |
| 6   | Classes Formed     | ~90%   | Create classes linking teacher + subject + students    | Team + Students      |
| 7   | Timetable Running  | ~95%   | Generate weekly schedule for all classes               | Classes Formed       |
| 8   | Attendance Tracked | ~95%   | Mark daily attendance per class                        | Classes + Timetable  |
| 9   | Grades Recorded    | ~90%   | Create exams, enter marks, generate report cards       | Classes + Students   |
| 10  | Fees Collected     | ~78%   | Set up fee structure, record payments                  | Students In          |
| 11  | Parents Connected  | ~85%   | Parents linked to students, see grades and attendance  | Students + Guardians |
| 12  | School Automated   | ~60%   | Full cycle: attendance + grades + fees + communication | All above            |

## 1. School Online — ~93%

Complete [onboarding](/docs/onboarding) (15 steps) and the platform provisions a live subdomain.

### Progress

- All 15 onboarding steps with routes, forms, validation, and server actions
- School creation is atomic (`$transaction`)
- Subdomain provisioning works via middleware rewrite (`school.databayt.org`)
- `completeOnboarding()` marks school active and fires `provisionSchoolDefaults()`
- Join code auto-generated during provisioning
- Location step uses Mapbox interactive map
- Full school dashboard available immediately at the subdomain

### Gaps

| Gap                                                                  | Effort | Impact     |
| -------------------------------------------------------------------- | ------ | ---------- |
| Duplicate subdomain files (`action.ts` + `actions.ts`)               | Quick  | Cleanup    |
| Stripe not integrated (price/discount steps store data only)         | Large  | Revenue    |
| DNS providers stubbed (custom domains scaffolded but not functional) | Large  | Enterprise |

### How to Test

1. Login with `user@databayt.org` / `1234`
2. Walk through all 15 onboarding steps
3. Verify congratulations page shows the school URL
4. Navigate to `yourschool.localhost:3000` and confirm the dashboard loads
5. After testing, reset: `pnpm db:reset-test-user`

## 2. Share & Invite — ~70%

Once the school is online, members can join. Backend is solid but key UI pieces are missing.

### Progress

- Full membership management (~1089 lines in `actions.ts`): invite, approve, reject, suspend, role change, bulk ops, CSV export
- `InviteDialog` with role selection
- `MembershipRequest` model with status tracking
- `joinSchoolByCode()` backend logic exists
- Invitation acceptance route (`/accept-invite?token=...`) with accept/decline flow
- Public school marketing pages exist (`school.databayt.org/` with about, academic, admissions, inquiry, tour)
- Newcomers multi-step modal
- `expire-invitations` cron job exists (not yet registered in vercel.json)

### Gaps

| Gap                                                                                | Effort | Impact          |
| ---------------------------------------------------------------------------------- | ------ | --------------- |
| No join code entry UI for users (backend `joinSchoolByCode()` exists, no frontend) | Quick  | Self-enrollment |
| No join code display/copy UI for admins (code generated but never shown)           | Quick  | Sharing         |
| No bulk invite UI (paste multiple emails)                                          | Medium | Efficiency      |
| Register `expire-invitations` cron in vercel.json                                  | Quick  | Cleanup         |

### How to Test

1. Login with `admin@databayt.org` / `1234` on `demo.localhost:3000`
2. Navigate to Membership > click "Invite Member"
3. Invited user receives email with accept link
4. Click accept link → `/accept-invite?token=...` → user joins school
