End-to-end testing uses [Playwright](https://playwright.dev/). Multi-environment, multi-tenant aware, and integrated with the Playwright MCP for browser-driven AI flows.

## Quick start

```bash
pnpm exec playwright install   # install browsers (one-time)
pnpm test:e2e                  # run all e2e tests against local dev (port 3000)

# against production
TEST_ENV=production pnpm test:e2e --project=production-chromium
```

## Commands

| Command                             | Use                                     |
| ----------------------------------- | --------------------------------------- |
| `pnpm test:e2e`                     | All e2e tests, default Chromium project |
| `pnpm test:e2e:ui`                  | Interactive UI runner                   |
| `pnpm test:e2e:debug`               | Debug with the Playwright inspector     |
| `pnpm test:e2e:report`              | View the last HTML report               |
| `pnpm test:e2e --project=chromium`  | Chromium only                           |
| `pnpm test:e2e --project=firefox`   | Firefox only                            |
| `pnpm test:e2e --headed`            | Show the browser window                 |
| `pnpm test:e2e tests/login.spec.ts` | Single file                             |

## Multi-tenant testing

### URL patterns

| Environment | Main domain                      | School subdomain                     |
| ----------- | -------------------------------- | ------------------------------------ |
| Local       | `http://localhost:3000/en/...`   | `http://demo.localhost:3000/en/...`  |
| Production  | `https://ed.databayt.org/en/...` | `https://demo.databayt.org/en/...`   |
| Preview     | `https://branch.vercel.app`      | `https://tenant---branch.vercel.app` |

### Scenarios

| Scenario            | Credential             | Domain                 | Expected                           |
| ------------------- | ---------------------- | ---------------------- | ---------------------------------- |
| SaaS Dashboard      | `dev@databayt.org`     | `localhost:3000`       | Access `/dashboard`, `/tenants`    |
| School Dashboard    | `admin@databayt.org`   | `demo.localhost:3000`  | Access to school platform          |
| Onboarding flow     | `user@databayt.org`    | `localhost:3000`       | "Get Started" → onboarding wizard  |
| RBAC                | `teacher@databayt.org` | `demo.localhost:3000`  | Blocked from `/admin`, `/settings` |
| Tenant isolation    | `admin@databayt.org`   | `other.localhost:3000` | Access denied (wrong school)       |
| Cross-subdomain SSO | any logged-in user     | main → subdomain       | Session persists                   |

### Example

```typescript
import { expect, test } from "@playwright/test"

test("admin can access school dashboard on subdomain", async ({ page }) => {
  await page.goto("http://demo.localhost:3000/en/login")
  await page.fill('[name="email"]', "admin@databayt.org")
  await page.fill('[name="password"]', "1234")
  await page.click('[type="submit"]')
  await page.waitForURL(/\/dashboard$/)
  await expect(page.getByRole("heading", { name: /dashboard/i })).toBeVisible()
})
```

## Configuration

The runner reads `playwright.config.ts`. Key levers:

- `TEST_ENV` — `local` (default) or `production`. Switches base URL and timeout.
- Per-project base URL: `local` uses `http://localhost:3000`, `production` uses `https://ed.databayt.org`.
- `webServer` boots `pnpm dev` for local runs. Skip it for production.
- Headless GPU acceleration enabled by default — ~30% faster than non-GPU headless.

## Browser modes

Default project is `chromium`; `firefox` is available for cross-engine spot checks. Pass `--headed` to watch the run.

For headless mode, the GPU-accelerated path uses Chromium with `--enable-gpu` and `--use-gl=angle`. Verify GPU acceleration with:

```ts
import { chromium } from "@playwright/test"

const b = await chromium.launch({ args: ["--enable-gpu", "--use-gl=angle"] })
const ctx = await b.newContext()
const p = await ctx.newPage()
await p.goto("chrome://gpu")
console.log(await p.textContent("body"))
```

## Writing tests

```typescript
import { expect, test } from "@playwright/test"

test("login redirects to dashboard", async ({ page }) => {
  await page.goto("/en/login")
  await page.fill('[name="email"]', "admin@databayt.org")
  await page.fill('[name="password"]', "1234")
  await page.click('[type="submit"]')
  await page.waitForURL(/\/dashboard$/)
})
```

For RBAC tests, log in as the role under test and assert that protected routes redirect or 403:

```typescript
test("teacher cannot access /admin", async ({ page }) => {
  await loginAs(page, "teacher@databayt.org")
  await page.goto("/en/admin")
  await expect(page).not.toHaveURL(/\/admin$/)
})
```

For shared auth state, save storage state once and reuse it across tests via `test.use({ storageState })`.

## Playwright MCP

Playwright MCP is the AI-assisted browser bridge. Configure once in your Claude Code (or other MCP-aware) client:

```json
{
  "mcpServers": {
    "playwright": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-playwright"]
    }
  }
}
```

The recommended local setup uses headless Chromium with GPU acceleration and a small persistent profile so cookies survive across calls. See `.claude/settings.json` and `playwright.config.ts` for the project's pinned config.

## Debugging

- `pnpm test:e2e:debug` — opens the Playwright Inspector.
- `npx playwright show-trace trace.zip` — replay a recorded trace.
- Failed tests automatically capture screenshot and video to `test-results/`.

## CI

GitHub Actions runs `pnpm test:e2e` on PRs against the local server. Production smoke runs `pnpm test:e2e:prod:smoke` against `ed.databayt.org` after deploys.
