Hogwarts automates the operational core of a school — admissions, classrooms, attendance, grading, fees, messaging — so staff spend less time on paperwork and more on teaching. Every line of code is open source, every contribution is attributed.

<StoryVideo />

## What it does

- **School operations.** Multi-tenant SaaS — one platform serves many schools. Each school gets its own subdomain, branding, roles, and data isolation.
- **Bilingual by default.** Arabic-first with full RTL support; English LTR mirror. UI labels and dynamic content both translate on demand.
- **AI-augmented.** Document intelligence for admissions, bank receipts, and (planned) exams, library, and onboarding.
- **Open source.** SSPL-1.0. Self-hostable today, managed SaaS for schools that prefer not to.

## Conventions

- **Single-language storage.** One `lang` field per row; generic field names (`title`, `body`, `name`); on-demand translation cached in `TranslationCache`.
- **`schoolId` scoping on every query.** Missing `schoolId` = data leak across tenants.
- **Semantic HTML.** No hardcoded `text-*` / `font-*` classes; logical CSS properties so RTL works without overrides — see [Style guide](/docs/documentation).

For full constraints and the role model see [PRD](/docs/prd).
