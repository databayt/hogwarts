## The Big Picture

The Catalog is the **universal shared backbone** of the platform. All curriculum content lives in the catalog once. Schools reference it via bridge tables. When a teacher creates content (quiz, video, book), it flows through SaaS review and gets indexed into the catalog -- enriching it for all schools.

```
+-------------------------------------------------------------+
|                    UNIVERSAL CATALOG                         |
|  (Global, no schoolId -- the single source of truth)        |
|                                                             |
|  CatalogSubject (~220 US K-12 + future: GB, IB...)         |
|    > CatalogChapter (~800 topic groups)                     |
|        > CatalogLesson (~4000 sub-topics)                   |
|                                                             |
|  Built-in at each level:                                    |
|  +-- Exams (CatalogExam)                                    |
|  +-- Questions (CatalogQuestion -- global QBank)            |
|  +-- Materials (CatalogMaterial -- books, syllabi, guides)  |
|  +-- Videos (LessonVideo -- multi-teacher LMS)              |
|  +-- Attachments (LessonAttachment -- PDFs, worksheets)     |
|  +-- Assignments (CatalogAssignment -- homework templates)  |
|  +-- Books (CatalogBook -- global library, /docs/books-catalog)|
|                                                             |
|  Images: exact ClickView thumbnails + fallback colors       |
+--------------------------+----------------------------------+
                           |
                   Bridge Layer (schoolId-scoped)
                           |
+--------------------------+----------------------------------+
|                    SCHOOL LAYER                              |
|  (schoolId-scoped -- per-school configuration)              |
|                                                             |
|  SchoolSubjectSelection > "we teach these catalog subjects" |
|  SchoolContentOverride  > "hide this chapter/lesson"        |
|  Teacher assignment, Classroom, Grade sections (A/B)        |
|  Timetable, Marks, Report cards                             |
|  Enrollment, Progress tracking, Certificates                |
|                                                             |
|  School-specific content (not in catalog):                  |
|  +-- Exam scheduling (date, time, room, proctoring)         |
|  +-- Mark entry & grade overrides                           |
|  +-- Student submissions                                    |
|  +-- School announcements, attendance, etc.                 |
+-------------------------------------------------------------+
```

### Why Shared Catalog (Not Per-School)

| Approach                       | Data for 1,000 schools x ~220 subjects                           | Content sharing                         |
| ------------------------------ | ---------------------------------------------------------------- | --------------------------------------- |
| Per-school (clone)             | 220,000 subject records, 800,000 chapters, 4,000,000 lessons     | None -- each school is an island        |
| **Shared catalog (reference)** | **~220 subjects, ~800 chapters, ~4000 lessons** + bridge records | Every contribution enriches all schools |

**Benefits:**

- **1,000x less data duplication** -- one set of subjects serves all schools
- **Community sharing** -- teacher uploads a great video, SaaS approves, all schools see it
- **Easy onboarding** -- new school auto-assigns subjects based on config, ready in seconds
- **Quality rises** -- engagement ranking surfaces the best content across all schools
- **Consistency** -- all schools reference the same curriculum structure

## Three Layers

### 1. Global Catalog (no schoolId)

```
CatalogSubject (~220 grade-specific subjects)
  > CatalogChapter (~800 chapters)
    > CatalogLesson (~4000 lessons)
```

These define the curriculum structure. No school owns them -- they belong to the platform. Each entity carries `country` and `system` fields for multi-country support.

### 2. Content Layer (user-contributed)

```
LessonVideo      -- video uploaded to a CatalogLesson
LessonAttachment -- PDF/file uploaded to a CatalogLesson
CatalogExam      -- exam at subject, chapter, or lesson level
CatalogQuestion  -- global question bank at any level
CatalogMaterial  -- books, syllabi, worksheets at any level
CatalogAssignment -- homework/project templates at any level
CatalogBook      -- global book metadata (see /docs/books-catalog)
```

Content is uploaded BY users (teachers, schools) TO catalog entities. SaaS admin approves, then content becomes visible based on its visibility tier.

### 3. School Bridge (schoolId-scoped)

```
SchoolSubjectSelection  -- "our school teaches this subject in grade 7"
SchoolBookSelection     -- "our school has this catalog book" (see /docs/books-catalog)
SchoolContentOverride   -- "hide this chapter for our school"
```

Bridge tables connect schools to the catalog without duplicating data.

## School Configuration and Auto-Assignment

Schools configure their curriculum profile in settings:

| Config Field  | On School Model            | Examples                                 | Purpose          |
| ------------- | -------------------------- | ---------------------------------------- | ---------------- |
| `country`     | Already exists (`String?`) | "US", "SD", "GB", "AE"                   | Country code     |
| `system`      | Already exists (`String?`) | "clickview", "national", "british", "ib" | Education system |
| `schoolLevel` | Already exists (`String?`) | "elementary", "middle", "high", "k12"    | Levels covered   |

### Auto-Assignment Flow (`setupCatalogForSchool()`)

1. Read school's `country` + `system` + `schoolLevel`
2. Query `CatalogSubject WHERE country AND system`
3. For each subject, check `subject.grades` array first (grade-specific subjects with explicit grade numbers)
4. Fall back to level-based mapping only if `grades` array is empty
5. Create `SchoolSubjectSelection` bridge records for all matching subjects mapped to the correct `AcademicGrade`
6. School dashboard immediately shows the right subjects

With grade-specific subjects (the new default), this produces a **1:1 mapping** between catalog subjects and academic grades -- "Math Grade 3" maps directly to the school's Grade 3. The old level-based subjects had N:1 mapping where "Elementary Math" would map to grades 1 through 6.

```typescript
import { setupCatalogForSchool } from "@/lib/catalog-setup"

// US K-12 school (ClickView curriculum) -- default
await setupCatalogForSchool(schoolId, { country: "US", system: "clickview" })

// International school in UAE (IB curriculum)
await setupCatalogForSchool(schoolId, { country: "AE", system: "ib" })

// British school
await setupCatalogForSchool(schoolId, { country: "GB", system: "british" })
```

**Example configurations:**

| School Type         | Config                                 | Subjects Auto-Assigned       |
| ------------------- | -------------------------------------- | ---------------------------- |
| US K-12 (demo)      | `country: "US"`, `system: "clickview"` | ~220 grade-specific subjects |
| International (UAE) | `country: "AE"`, `system: "ib"`        | IB curriculum subjects       |
| British school      | `country: "GB"`, `system: "british"`   | UK National Curriculum       |

The function defaults to `system: "clickview"` when no system is specified. Schools can also **browse** the full catalog and add subjects from other systems.

## ClickView K-12 Hierarchy (First Curriculum Set)

The first complete curriculum set replicates ClickView's exact US K-12 hierarchy, expanded into **grade-specific subjects**. Each of the original 62 level-based entries (e.g., "Elementary Math") is expanded into one subject per grade (e.g., "Math Grade 1", "Math Grade 2", ..., "Math Grade 6"), producing ~220 grade-specific subjects. Chapters and lessons are duplicated under each grade-specific subject. All subjects, chapters, and lessons include descriptions.

### Grade-Specific Subjects (~220 total)

| Level      | Original Entries | Grades | Grade-Specific Subjects | Chapters (approx) | Lessons (approx) |
| ---------- | ---------------- | ------ | ----------------------- | ----------------- | ---------------- |
| Elementary | 18               | 1-6    | ~108                    | ~294              | ~2,076           |
| Middle     | 21               | 7-9    | ~63                     | ~219              | ~828             |
| High       | 23               | 10-12  | ~69                     | ~237              | ~1,092           |
| **Total**  | **62**           |        | **~220**                | **~800**          | **~4,000**       |

All subjects use `country: "US"`, `system: "clickview"`. Each subject has `grades: [N]` (single grade number) instead of spanning multiple grades.

**Slug convention:** `us-{subject-slug}-grade-{N}` (e.g., `us-math-grade-3`, `us-chemistry-grade-11`)

**Old slug convention (deprecated):** `{level}-{subject-slug}` (e.g., `elementary-math`, `high-chemistry`). Old level-based subjects are automatically deprecated when the new grade-specific seed runs.

For per-subject breakdowns, see:

- [ClickView Overview](/docs/clickview) -- all 62 original entries in one table
- [Elementary](/docs/clickview-elementary) -- 18 subjects, 49 chapters, 346 lessons
- [Middle](/docs/clickview-middle) -- 21 subjects, 73 chapters, 276 lessons
- [High](/docs/clickview-high) -- 23 subjects, 79 chapters, 364 lessons

### Images -- Exact ClickView Match

Every Subject, Chapter, and Lesson gets the exact same image and fallback color as ClickView:

| Entity         | Image Source                                                  | Fallback                                             |
| -------------- | ------------------------------------------------------------- | ---------------------------------------------------- |
| Subject (~220) | ClickView hero banner (Banner ID)                             | `color` field (e.g., Math `#3b82f6`, Arts `#f43f5e`) |
| Chapter (~800) | First child lesson's thumbnail                                | Parent subject's `color`                             |
| Lesson (~4000) | ClickView cover image (Cover ID from `master-inventory.json`) | Parent subject's `color`                             |

**Pipeline:** Local `/public/clickview/` -> Sharp -> 4 WebP (sm/md/lg/original) -> S3 -> CloudFront

**URL resolution:** `thumbnailKey` (CDN) -> `imageKey` (direct path/URL) -> `color` (hex background)

## Community Contribution Flow

### Content Contribution to Review to Catalog

```
Teacher/School creates content
  > Saved with approvalStatus: PENDING, visibility: PRIVATE
    > SaaS admin reviews
      > APPROVED: indexed in catalog, visibility set by creator
        > PRIVATE: only creator's school sees it
        > SCHOOL: any school using that subject sees it (free)
        > PUBLIC: anyone sees it (free or paid)
      > REJECTED: feedback sent to creator
```

This pattern already exists for `LessonVideo` (has `approvalStatus`, `visibility`, `price`). The same pattern extends to:

- **CatalogQuestion** -- teacher submits a question, reviewed, available in global QBank
- **CatalogMaterial** -- teacher uploads a textbook/worksheet, reviewed, available to all
- **CatalogExam** -- teacher creates an exam template, reviewed, reusable by all schools
- **CatalogAssignment** -- teacher creates homework, reviewed, template for all

### Visibility Tiers

| Tier    | Who Sees It                          |
| ------- | ------------------------------------ |
| PRIVATE | Only uploader's school               |
| SCHOOL  | Any school using that subject (free) |
| PUBLIC  | Anyone (free or paid)                |

### Engagement Ranking

Content within the catalog is ranked by engagement, so the best rises to the top:

| Content Type | Ranking Signals                                              |
| ------------ | ------------------------------------------------------------ |
| Videos       | `viewCount`, `averageRating`, `isFeatured`                   |
| Questions    | `usageCount`, `averageScore`, `qualityScore` (psychometrics) |
| Materials    | `downloadCount`, `averageRating`                             |
| Exams        | `usageCount`, `averageScore`                                 |

Schools see: platform default content + their own + other schools' approved public content, ranked by engagement.

## Dual-Use Architecture

The same CatalogSubject hierarchy serves **both** the physical classroom path and the LMS/e-learning path. Key linking fields make this possible:

- **`CatalogBook.catalogSubjectId`** -- Links books in the global library to specific grade-level catalog subjects
- **`Subject.catalogSubjectId`** -- The school-scoped Subject model links back to its CatalogSubject origin
- **`SchoolBookSelection.gradeId`** -- Optional grade scope for book assignments within a school

### Physical Classroom Path

```
CatalogSubject --> SchoolSubjectSelection --> Subject (school-scoped) --> Class --> Timetable
CatalogSubject --> CatalogBook --> SchoolBookSelection --> Book (library)
CatalogSubject --> CatalogExam --> School Exam (scheduled)
CatalogSubject --> CatalogQuestion --> QuestionBank (school exams)
```

### LMS / E-Learning Path

```
CatalogSubject --> Enrollment --> LessonProgress --> SubjectCertificate
CatalogLesson --> LessonVideo (multi-teacher)
CatalogLesson --> LessonAttachment
CatalogLesson --> CatalogQuestion (quizzes)
```

### Why This Matters

A single grade-specific `CatalogSubject` (e.g., "Math Grade 3") feeds:

1. **Classroom scheduling** -- The school selects it via `SchoolSubjectSelection`, creates a school-scoped `Subject`, assigns it to classes and the timetable
2. **Library management** -- `CatalogBook` records link to the subject, schools pick books via `SchoolBookSelection`
3. **Exam generation** -- `CatalogExam` and `CatalogQuestion` records at subject/chapter/lesson level feed the school's exam system
4. **Online learning** -- Students enroll via `Enrollment`, watch `LessonVideo` content, track progress, earn certificates

No duplication between paths -- both reference the same catalog hierarchy.

### Grade Matching for Admission Placement

When admission places a student into a class, the free-text `applyingForClass` field (e.g., "Grade 5", "الصف الخامس", "5th") must be matched to an `AcademicGrade` record. The `extractGradeNumber()` utility in `src/lib/grade-utils.ts` handles this parsing, supporting Arabic ordinals, English ordinals, numeric values, and mixed formats. This connects admission (Layer 5b) to the catalog's grade-specific subject structure -- ensuring the student lands in the correct grade's classes and sees the right catalog content.

```prisma
model CatalogQuestion {

  difficulty       DifficultyLevel

  points           Decimal @default(1) @db.Decimal(5, 2)
  options          Json?
  sampleAnswer     String? @db.Text

  @@index([catalogChapterId])
  @@index([catalogLessonId])
```

### URL Resolution Priority

```typescript
getCatalogImageUrl(thumbnailKey, imageKey, size)
// 1. CloudFront CDN URL from thumbnailKey > "{CDN}/{thumbnailKey}-{size}.webp"
// 2. imageKey returned directly (local path or external URL)
//    - Local: "/clickview/illustrations/elementary-math.jpg"
//    - External: "https://img.clickviewapp.com/v2/covers/{id}?width=2048"
// 3. null (no image available -- UI falls back to color background)
```

### CatalogImage Component

`src/components/ui/catalog-image.tsx` -- a Next.js `Image` wrapper for catalog entities:

- Resolves URL from `thumbnailKey` (CDN WebP) or `imageKey` (local path / external URL)
- For CDN images, renders native `<img>` with responsive `srcSet` (200w/600w/1200w/3840w)
- For imageKey URLs, uses Next.js `<Image>` with `unoptimized` for external domains
- Shows a "No image" placeholder when no URL is available
- Size presets: sm (200px), md (600px), lg (1200px), original (1200px)

## Enrollment Flow

### Steps

1. User clicks "Enroll" on a catalog subject
2. `enrollInCatalogSubject` server action authenticates and checks stream permission
3. For paid subjects: Stripe checkout session created (stored as `stripeCheckoutSessionId`)
4. `Enrollment` record created or reactivated (with optional `schoolId`)
5. Enrollment confirmation email sent asynchronously
6. User redirected to `/stream/dashboard/{slug}`

### Auto-Sync from StudentClass

When a student is enrolled in a `Class` (for timetable/attendance), the `syncStudentClassToEnrollment()` function in `src/lib/enrollment-sync.ts` automatically creates a corresponding `Enrollment` record for LMS access. This bridges the two previously disconnected systems:

```
StudentClass (classroom scheduling)
    |
    | syncStudentClassToEnrollment()
    | follows: Class → Subject → catalogSubjectId → CatalogSubject
    v
Enrollment (LMS/catalog access, progress tracking)
```

Key behavior:

- **Non-blocking**: logs failures but never throws -- class enrollment succeeds even if LMS sync fails
- **Idempotent**: uses upsert on `[userId, catalogSubjectId]` -- safe to call multiple times
- **Skips gracefully**: if the class's subject has no `catalogSubjectId`, no enrollment is created
- **Wired into**: `placeStudentInClass()` (admission) and class enrollment actions

## Lesson Progress and Certificates

### Progress Tracking

- `LessonProgress` tracks per-lesson state: `watchedSeconds` for video resume, `isCompleted` flag, `watchCount`
- `lessonVideoId` tracks which specific video the user is watching
- Progress is linked to both the `Enrollment` and the `CatalogLesson`

### Completion Detection

When `markCatalogLessonComplete` is called:

1. Upsert `LessonProgress` with `isCompleted: true`
2. Query all published lessons in the subject
3. Check if all lessons have completed progress records
4. If complete: update `Enrollment.status` to `COMPLETED`
5. Generate `SubjectCertificate` with unique certificate number
6. Send completion email asynchronously


```
   - Middle entry (grades 7-9) --> 3 grade-specific subjects
   - High entry (grades 10-12) --> 3 grade-specific subjects

   - Each has grades: [N] (single grade number)
   - Each has a description
4. CatalogChapters (~800 -- duplicated under each grade-specific subject)
   - Each has a description and sequenceOrder
5. CatalogLessons (~4000 -- duplicated under each grade-specific chapter)
   - Each has a description, coverIds, and video/resource counts
6. Old level-based subjects (if any exist) are automatically deprecated
7. Images: local /public/clickview/ > Sharp > 4 WebP > S3/CDN
8. SchoolSubjectSelections for demo school
```
