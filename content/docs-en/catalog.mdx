The catalog is the single source of truth for curriculum content. All schools reference the same subjects via bridge tables; teacher-contributed content (videos, books, exams) flows through SaaS review and enriches the catalog for everyone.

## Why a shared catalog

| Approach           | Data for 1,000 schools Ă— ~240 subjects                         | Sharing                                 |
| ------------------ | -------------------------------------------------------------- | --------------------------------------- |
| Per-school clone   | 220k subjects, 800k chapters, 4M lessons                       | None                                    |
| **Shared catalog** | **~240 subjects, ~800 chapters, ~4k lessons** + bridge records | Every contribution enriches all schools |

Onboarding a school takes seconds â€” `setupCatalogForSchool()` reads `country` + `curriculum` and creates `SchoolSubjectSelection` bridge rows.

## Three layers

| Layer                 | Models                                                                                                                     | Scope                                      |
| --------------------- | -------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------ |
| Global catalog        | `CatalogSubject`, `CatalogChapter`, `CatalogLesson`                                                                        | No `schoolId`                              |
| Content (contributed) | `LessonVideo`, `LessonAttachment`, `CatalogExam`, `CatalogQuestion`, `CatalogMaterial`, `CatalogAssignment`, `CatalogBook` | Optional `schoolId` for source attribution |
| School bridge         | `SchoolSubjectSelection`, `SchoolBookSelection`, `SchoolContentOverride`                                                   | Carries `schoolId`                         |

## School configuration

| Field         | Type      | Examples                              |
| ------------- | --------- | ------------------------------------- |
| `country`     | `String?` | `US`, `SD`, `GB`, `AE`                |
| `curriculum`  | `String?` | `us-k12`, `national`, `british`, `ib` |
| `schoolLevel` | `String?` | `elementary`, `middle`, `high`, `k12` |

## Auto-assignment

`setupCatalogForSchool(schoolId, opts)` creates bridge rows via 4-step progressive fallback:

| Step | Filter                                 |
| ---- | -------------------------------------- |
| 1    | `country + curriculum + schoolType`    |
| 2    | `country + curriculum`                 |
| 3    | `country="*" + curriculum` (worldwide) |
| 4    | `US + us-k12` (baseline fallback)      |

Grade-specific subjects produce a 1:1 mapping â€” "Math Grade 3" maps directly to the school's Grade 3.

```ts
import { setupCatalogForSchool } from "@/lib/catalog-setup"

await setupCatalogForSchool(schoolId, { country: "US", curriculum: "us-k12" })
await setupCatalogForSchool(schoolId, { country: "AE", curriculum: "ib" })
await setupCatalogForSchool(schoolId, { country: "GB", curriculum: "british" })
```

```

  â†’ approvalStatus: PENDING, visibility: PRIVATE
SaaS admin reviews
  â†’ APPROVED â†’ indexed, visibility set by creator
```

## Data flows

**Operational** â€” drives timetabling, library, exams:

```
CatalogSubject â†’ SchoolSubjectSelection â†’ Subject â†’ Class â†’ Timetable
CatalogSubject â†’ CatalogBook â†’ SchoolBookSelection â†’ Book (library)
CatalogSubject â†’ CatalogExam â†’ School Exam
CatalogSubject â†’ CatalogQuestion â†’ QuestionBank
```

**LMS** â€” drives e-learning + certificates:

```
CatalogSubject â†’ Enrollment â†’ LessonProgress â†’ SubjectCertificate
CatalogLesson â†’ LessonVideo (multi-teacher)
CatalogLesson â†’ LessonAttachment / CatalogQuestion
```

The `syncStudentClassToEnrollment()` helper bridges classroom enrollment to LMS access via `Class.Subject.catalogSubjectId`. It's non-blocking, idempotent, and skips gracefully when the catalog FK is missing.

## Content models

### `CatalogMaterial`

Books, syllabi, worksheets at any catalog level.

| Field                                                            | Type                | Notes                                                     |
| ---------------------------------------------------------------- | ------------------- | --------------------------------------------------------- |
| `catalogSubjectId` / `catalogChapterId` / `catalogLessonId`      | `String?`           | One of these is set                                       |
| `title`                                                          | `String`            |                                                           |
| `lang`                                                           | `String`            | Default `"ar"`                                            |
| `type`                                                           | `MaterialType`      | `TEXTBOOK`, `SYLLABUS`, `WORKSHEET`, `PRESENTATION`, etc. |
| `fileUrl` / `fileKey` / `fileSize` / `mimeType` / `pageCount`    | mixed               | S3-backed                                                 |
| `externalUrl`                                                    | `String?`           | For non-uploaded resources                                |
| `tags`                                                           | `String[]`          | Searchable                                                |
| `contributedBy` / `contributedSchoolId`                          | `String?`           | Attribution                                               |
| `approvalStatus`                                                 | `ApprovalStatus`    | `PENDING`, `APPROVED`, `REJECTED`                         |
| `visibility`                                                     | `ContentVisibility` | `PRIVATE`, `SCHOOL`, `PUBLIC`, `PAID`                     |
| `usageCount` / `averageRating` / `ratingCount` / `downloadCount` | `Int` / `Float`     | Engagement                                                |

### `CatalogAssignment`

Reusable homework/project templates. Same shape as `CatalogMaterial` plus:

| Field                     | Type       | Notes                                                 |
| ------------------------- | ---------- | ----------------------------------------------------- |
| `instructions` / `rubric` | `String?`  | Long-form text                                        |
| `totalPoints`             | `Decimal?` |                                                       |
| `estimatedTime`           | `Int?`     | Minutes                                               |
| `assignmentType`          | `String?`  | `homework`, `project`, `lab`, `essay`, `presentation` |

### `CatalogExamQuestion`

Junction table linking `CatalogExam` â†” `CatalogQuestion` with ordering. Replaces the legacy `CatalogExam.questions` JSON field.

| Field                                | Notes                                |
| ------------------------------------ | ------------------------------------ |
| `catalogExamId`, `catalogQuestionId` | FKs (cascade delete)                 |
| `order`                              | Question order within exam           |
| `points`                             | `Decimal` â€” per-exam points override |

Indexes: `@@unique([catalogExamId, order])`, `@@unique([catalogExamId, catalogQuestionId])`.

### `LessonVideo`, `LessonAttachment`

Full schemas in `prisma/models/lesson-content.prisma`. Key fields on `LessonVideo`: `videoUrl`, `provider` (`youtube`/`vimeo`/`self-hosted`), `visibility`, `price`, `approvalStatus`, plus engagement (`viewCount`, `likeCount`, `averageRating`, `isFeatured`).

## Image pipeline

Server-side via Sharp, served from CloudFront CDN. Pure client/server split.

| Variant          | Width  | Quality  | Use             |
| ---------------- | ------ | -------- | --------------- |
| `-sm.webp`       | 200px  | 95       | Grid thumbnails |
| `-md.webp`       | 600px  | 97       | Card images     |
| `-lg.webp`       | 1200px | 100      | Hero banners    |
| `-original.webp` | Full   | Lossless | Full resolution |

| File                           | Environment | Purpose                                              |
| ------------------------------ | ----------- | ---------------------------------------------------- |
| `src/lib/catalog-image.ts`     | Server only | `processAndUploadCatalogImage`, `deleteCatalogImage` |
| `src/lib/catalog-image-url.ts` | Client-safe | `getCatalogImageUrl`, `getCatalogImageSrcSet`        |

URL resolution priority (unified system â€” `thumbnail`, `banner`, `cover`, `pdf` keys):

1. CloudFront CDN URL from `thumbnailKey` â†’ `{CDN}/{thumbnailKey}-{size}.webp`
2. Concept-based static fallback (shared concept thumbnails per [/docs/concept](/docs/concept))
3. `null` â€” UI falls back to color background

## Lesson progress and certificates

`LessonProgress` tracks per-lesson state: `watchedSeconds` (resume), `isCompleted`, `watchCount`, `lessonVideoId`. Linked to both `Enrollment` and `CatalogLesson`.

When `markCatalogLessonComplete` is called:

1. Upsert `LessonProgress` with `isCompleted: true`
2. Query all published lessons in the subject
3. If all complete â†’ set `Enrollment.status = COMPLETED`
4. Generate `SubjectCertificate` with unique number
5. Send completion email asynchronously

## Query patterns

```ts
// All subjects for a school + grade
await db.schoolSubjectSelection.findMany({
  where: { schoolId, gradeId, isActive: true },
  include: {
    catalogSubject: {
      include: {
        chapters: {
          where: { status: "PUBLISHED" },
          orderBy: { sequenceOrder: "asc" },
        },
      },
    },
  },
})

// Lesson with ranked videos
await db.catalogLesson.findUnique({
  where: { id: lessonId },
  include: {
    videos: {
      where: {
        approvalStatus: "APPROVED",
        OR: [{ visibility: "PUBLIC" }, { schoolId }, { isFeatured: true }],
      },
      orderBy: [
        { isFeatured: "desc" },
        { viewCount: "desc" },
        { averageRating: "desc" },
      ],
    },
  },
})

// Catalog questions for a chapter test
await db.catalogQuestion.findMany({
  where: {
    catalogChapterId: chapterId,
    approvalStatus: "APPROVED",
    visibility: "PUBLIC",
    status: "PUBLISHED",
  },
  orderBy: [{ qualityScore: "desc" }, { usageCount: "desc" }],
})
```

