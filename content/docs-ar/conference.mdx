كتلة واحدة قائمة بذاتها في `src/components/school-dashboard/conference/`، منعكسة بنسبة 1:1 على المسار
`/conference`. تشغّل ثلاثة خلفيات اجتماع خلف واجهة واحدة: مزوّد **الرابط الخارجي الملصوق** فعّال في كل
مكان؛ و**خادم LiveKit SFU** (غرف داخل التطبيق + تسجيل) مكتوب بالكامل لكنه خامل حتى تُجهَّز البنية
التحتية؛ ومحوّلات **Meet / Zoom / Teams الأصلية** تنشئ اجتماعات حقيقية عبر واجهة كل مزوّد لكنها تبقى
خاملة حتى تصل بيانات اعتماد OAuth.

## المكوّنات

تتعايش طبقتان للصلاحيات والتحقّق عن قصد: **طبقة الجلسات الغنية** (`authorization.ts` · `permissions.ts`
· `validation.ts`) هي البوابة الصارمة وقت التشغيل لتدفّق غرفة LiveKit (الانضمام / الرمز / البدء /
الإنهاء)، بينما **طبقة القائمة** (`list-*.ts`) هي بوابة الـ CRUD لجدول لوحة التحكّم وجلسات الروابط
الخارجية.

| الملف                                                                             | الغرض                                                                                                                                                   |
| --------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------- |

## تدفّق البيانات

```
index (/conference)            content.tsx (server) → table.tsx (client) → form.tsx (modal create)
   └─ row Join / detail
detail (/conference/[id])      detail.tsx → Join (live/scheduled) | Recordings (ended)
   └─ Join
room  (/conference/[id]/room)  joinLiveClass → eligibility → 5-min JWT → room.tsx (LiveKit prebuilt)
                                              ↑ refreshLiveClassToken every ~60s re-runs eligibility
recordings (/[id]/recordings)  recordings.tsx → recording-player.tsx (signed S3 URL, re-signed/4min)
```

- **آلة حالة الجلسة** (`actions/sessions.ts`): `scheduled → live → ended`، إضافةً إلى `cancelled`
  و`failed`. يجهّز `startLiveClass` غرفة الـ SFU (يتحقّق من `conferenceMaxConcurrent`)؛ و`endLiveClass`
  عديم التأثير عند التكرار. **الـ Webhook هو الكاتب الموثوق** لحالتي `live` (`room_started`) و`ended`
  (`room_finished`).
- **الأهلية** تُحَلّ بكسل عند كل انضمام (`actions/tokens.ts`): المعلّم→مضيف (لجلسته) / مضيف مساعد،
  الطالب→مشارك (مطابقة الشعبة)، وليّ الأمر→مراقب (ابنه في الشعبة)، المدير→مضيف مساعد، المطوّر→مضيف.
  المستخدمون المطرودون (`status="removed"`) يُحجبون عند تحديث الرمز التالي.
- **التسجيلات** (`pending → processing → ready → expired`): LiveKit Egress → S3 `me-central-1`؛
  الاحتفاظ لكل مدرسة تفرضه المهمة المجدولة اليومية.
- **الإشعارات** (`actions/notifications.ts`): جميع أحداث `live_class_*` الخمسة تحدّد جمهورها (المعلّم +
  طلاب الشعبة + أولياء الأمور) وتُرسَل عبر الموزّع المشترك (`dispatchNotificationsToAudience`)، فتصل إلى
  **جرس التطبيق والبريد الإلكتروني**، وتحترم تفضيلات القنوات لكل مستخدم، وتحمل رابطاً مطلقاً. يُرسِل كلا
  المسارين: غرفة التطبيق (`sessions.ts`) ومسار الرابط الخارجي (`list-actions.ts` إنشاء / تعديل / إلغاء).
- **الحضور** (`actions/attendance-sync.ts`، اختياري عبر `School.conferenceAttendanceSync`): عند انتهاء
  الجلسة (Webhook `room_finished` أو مهمة الجلسات المعلّقة)، يسجّل `syncConferenceAttendance` كل طالب في
  الشعبة حاضراً/متأخراً حسب وجوده في الغرفة، وغائباً لمن لم ينضمّ، بكتابة سجلات `Attendance` بالطريقة
  `VIRTUAL`. يتطلّب `sectionId` + `timetableId`؛ **LiveKit فقط** — الروابط الخارجية لا تحمل بيانات حضور.
  عديم التأثير عند التكرار (مفتاح الشعبة الفريد، إحياء عند التحديث).

| ----------------------------- | ---------------- | ---------------------------------------------------- | --------------------------------------------- |

تقع غرفة المسار تحت مجموعة **`(live-room)`** منفصلة بتخطيط عارٍ بملء الشاشة (بلا شريط جانبي / زخارف
لوحة التحكّم) كي يمتلك بثّ الـ SFU الشاشة؛ وهي `force-dynamic` لبناءات آمنة من نفاد الذاكرة. واجهات
API (مُستدعاة خارجياً / عبر المجدول → تبقى مسارات، لا إجراءات خادمية): `api/webhooks/livekit` (HMAC)،
`api/cron/live-class-reminders` (كل 5 دقائق)، `api/cron/end-stale-live-classes` (كل 15 دقيقة — يُنهي
الجلسات العالقة في حالة `live` بعد وقتها ويشغّل مزامنة الحضور)، `api/cron/expire-live-recordings` (يومياً).

## المخطّط — `prisma/models/conference.prisma`

`LiveClass*` → `Conference*`؛ مع الحفاظ على كل اسم جدول / عمود / نوع تعداد عبر `@@map`/`@map` →
**ترحيل بلا تغيير** (فرق `migrate diff` مطابق تماماً للأساس).

| النموذج                 | الغرض                                                                        |
| ----------------------- | ---------------------------------------------------------------------------- |
| `Conference`            | جلسة. `provider` (`livekit`\|`external`)، `meetingUrl?`، `meetingProvider?`. |
| `ConferenceParticipant` | صفّ لكل مستخدم مدعوّ + قياسات الاتصال (RTT، علامة الرجوع لـ TCP).            |
| `ConferenceRecording`   | بيانات الـ Egress وموقع S3 + `expiresAt` للاحتفاظ.                           |
| `ConferenceEvent`       | سجلّ تدقيق الـ Webhook + `eventId @unique` لضمان عدم التكرار.                |
| `ConferenceLink`        | رابط متكرّر يُضبط مرة واحدة `[schoolId, subjectId, sectionId, termId]`.      |

التعدادات `ConferenceProvider` · `ConferenceStatus` · `ConferenceRecordingStatus` ·
`ConferenceParticipantRole` · `ConferenceParticipantStatus`؛ و4 أعمدة إعداد `School.conference*`
(`conferenceRetentionDays` 90 · `conferenceMaxConcurrent` 50 · `conferenceMaxDuration` 120 ·
`conferenceRecordingDefault` true).

## المزوّدون

| المزوّد         | الحالة                         | الناقص                                                   |
| --------------- | ------------------------------ | -------------------------------------------------------- |
| الرابط الخارجي  | ✅ فعّال                       | — (الافتراضي العام بلا بنية تحتية)                       |
| LiveKit SFU     | 🟡 مكتوب، خامل                 | 6 بوابات بنية تحتية → `conference/RUNBOOK.md`            |
| Google Meet     | 🟡 `createMeeting` موصول، خامل | بيانات اعتماد OAuth من Google Cloud (`GOOGLE_MEET_*`)    |
| Zoom            | 🟡 `createMeeting` موصول، خامل | بيانات اعتماد Server-to-Server OAuth (`ZOOM_*`)          |
| Microsoft Teams | 🟡 `createMeeting` موصول، خامل | بيانات اعتماد Azure AD (`AZURE_*`، Graph OnlineMeetings) |

ينشئ كل محوّل أصلي اجتماعاً حقيقياً عبر واجهة المزوّد (`createMeeting`) محكوماً بـ `isConfigured()`؛
أما `getRecording`/`getAttendance` فما زالا يطلقان `ProviderNotImplementedError`. تُحفظ المزوّدات
الأصلية كـ `provider="external"` + `meetingProvider="<id>"` → **دون ترحيل تعداد**.

## القرارات

- **كتلة واحدة، تجريد واحد للمزوّد.** الرابط الخارجي هو الحدّ الأدنى المتاح لكل مدرسة؛ وLiveKit هو
  السقف المتميّز الاختياري، محكوم بـ `isLiveKitConfigured()`. تبقى دورة حياة SFU خارج طبقة
  `providers/` (لأنها قائمة على الغرف لا الروابط).
- **إعادة التسمية شفّافة لقاعدة البيانات.** `@@map`/`@map` تحفظ كل اسم جدول / عمود / نوع تعداد.
- **الـ Webhook والمهام المجدولة تبقى مسارات API** — مُستدعاة خارجياً / عبر المجدول؛ لا يستطيع الإجراء الخادمي استقبالها.
- **مدة الرمز = 5 دقائق، مُحدَّث من العميل.** رموز LiveKit عديمة الحالة، لذا يسري إلغاء الوصول عند حدّ
  التحديث التالي (≤5 دقائق) لا فوراً؛ ويُفرَض الطرد عبر بوابة `status="removed"` عند الانضمام.
- **قيد الانتظار (غير برمجي):** تجهيز بنية LiveKit التحتية (الدليل جاهز)، وبيانات اعتماد OAuth الأصلية.
