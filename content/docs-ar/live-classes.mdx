تتيح كتلة الفصول المباشرة للمعلم عقد فصل مباشر وللطالب الانضمام إليه **بزر واحد على جدوله الدراسي**. تعمل عبر **مزوّدَين متبادلين**:

- **رابط خارجي** — يلصق المعلم رابط Google Meet / Zoom / Teams (يُضبط مرة واحدة لكل مادة ويُعاد استخدامه، أو لكل حصة). يعمل على **جميع مستويات النشر اليوم** بلا أي بنية تحتية. وهو الخيار الافتراضي.
- **فيديو مدمج (LiveKit)** — خادم SFU مستضاف ذاتياً على G42 Cloud مع احتياطي TURN عبر TCP/443 وتسجيلات على AWS S3 `me-central-1` (احتفاظ وفق PDPL). مبني لتجربة **ألدار في الإمارات** ([المرحلة 03](https://kun.databayt.org/en/docs/aldar)). **جاهز للتطوير ومحجوب في الإنتاج** بناءً على وجود متغيرات `LIVEKIT_*` — يُفعَّل تلقائياً عند تجهيز الخادم دون أي تغيير في الشيفرة.

يتشارك المزوّدان نموذج بيانات واحداً ومجموعة مسارات واحدة ومصفوفة صلاحيات واحدة ونفس زر الانضمام في الجدول — المزوّد مجرد مُميِّز على الجلسة.

> **الحالة**: المزوّد الخارجي **مفعّل في الإنتاج**؛ مزوّد LiveKit **مكتمل الشيفرة وجاهز للتطوير** (بانتظار تجهيز الخادم + اختبار الشبكة). **181 اختبار وحدوي بـ Vitest** + **103 صف اختبار Playwright** ناجحة · المتتبع: [databayt/hogwarts#3](https://github.com/databayt/hogwarts/issues/3).

## زر الانضمام في الجدول

تربط `getTodaySchedule()` هدف انضمام بكل حصة في الجدول — جلسة مجدولة اليوم لتلك الشعبة والمادة، وإلا فالرابط الافتراضي الثابت. يظهر الزر (`LiveJoinButton`) على **بطاقة الحصة الحالية/التالية** في عرضَي الطالب والمعلم، ويظهر تلقائياً ضمن نافذة البث (من ~10 دقائق قبل البداية حتى نهاية الحصة، يُعاد حسابه كل 60 ثانية). الرابط الخارجي → يفتح رابط الاجتماع؛ LiveKit → يربط بالغرفة المدمجة.

## المسارات

تشحن الكتلة مجموعتين من المسارات — سطح لوحة التحكم للتصفح والجدولة، ومجموعة `(live-room)` منفصلة بتخطيط مجرّد حتى تستحوذ واجهة SFU على كامل الشاشة.

| المسار                          | التخطيط         | الأدوار المسموح بها                                       |
| ------------------------------- | --------------- | --------------------------------------------------------- |
| `/live-classes`                 | لوحة المدرسة    | DEV, ADMIN, TEACHER, STUDENT, GUARDIAN, STAFF, ACCOUNTANT |
| `/live-classes/[id]`            | لوحة المدرسة    | نفس ما سبق                                                |
| `/live-classes/[id]/recordings` | لوحة المدرسة    | DEV, ADMIN, TEACHER, STUDENT, GUARDIAN, STAFF             |
| `/live-classes/schedule`        | لوحة المدرسة    | DEV, ADMIN, TEACHER                                       |
| `/live-classes/network-test`    | لوحة المدرسة    | DEV, ADMIN                                                |
| `/live-classes/[id]/room`       | **(live-room)** | المشاركون فقط — ملء الشاشة بدون أي زخارف                  |

## دورة الحياة

```
مجدول ─المعلم يضغط بدء─▶ مباشر ─المعلم ينهي أو ينتهي الوقت─▶ منتهي
   │                              │
   ├─المعلم يلغي─▶ ملغى            └─انهيار/مهلة─▶ فاشل
```

تُفرض تحولات الحالة في `actions/sessions.ts` وترجع `LIVE_CLASS_INVALID_STATE` عند أي انتهاك. خطاف الويب الخاص بـ LiveKit (`/api/webhooks/livekit`) هو الكاتب الموثوق لحالتي `مباشر` (عند `room_started`) و`منتهي` (عند `room_finished`) لذا تفوز رؤية SFU على رؤية التطبيق.

## الأدوار داخل الغرفة

| الدور         | صلاحيات LiveKit                       | ملاحظات                                          |
| ------------- | ------------------------------------- | ------------------------------------------------ |
| `HOST`        | نشر + اشتراك + roomAdmin + roomRecord | المعلم المضيف. يبدأ/ينهي الغرفة، يطرد المشاركين. |
| `CO_HOST`     | نشر + اشتراك                          | معلم مساعد / مراقب صفي بصلاحيات نشر.             |
| `PARTICIPANT` | نشر + اشتراك                          | طالب — يمكنه طرح الأسئلة بميكروفون وكاميرا.      |
| `OBSERVER`    | اشتراك فقط                            | ولي أمر — للمشاهدة فقط، بدون رفع صوت أو فيديو.   |

يُحلّ الدور عند كل انضمام في `actions/tokens.ts:joinLiveClass` بمطابقة المستدعي مع `sectionId` للجلسة. أي شخص آخر يحصل على `LIVE_CLASS_PARTICIPANT_DENIED`. يجري فحص الأهلية عند كل إصدار رمز، لذا تأخذ التسجيلات المُلغاة مفعولها عند التحديث التالي (خلال 5 دقائق).

## نموذج البيانات

أربعة نماذج Prisma في `prisma/models/live-class.prisma`، كلها مُحدّدة النطاق بـ `schoolId` مع `@@index([schoolId, …])` وحذف ناعم عبر `deletedAt`.

| النموذج                | الغرض                                                                           |
| ---------------------- | ------------------------------------------------------------------------------- |
| `LiveClassSession`     | فصل مجدول أو فوري. روابط اختيارية لـ `timetableId` / `sectionId` / `subjectId`. |
| `LiveClassParticipant` | صف لكل مستخدم مدعو (`HOST` / `CO_HOST` / `PARTICIPANT` / `OBSERVER`).           |
| `LiveClassRecording`   | بيانات تسجيل Egress + موقع S3.                                                  |
| `LiveClassEvent`       | سجل تدقيق الخطاف + ضمان عدم التكرار (فريد على `eventId`).                       |

مع 4 أعمدة على `School` للإعدادات لكل مدرسة:

```prisma
liveClassRecordingRetentionDays  Int     @default(90)   // قابل للتهيئة وفق PDPL
liveClassMaxConcurrentPerSchool  Int     @default(50)
liveClassMaxDurationMinutes      Int     @default(120)
liveClassRecordingDefault        Boolean @default(true)
```

## أمان عزل المستأجرين

كل قراءة وكتابة تُصفّى بـ `schoolId` من `getTenantContext()` — وليس أبداً من مدخل العميل. اسم الغرفة نفسه يحوي المستأجر: `sch-{schoolId}-lc-{sessionId}`. هذا يعني أن مساحة أسماء غرف SFU لا يمكن أن تتصادم عبر المدارس، ويسترجع معالج الخطاف `schoolId` من اسم الغرفة وحدها عبر `parseRoomName()` فلا يستطيع حدث موجّه خطأً أن يكتب لمستأجر آخر.

## خط أنابيب التسجيل

تتدفق التسجيلات المركبة للغرفة عبر LiveKit Egress إلى AWS S3 `me-central-1`. يُملأ `s3Bucket` و`s3Region` من `getLiveKitConfig()` عند `egress_started` حتى يتمكن موقّع رابط التشغيل (`@aws-sdk/s3-request-presigner`، صلاحية 5 دقائق) من العمل بمجرد وصول `egress_ended`. مدة الاحتفاظ الافتراضية لكل مدرسة 90 يوماً؛ تُشغّل مهمة `/api/cron/expire-live-recordings` يومياً الساعة 02:30 وتحذف الكائن من S3 والصف معاً (حذف ناعم عبر `deletedAt`).

## الإشعارات

خمس قيم جديدة في `NotificationType` موصولة عبر نقاط المزامنة الخمس (config, validation, email-service typeLabels × لغتين, dictionaries × لغتين):

| النوع                        | متى                                    | الأولوية |
| ---------------------------- | -------------------------------------- | -------- |
| `live_class_scheduled`       | عند إنشاء الفصل                        | عادية    |
| `live_class_starting_soon`   | قبل البداية بـ 10 دقائق (cron)         | عالية    |
| `live_class_started`         | عند بث `room_started` من SFU (webhook) | عالية    |
| `live_class_cancelled`       | عند إلغاء المعلم                       | عادية    |
| `live_class_recording_ready` | عند بث `egress_ended` من SFU (webhook) | عادية    |

كل عمليات الإرسال هي «بذل أفضل ما يمكن» (استدعاءات `void` من الإجراءات ومعالج الخطاف) — لا يجوز أن يتسبب فشل الإشعار في التراجع عن تحول الحالة الأساسي.

## مرونة الشبكة

تقييد VoIP في الإمارات أعمق المخاطر في صفقة ألدار. تشحن الكتلة معالجتين:

1. **TURN عبر TCP/443** — يُستضاف coturn بجانب SFU على المنفذ TCP 443 (يبدو كأنه HTTPS لجدار حماية مقيّد). يعود عميل LiveKit تلقائياً عند فشل UDP.
2. **صفحة اختبار الشبكة** على `/live-classes/network-test` (DEV / ADMIN فقط) — تُجري اتصال معايرة على SFU باستخدام `livekit-client`، تلتقط زمن الإعداد وجودة الاتصال واستخدام TCP fallback في الأعمدة `LiveClassParticipant.{avgRttMs, hadTcpFallback, lastIceState}`.

**اختبار شبكة الاجتماع الثالث داخل المدرسة** هو أهم بوابة قبل التوقيع. شغّلها من داخل شبكة WiFi مدرسة ألدار بحساب `admin@kingfahad.databayt.org`؛ توقّف إن فشل TURN/443.

## القرارات

- **استضافة SFU**: مستضاف ذاتياً على G42 Cloud في منطقة الإمارات منذ اليوم الأول. تفضيل ألدار المعلن؛ موقف PDPL؛ مسار شبكي واحد يُتحقق منه عبر اختبار الاجتماع الثالث. LiveKit Cloud (البحرين) هو خيار التعافي الموثّق.
- **تخزين التسجيل**: AWS S3 `me-central-1` (PDPL عبر المنطقة، لا عبر المباني). خيار النجاة إلى MinIO على الموقع هو تبديل إعداد لكل مدرسة — يحمل المخطط مسبقاً `s3Bucket`/`s3Region` على كل صف `LiveClassRecording`.
- **TTL الرمز**: 5 دقائق مع تحديث من العميل قبل الانتهاء بنحو 60 ثانية. يعيد فحص الأهلية مع كل تحديث، لذا تأخذ صلاحيات الوصول المُلغاة مفعولها فوراً.
