تُسجِّل وثيقة [MVP](/ar/docs/mvp) ما هو مشحون اليوم عبر 14 ملحمة. خارطة الطريق هذه تُسجِّل ما هو قادم. المواضيع هنا مرتبطة بفروع وقضايا ومراجعات حقيقية قيد التنفيذ — وليست قائمة أمنيات. التاريخ المرجعي: 2026-05-10.

للمتطلبات والقيود راجع [PRD](/ar/docs/prd). للتوجه التجاري راجع [Business model](/en/docs/business-model).

## المبادرات النشطة

هذه التغييرات الهيكلية تُشكِّل الفصلين القادمين. كل واحدة منها مراجعة شاملة أو إعادة هيكلة من البداية للنهاية — وليست تذكرة واحدة.

| المبادرة                        | الملخص                                                                                                                                                                                                                             |
| ------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| مظلة المدفوعات (#263)           | مراجعة مالية شاملة + خارطة طريق مرحلية P0/P1/P2/P3. اليوم Stripe فقط يعمل E2E؛ Tap عبارة عن stub؛ محرك القيد المزدوج بلا مستدعين. راجع [Fees](/en/docs/fees).                                                                      |
| التزويد التلقائي للرسوم (#264)  | مزامنة هجينة على غرار `catalog/setup.ts` بحيث لا ترى المدارس الجديدة صفحة رسوم فارغة أبداً. تحلّ محل نهج سكربت البذر في P0-3 ضمن #263.                                                                                             |
| مراجعة إنتاج بلوك الامتحانات    | 14 P0 / 22 P1 / 18 P2 عبر بلوك الامتحانات. أبرز P0: بق التسليم بسبب استخدام User.id كـStudent.id، rate limiter في الذاكرة على serverless، 17 نموذجاً تنقصه `lang`، تغطية اختبار 0% لـRBAC + الحاسبة. راجع [Exams](/en/docs/exams). |
| تدقيق الترجمة                   | تدقيق منهجي بـPlaywright MCP لصفحات لوحة المدرسة بحثاً عن الإنجليزية المضمَّنة. الإصلاح صفحة بصفحة قبل الانتقال. راجع [Translation](/en/docs/translation).                                                                         |
| المسح الشامل لـUI واعٍ بالأدوار | تقييد الـSidebar / PageNav / Toolbar / إجراءات الصف عبر لوحة المدرسة. القوائم موصولة بـ`permissions.ts` + UIPermissions؛ المالية + المدرسة + المبيعات معلقة.                                                                       |
| البنية القائمة على الأقسام      | تم النشر وتم ترحيل قاعدة البيانات. تحتاج اختبار Playwright MCP على سطوح الحضور + الجدول المرتكزة على القسم. راجع [Attendance](/en/docs/attendance).                                                                                |
| دمج WhatsApp في Messages (#164) | نقل إعدادات WhatsApp إلى حوار إعدادات Messages وجسر الرسائل تلقائياً إلى WhatsApp.                                                                                                                                                 |
| النقل الجاهز للإنتاج            | شُحن في 2026-05. العمل المتبقي تصليب: دقة geofence أكثر إحكاماً، دقة معاينة الرسوم، نوافذ إشعارات تطبيق ولي الأمر. راجع [transportation block](/ar/docs/structure).                                                                |

## ليست على خارطة الطريق (هذا العام)

بعض الأمور تبدو مجاورة لكنها مؤجلة بشكل صريح. ندرجها هنا حتى لا يبدأ المساهمون عملاً تخمينياً.

- **إعادة هيكلة تقسيم الكتالوج.** تقسيم `catalog/setup.ts` إلى تكوينات لكل منهج موقوف حتى تنضم مدرسة غير سعودية. جدول DB واحد كافٍ في الوقت الحالي.
- **API الموبايل خارج المسارات الحالية.** المسارات القائمة (auth + الصعود بـgeofence للنقل) كافية. لا سطح API أوسع موجَّه للموبايل أولاً هذا العام.
- **خطط السوق.** لا سوق للمعلمين / المحتوى حتى تُشير إشارات تبنّي الكتالوج عبر المدارس إلى طلب حقيقي.
- **وحدات السكن والكافيتيريا.** ذكر في القاموس فقط؛ بلا schema، بلا UI، بلا قيد على خارطة الطريق.
- **تطبيقات iOS / Android أصلية.** mobile web يعمل اليوم. مسار React Native لا يُفتح إلا بعد وضوح PMF.

## كيف تُتَّخذ القرارات

- **المتطلبات:** يُعرِّف [PRD](/ar/docs/prd) النطاق وNFRs والقيود. أي عمل جديد يُثبت ملاءمته لها قبل إضافته هنا.
- **الحالة المشحونة:** [MVP](/ar/docs/mvp) هو مصدر الحقيقة لما يعمل اليوم. إذا تعارض مع هذه الوثيقة، يُرجَّح MVP لـ«ما هو موجود» وخارطة الطريق لـ«ما هو قادم».
- **التوجه التجاري:** [Business model](/en/docs/business-model) يُشكِّل أولوية المبادرات — المالية والقبول تُشحن أولاً لأنها مرتبطة بالإيرادات.
- **البنية:** [Architecture](/ar/docs/architecture) هي وثيقة التصميم القانونية. المبادرات التي تنتهكها (مثل إضعاف عزل multi-tenant، أعمدة ثنائية اللغة، كسر نمط المرآة) تُرفض.
- **القضايا قبل الدردشة.** كل ما على هذه الصفحة مُتتبَّع أيضاً في GitHub issues. إذا لم يكن لبند خارطة الطريق قضية، فهو مسوّدة ولا يجب التقاطه بعد.
