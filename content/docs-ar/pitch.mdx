Hogwarts نظام تشغيل مدرسي مفتوح المصدر. نؤتمت العمليات الأساسية للمدارس — القبول، الفصول، الحضور، التقييم، الرسوم، الاتصال — حتى يقضي المعلمون وقتاً أقل في الأعمال الورقية ووقتاً أكثر في التدريس.

للتسعير والمستويات راجع [نموذج العمل](/ar/docs/business-model). للعملية التجارية راجع [المبيعات](/en/docs/sales).

## ما نقدّمه

| القدرة                          | الناتج                                                  |
| ------------------------------- | ------------------------------------------------------- |
| SaaS متعدد المستأجرين           | منصة واحدة تخدم عدة مدارس بعزل عبر `schoolId`           |
| تجربة عربية أولاً مع RTL كامل   | مبنية لمنطقة الشرق الأوسط وأفريقيا، لا مُلصقة           |
| عمليات بمساعدة الذكاء الاصطناعي | مراجعة المستندات، ترتيب الجدارة، استخراج إيصالات البنوك |
| مفتوحة المصدر بترخيص SSPL-1.0   | استضافة ذاتية مجانية، أو SaaS مُدارة                    |

## المشكلة التي نحلّها

| الألم                             | المعدّل في الصناعة         |
| --------------------------------- | -------------------------- |
| وقت إداري ضائع في الأعمال الورقية | 40+ ساعة/شهر لكل إداري     |
| تكلفة الأدوات المتفرقة            | $15K–$50K سنوياً لكل مدرسة |
| نسبة الخطأ في الإدخال اليدوي      | ~23%                       |
| استياء أولياء الأمور من التواصل   | ~65%                       |
| ارتباط المدارس بمزوّدين مغلقين    | الافتراض السائد            |

## أثرنا

| الناتج                               | الهدف                         |
| ------------------------------------ | ----------------------------- |
| تخفيض الوقت الإداري                  | 80%                           |
| توفير التكلفة مقابل الأنظمة المتفرقة | 60%                           |
| كود مفتوح، مساهمات موثّقة            | 100%                          |
| نسبة الخطأ بعد الإطلاق               | < 1%                          |
| الوقت الإداري للمعلم                 | 11 ساعة/أسبوع → 4 ساعات/أسبوع |

## القدرات الأساسية

- **أكاديمية** — معلومات الطلاب، إدارة المعلمين والكادر، الجدولة، الحضور، التقييم، شهادات التقدير.
- **مالية** — إدارة الرسوم، محاسبة مزدوجة القيد، الفوترة، الرواتب، البنوك.
- **اتصال** — بوابة ولي الأمر، الإعلانات، المراسلة داخل التطبيق، جسر WhatsApp Evolution API.
- **تحليلات** — لوحات تحكم خاصة بالأدوار، اتجاهات الحضور، تدخّلات الإنذار المبكر، التقارير المالية.

راجع [MVP](/ar/docs/mvp) للحالة الكاملة لكل ملحمة و[Roadmap](/ar/docs/roadmap) للتوجه.

## مسارات الشراكة

نقدّم سبع طرق للعمل معنا. كلها تخدم الرسالة المشتركة: التميّز التعليمي والبنية المستدامة.

| المسار               | شكل المشاركة                               |
| -------------------- | ------------------------------------------ |
| التزكية              | دعم علني ومناصرة من مؤسستك                 |
| برنامج تجريبي        | تطبيق في مدارس مختارة ضمن شبكتك            |
| تعاون استراتيجي      | تخصيص لاحتياجات منطقة أو قطاع              |
| تدريب وبناء قدرات    | برامج للإداريين والمعلمين والكوادر التقنية |
| مساهمة مفتوحة المصدر | كود، وثائق، ترجمة، تصميم من مجتمعك         |
| تطوير مشترك          | ميزات جديدة لاحتياجات خاصة (مهني، ريفي، …) |
| تمويل                | تطوير منصة، نشر إقليمي، مبادرات محددة      |

## لماذا مفتوح المصدر

- **المدارس تستحق كوداً قابلاً للتدقيق.** بيانات الطلاب حساسة، والمزوّدون المغلقون مخاطرة.
- **المدارس تستحق برمجيات قابلة للتوسعة.** المناهج والسياسات تتغيّر، والمنصات المغلقة قيود.
- **المساهمون يستحقون الشفافية.** كل مساهمة موثّقة علنياً وتتحوّل إلى تقدير أو حصة من الإيراد.

ترخيص SSPL-1.0 يبقي الكود حرّاً للمستضيفين الذاتيين ويحمي من إعادة بيع المنصة كخدمة طفيلية. النمط: **افعل fork، أصلح، ساهم بالعودة**.

## الخطوات التالية

1. **عرض حي** — جولة 30 دقيقة على المنصة الحية.
2. **برنامج تجريبي** — تجربة 14 يوماً أو تجربة فصل لـ3 شهور.
3. **شراكة** — نقاش نموذج تعاون يناسب رسالتك.
4. **تصميم مشترك** — رسم خارطة الطريق وفق احتياجات مجتمعك.

نعمل مع الجهات الحكومية والمنظمات غير الربحية والمؤسسات والمنظمات التنموية والمبادرات المجتمعية والمؤسسات الدولية. إن شاركتَنا القيم، فثمّة مسار.
