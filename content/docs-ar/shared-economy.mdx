**العدالة = وصول عادل للموارد بناءً على دليل المساهمة.** كل أشكال المساهمة — كود، وثائق، تصميم، مجتمع — تُقدَّر وتُتتبَّع. المساهمات غير قابلة للتعديل، والملكية تنتقل عبر تغيّرات القيادة.

## المبادئ الأساسية

| المبدأ            | المعنى                                       |
| ----------------- | -------------------------------------------- |
| قيمة قابلة للقياس | لكل مساهمة أثر قابل للقياس                   |
| احترام متساوٍ     | الكود والوثائق والعمل المجتمعي بقيمة متساوية |
| مقاومة التلاعب    | النظام يكتشف ويمنع التلاعب                   |
| تقييمات شفّافة    | كل الصيغ علنية، وكل النتائج قابلة للطعن      |
| ملكية ثابتة       | المساهمون يملكون قيمتهم للأبد                |

## مصفوفة المساهمات

### تقني (وزن 40%)

| الفئة                | النقاط                                        |
| -------------------- | --------------------------------------------- |
| أسطر الكود           | LOC × وزن اللغة (TS 1.0، CSS 0.3، Config 0.1) |
| مضاعف التعقيد        | بسيط 1× → معقد 2.5×                           |
| ميزة جديدة           | 10 × التعقيد                                  |
| إصلاح خطأ            | 5 × الخطورة (حرج 5×، طفيف 1×)                 |
| Refactoring          | 3 × الملفات المتأثرة                          |
| تغطية الاختبارات     | 2 × زيادة % التغطية                           |
| PRD / وثائق تقنية    | 50 × النطاق (ميزة 1×، وحدة 3×، نظام 10×)      |
| قرارات معمارية (ADR) | 100                                           |
| تحسين أداء           | 20 × % التحسين                                |
| مراجعة كود           | 5 لكل تعليق جوهري                             |
| إرشاد في المراجعة    | 10 إن علّم المراجِع                           |
| اكتشاف أخطاء حرجة    | 50 لكل خطأ أمن / فقدان بيانات                 |

**مضاعفات الجودة** (3.9× كحد أقصى مجتمعة): الاختبارات تنجح من أول مرة (1.2×) · صفر iterations مراجعة (1.3×) · يتضمّن اختبارات (1.5×) · يتضمّن وثائق (1.4×) · صفر regressions في 30 يوماً (1.5×).

## مكافحة التلاعب

| الآلية           | القاعدة                                          |
| ---------------- | ------------------------------------------------ |
| منع Sybil        | حساب GitHub > 6 أشهر، بريد متحقّق                |
| حدّ السرعة       | 100 commit مراجَع/يوم كحد أقصى                   |
| تحقّق الأقران    | المساهمات الكبيرة تحتاج مراجعتين+ من maintainers |
| تجزئة الـcommits | إن أمكن دمج 10+ في 1، يطبَّق مضاعف 0.5×          |
| ضريبة churn      | كود يُحذَف خلال 30 يوماً = −50% رجعي             |
| جودة الاختبارات  | اختبارات تُحاكي كل شيء = 0 نقطة                  |
| اكتشاف التواطؤ   | تداخل مراجعات 80%+ → مراجعة بشرية                |
| تأييد متبادل     | A↔B مراجعات متبادلة = مضاعف 0.7×                 |

## صيغة التقييم

أجر ساعي: **45$ USD/ساعة كحدّ أدنى** (معيار COCOMO 2024).

```
قيمة المساهمة = (نقاط أساس × مضاعف جودة × عامل أثر) / ساعات الجهد

إن كانت < 45$/ساعة → القيمة المعدَّلة = 45$ × ساعات الجهد
```

| تقدير الجهد | الطريقة                            |
| ----------- | ---------------------------------- |
| ذاتي        | نظام شرف + تحقّق أقران             |
| تلقائي      | طوابع زمنية commits عبر GitHub API |
| كود         | صيغة COCOMO II                     |
| استثناءات   | ادعاء 100 سطر بـ20 ساعة → علامة    |

### تنمية المهارات

- **1.5×** لأول مساهمة في مجال جديد.
- **+20%** المرشد يحصل على 20% من نقاط المتدرّب.
- **2×** لأدلّة نقل المعرفة في مجالات معقّدة.

## مشاركة الإيراد

لا رواتب تقليدية. Databayt مملوكة جماعياً للمساهمين. عند ورود مشاريع، الجميع يأخذ حصته بحسب المساهمة.

| المسار              | المصدر                            | الدخل                                              |
| ------------------- | --------------------------------- | -------------------------------------------------- |
| لكل مشروع           | حلول مخصّصة لعملاء                | حصة مباشرة بحسب مساهمة المشروع                     |
| شراكات              | عقود مُسنَدة                      | حصة بحسب العقود التي تنفّذها                       |
| إعادة استخدام الكود | مكونات مُعاد استخدامها عبر مشاريع | عند إعادة استخدام مكوناتك المفهرَسة (الأعلى دخلاً) |
| منتجات SaaS         | منتجات جاهزة للنشر                | عوائد مستمرة مع توسّع المنتج                       |

### الجدول الزمني المتوقع

| الفترة     | النتيجة                                              |
| ---------- | ---------------------------------------------------- |
| الشهر 1–3  | تعلّم الأنظمة، المساهمة، الحصص الأولى                |
| الشهر 6–12 | بناء مكونات قابلة لإعادة الاستخدام، الانضمام لشراكات |
| السنة 2+   | مساهم رئيسي مع SaaS، إيراد متكرّر                    |

## الحوكمة

### تطوّر الصيغة

1. اقتراح تغيير عبر RFC.
2. محاكاة على بيانات تاريخية، عرض الأثر.
3. فترة نقاش علنية: 30 يوماً.
4. تصويت: مرجَّح بالـtoken بحسب تاريخ المساهمة.
5. أغلبية مطلوبة: 67% لتغيير التقييمات الأساسية.
6. طرح تدريجي: اختبار 10% لشهر أولاً.

### الحماية من الاستحواذ

| الإجراء      | التفصيل                                |
| ------------ | -------------------------------------- |
| سقف التصويت  | لا مساهم > 10% قوة تصويت               |
| تلاشي السلطة | فريق التأسيس −10%/سنة                  |
| فرملة طوارئ  | 100+ اعتراض يوقف تغييراً للمراجعة      |
| تحكيم        | وسيط خارجي للنزاعات الكبرى (DAO-style) |

## التاريخ غير القابل للتغيير

| الميزة           | التفصيل                                             |
| ---------------- | --------------------------------------------------- |
| سجلات دائمة      | لا يمكن محو المساهمات أو تخفيض قيمتها               |
| إثبات لا مركزي   | تغيّرات القيادة لا تؤثر على حقوق المساهمين          |
| Smart contracts  | توزيع أرباح آلي                                     |
| Git timestamping | commit → hash → Ethereum / Polygon (< $0.01/commit) |
| Contribution NFT | المساهمات الكبرى تحصل على NFT بـmetadata            |
| سمعة محمولة      | إثبات القيمة عبر مشاريع                             |

## اكتشاف التحيّز

- تتبّع منهجي لأنواع المساهمين المُبخَسين.
- تدقيق ربعي بحسب النوع / الجغرافيا / الخبرة.
- تصحيح رجعي + تغيير سياسة.

## تقييمات قابلة للطعن

1. مساهم يفتح issue: "Commit X مُبخَس لأن Y".
2. وسم تلقائي `valuation-review`.
3. ثلاثة maintainers يراجعون خلال 7 أيام.
4. تصويت: رفع / خفض / لا تغيير.
5. النتيجة تُسجَّل علنياً مع الأسباب.

## النسب الفكري والمراجع

هذه الصفحة هي **طبقة القياس** — كيف تُقيَّم المساهمة، آليات منع التلاعب، والحوكمة. أما **الطبقة العقدية** — ما نعنيه بـ"اقتصاد المشاركة"، والنسب الفكري، والمفردات العربية الإسلامية الدقيقة — فتعيش في مجلد استراتيجية kun عند [`SHARE-ECONOMY-MODEL.md`](https://github.com/databayt/kun/blob/main/docs/SHARE-ECONOMY-MODEL.md). و**طبقة المواصفات التشغيلية** — تحجيم تجمّع CU، احتياطات المؤسس/التشغيل، مخطط CU متعدد الفئات، آليات التحويل إلى نقد، نموذج عقد المضاربة، شروط تفعيل صندوق الوقف — يصاغها فريق البحث والتطوير (سامية) للربع الثالث 2026 وستعيش في مستودع `databayt/revenue` (لم يُنشَأ بعد).

المراجع الأربعة التي تؤسس العقيدة:

- **د. محمد حسام خضر** (2019). _رائد الأعمال Inside Out._ دار دون. ISBN 9789778061598. — منهجية توزيع الحصص في السياق العربي-الإسلامي.
- **د. جميل عبد القادر أكبر** (قيد الإنجاز). _قص الحق._ [en.jamelakbar.com](https://en.jamelakbar.com/) · ندوة Ummatics 2024. — الحقوق (الوصول القائم على الحق) كبديل للملكية القائمة على الإقصاء.
- **الحركة الوطنية للبناء والتنمية**. _ورقة الاقتصاد التشاركي._ [nmbdsd.org](https://www.nmbdsd.org/). — _"التضامن في الإنتاج، لا في الاستهلاك فقط."_
- **مفاهيم الاقتصاد الإسلامي** — المضاربة، شِركة الأعمال، الوقف، صندوق العون (الزكاة كمرجع). كتاب مرجعي: إقبال وميرَخور، _An Introduction to Islamic Finance_ (Wiley, 2011); معايير: [AAOIFI](https://aaoifi.com/).

إطار CU في هذه الصفحة يُعبّر عن مبدأ **شِركة الأعمال** (شراكة العمل): المساهمون مالكون بقدر المساهمة المُقاسة، لا بالعضوية. اختلاف صادق عن البنية الكلاسيكية: هذا النموذج **يضع سقفاً للخسارة على المساهم** — الخسائر التشغيلية يتحملها احتياطي التشغيل، ولا تُعاد توزيعها بالتناسب على المساهمين.
