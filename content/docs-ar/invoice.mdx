تُدير وحدة الفاتورة الفوترة من المسودّة إلى الدفع. تدفّقا إنشاء (modal من 3 خطوات + wizard متعدّد الصفحات مع حفظ المسودّات)، توليد بالجملة من هياكل الرسوم، تسليم بالبريد عبر Resend، ولوحة تحليلات. كل العمليات تفرض `getTenantContext()` + `checkCurrentUserPermission()`.

## Server actions

كلها في `actions.ts`، محميّة بـ`requireAuthAndTenant()`:

```typescript
async function requireAuthAndTenant() {
  const session = await auth()
  if (!session?.user?.id) return { success: false, error: "Not authenticated" }
  const { schoolId } = await getTenantContext()
  if (!schoolId) return { success: false, error: "Missing school context" }
  return { userId: session.user.id, schoolId }
}
```

## تدفّقات الإنشاء

### Modal (3 خطوات)

```
الخطوة 1 — معلومات أساسية: من (مُعبّأ مسبقاً)، إلى (اسم العميل، البريد، العنوان)
الخطوة 2 — العميل والبنود: رقم الفاتورة (تلقائي: I{YY}{NNN})، التواريخ، العملة، البنود
الخطوة 3 — مراجعة: ملخّص، subtotal / discount / tax / total، ملاحظات
```

تحقّق لكل خطوة عبر `form.trigger(STEP_FIELDS[step])` مقابل `InvoiceSchemaZod`.

### Wizard (متعدّد الصفحات)

```
/add/[id]/details  →  /add/[id]/items  →  Complete
```

تُحفظ المسودّات في قاعدة البيانات بحقل `wizardStep`. أكشن الـwizard (`wizard/actions.ts`):

- `createDraftInvoice` — ينشئ مسودّة بـ`wizardStep`
- `getInvoiceForWizard` — يحمّل المسودّة (بنطاق المستأجر)
- `completeInvoiceWizard` — يتحقّق ويُنهي
- `updateInvoiceWizardStep` — يتتبّع التقدّم
- `deleteDraftInvoice` — للمسودّات فقط مع تنظيف cascading

### البريد (Resend)

`send-invoice-email.tsx` — قالب React بأنماط مدمجة:

- `"Hello {firstName},"` + "Your invoice is ready for review."

## هوية المدرسة

`settings/content.tsx` يوفّر accordion-ين:

1. **شعار الفاتورة** — `FileUploader` إلى `finance/invoice-logos/`، حدّ أقصى 5 MB.
2. **توقيع الفاتورة** — اسم + صورة إلى `finance/signatures/`، حدّ أقصى 2 MB.

التخزين:

```
UserInvoiceSettings { id, userId (unique), schoolId, invoiceLogo?, signature → UserInvoiceSignature }
UserInvoiceSignature { id, settingsId (unique), schoolId, name?, image? }
```

الهوية مُخزَّنة لكنّها **غير مُستهلَكة**: PDF يتوقّع `schoolLogo` / `schoolName` / `schoolAddress`؛ البريد لا يتوقّع حقول هوية. حالياً بيانات يتيمة.

## مناطق الخدمة

ستّة مستهلكين عبر المنصّة:

1. **إدارة Admin / Accountant** — CRUD كامل عبر `/finance/invoice/`
2. **فواتير رسوم الطلاب** — تُولَّد تلقائياً بـ`confirmEnrollment()`:

   ```
   FeeStructure → FeeAssignment (PENDING) → confirmEnrollment()
     → createInvoiceFromEnrollment() → UserInvoice (ENR- prefix)
   ```

   يُحوّل `FeeStructure.name → item_name`, `FeeAssignment.finalAmount → price`. الحالة `UNPAID`، الاستحقاق 30 يوماً. غير قاتل — التسجيل ينجح حتى لو فشلت الفاتورة. لا FK بين `FeeAssignment` و`UserInvoice`؛ الربط في زمن التشغيل فقط.

3. **لوحة الطالب** — `InvoiceHistorySection` ("Fee Payments") عبر `getInvoicesByRole("STUDENT")`
4. **لوحة وليّ الأمر** — نفس المكوّن، يجلب فواتير الأبناء عبر `getInvoicesByRole("GUARDIAN")`
5. **لوحة Accountant** — `accountant.tsx` مع المقاييس، `ProgressCard` ("Invoices Paid")، `InvoiceHistorySection`
6. **محور المالية** — `finance/content.tsx` يعرض إحصائية "Unpaid Invoices" + رابط سريع

## Schema

5 نماذج Prisma:

- **`UserInvoice`** — السجلّ الأساسي. الحقول: `id`, `invoice_no`, `invoice_date`, `due_date`, `currency` (افتراضي `USD`), `sub_total` / `discount` / `tax_percentage` / `total` (Decimal), `notes`, `status` (`InvoiceStatus`, افتراضي `UNPAID`), `wizardStep` (null = مكتملة). العلاقات: `userId`, `schoolId`, `journalEntryId`. العناوين: واحد-إلى-واحد `fromAddressId` + `toAddressId`. الفهارس: `@@unique([schoolId, invoice_no])`, `[userId]`, `[schoolId]`, `[status]`.
- **`UserInvoiceItem`** — `id`, `item_name`, `quantity`, `price` (Decimal), `total` (Decimal), `invoiceId`, `schoolId`.
- **`UserInvoiceAddress`** — `id`, `name`, `email?`, `address1`, `address2?`, `address3?`, `schoolId`.
- **`UserInvoiceSettings`** — إعدادات لكل مستخدم (راجع الهوية).
- **`UserInvoiceSignature`** — تخزين التوقيع (راجع الهوية).

```typescript
enum InvoiceStatus {
  PAID
  UNPAID    // افتراضي
  OVERDUE
  CANCELLED
}
```

التمييز: **`UserInvoice`** = على مستوى المدرسة (هذه الوحدة). **`Invoice`** = فوترة منصّة Stripe (`@@map("invoices")`)؛ نموذج `Receipt` يرتبط بـ`Invoice`، لا بـ`UserInvoice`.

## الإعدادات

خيارات العملة: `USD`, `EUR`, `GBP`, `INR`, `CAD`, `AUD`. الـwizard يضيف `SDG`, `SAR`, `AED`.

```typescript
STEP_FIELDS = {
  1: ["from", "to"],
  2: ["invoice_no", "currency", "invoice_date", "due_date", "items"],
  3: ["sub_total", "discount", "tax_percentage", "total", "notes"],
```

صيغ الترقيم:

- Modal — `I{YY}{NNN}` (مثلاً، `I26042`)، يُولَّد على العميل

- Enrollment — `INV-{NNNNNN}` نفس نمط الترقيم التلقائي
