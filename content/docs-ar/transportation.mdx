تدير وحدة النقل عملية حافلات المدرسة من البداية إلى النهاية: المركبات والسائقون، ومسارات مُحدَّدة جغرافيًا بمحطات تُعيَّن على الخريطة، وتعيينات الطلاب، والرحلات اليومية مع سجلات الركوب، والرسوم، وعرض الخدمة الذاتية للطلاب وأولياء الأمور. تضيف ترقية 2026 **الذكاء** — حساب المسار الأكفأ، وتقدير أوقات الوصول حسب جرس المدرسة، وإعادة التوجيه التلقائي عند غياب طالب، وتتبع الحافلة مباشرة، وتنبيهات أولياء الأمور من الباب إلى الباب.

## الأدوار

| الدور                  | الصلاحيات                                                                             |
| ---------------------- | ------------------------------------------------------------------------------------- |
| **المدير / المطوّر**   | كل شيء — الأسطول والمسارات والمحطات والسائقون والتعيينات والرحلات والإعدادات والمخاطر |
| **الموظف**             | التعيينات والرحلات والركوب والموافقة على طلبات التخطّي (عرض فقط للأسطول/المسارات)     |
| **المعلّم**            | عرض الرحلات وتسجيل الركوب                                                             |
| **المحاسب**            | معاينة الرسوم                                                                         |
| **الطالب / ولي الأمر** | مسارات أبنائه، خريطة الحافلة المباشرة، طلبات «تخطّي الاصطحاب»                         |

## المفاهيم الأساسية

- **الأسطول** — مركبات (حافلة / فان / حافلة صغيرة / سيارة) مع السعة وتتبّع التسجيل/التأمين/الفحص.
- **السائقون** — سجلات الرخصة (الفئة + الانتهاء) مع روابط اختيارية بموظف وبحساب مستخدم.
- **المسارات** — مسارات مُسمّاة (اصطحاب / توصيل / ذهاب وإياب) بمحطات مُرتَّبة ومحدَّدة على الخريطة. تُحسب مسافة المسار تلقائيًا متى توفّرت إحداثيات كل المحطات.
- **نقاط الاصطحاب (من الباب إلى الباب)** — لكل طالب نقطة اصطحاب (ونقطة توصيل منفصلة اختياريًا) محدَّدة جغرافيًا في ملفه. عند تعيين الطالب على مسار تصبح نقطته محطة يرتّبها المُحسِّن.
- **الرحلات** — رحلة واحدة لكل مسار + تاريخ + اتجاه، محميّة بآلة حالة (مجدولة → جارية → مكتملة / ملغاة). يُنشئ بدء الرحلة سجل ركوب لكل راكب نشط.
- **الركوب** — لكل طالب: بانتظار / صعد / نزل / فُوِّت / مُعفى، يسجّله السائق/الموظف أو جهاز السياج الجغرافي.

## تحسين المسار وأوقات الوصول

فعّل **الإعدادات ← تحسين المسار** لحساب أكفأ ترتيب للمحطات وأوقات الوصول لكل محطة.

- **ترتيب فعّال** — يقلّل المُحسِّن زمن التنقل بين المحطات والمدرسة. مع رمز خادم Mapbox يستخدم أوقات طرق **واعية بحركة المرور** آنيًا، وإلا يعود إلى ترتيب هندسي (Haversine). لا يعطّل الرحلة أبدًا.
- **أوقات الوصول حسب الجرس** — تُوقَّت رحلات الاصطحاب للوصول إلى المدرسة قبل الحصة الأولى مباشرة (ناقص مهلة الاصطحاب)؛ وتنطلق رحلات التوصيل بعد الحصة الأخيرة. تظهر التقديرات لكل محطة في صفحة تفاصيل الرحلة.
- **تحسين مسار** — يعيد زر «تحسين المسار» ترتيب محطاته؛ وتُجمَّد خطة كل رحلة (الترتيب + أوقات الوصول + خط الخريطة) على الرحلة كي لا تُعيد تعديلات المسار لاحقًا كتابة السجل.

## إعادة التوجيه الواعية بالغياب

عند خروج طالب من المعادلة، تُعاد حسبة الرحلة التالية بدونه.

- **«تخطّي الاصطحاب» لولي الأمر** — من عرض الخدمة الذاتية يطلب ولي الأمر يومًا خارج الحافلة. يبقى الطلب **معلّقًا** حتى يوافق المشرف عليه (في شاشة الرحلات) — بوابة أمان كي لا يُخلى مقعد بالخطأ.
- **بنّاء الرحلات الليلي** — مهمّة مجدولة تبني رحلات يوم العمل التالي (مع مراعاة أسبوع الدراسة والإجازات)، وتستبعد الطلاب الغائبين (الحضور) أو المعتمَد تخطّيهم، وتُسقِط أي محطات صارت فارغة، وتعيد تحسين المسار المختصر.

## التتبع المباشر

أثناء سير الرحلة:

- **السائق** — يفتح الرحلة ويضغط **بدء المشاركة**؛ فيبثّ الجهاز موقعه.
- **المدير وأولياء الأمور** — يرون الحافلة تتحرك على خريطة مباشرة إلى جانب المحطات المرتّبة وخط المسار. يرى ولي الأمر رحلة ابنه فقط.
- **«الحافلة تقترب»** — عند اقتراب الحافلة من محطة ضمن النطاق المحدَّد، تصل الأسر المنتظرة تنبيهات.

تستخدم التحديثات المباشرة خادم البثّ عند توفّره، وإلا تعود إلى الاستطلاع كل ~10 ثوانٍ.

## الإشعارات

تُرسَل أحداث الرحلة (انطلقت، وصلت، أُلغيت، فُوِّتت، صعد، نزل، تقترب) إلى أولياء الأمور **داخل التطبيق وعبر واتساب** (متى توفّر رقم هاتف)، بلغة المدرسة. وتوجد خيارات إيقاف لكل حدث في الإعدادات.

## مخاطر الطريق

يمكن للمشرفين تحديد الإغلاقات أو الحوادث أو الفيضانات أو أعمال الإنشاء (الإعدادات ← مخاطر الطريق) مع نطاق. يتجنّب مستوى المُحسِّن الاحتياطي المخاطر النشطة، ويُنبّه إنشاؤها أولياء أمور المسارات المتأثرة.

## ويب هوك الركوب بالسياج الجغرافي

يمكن لأجهزة GPS/RFID الخارجية تسجيل الركوب عبر `POST /api/transportation/geofence-boarding` باستخدام رمز واجهة برمجة للمدرسة (أنشئه في الإعدادات). مدرسة الرمز هي المستأجر — ولا يُقرأ أبدًا من جسم الطلب.

## قائمة الإعداد

1. أضِف المركبات والسائقين.
2. أنشئ المسارات؛ وحدّد المحطات على الخريطة (واضبط نقاط اصطحاب الطلاب على تعييناتهم).
3. (اختياري) اضبط `MAPBOX_SERVER_TOKEN` للتحسين الواعي بحركة المرور، وفعّل تحسين المسار في الإعدادات.
4. عيّن الطلاب على المسارات؛ وجدوِل الرحلات (أو دع المهمّة الليلية تبنيها).
5. (اختياري) يثبّت السائقون التطبيق للتتبع المباشر؛ وأنشئ رمز واجهة لأجهزة السياج/الـ GPS.

## ملاحظات وحدود

- **تأخيرات الطقس** مُصمَّمة لكنها غير مُفعَّلة — تتطلّب مفتاح واجهة طقس.
- **البثّ المباشر** يحتاج إلى نشر خادم البثّ؛ وحتى ذلك تستطلع الخريطة المباشرة.
- تتطلّب ميزات الخريطة `NEXT_PUBLIC_MAPBOX_TOKEN`؛ وبدونه يمكن إدخال العناوين يدويًا.
