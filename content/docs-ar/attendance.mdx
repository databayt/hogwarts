تتبّع وحدة الحضور حضور الطلاب عبر طرق التقاط متعدّدة، مع وصول مبني على الأدوار، إشعارات للأهل، مستويات خطر وفق US DoE، تتبّع للتدخّلات، وتصدير CSV / Excel / PDF. المعلمون يرون فصولهم المسندة فقط؛ الطلاب وأولياء الأمور يرون سجلاتهم فقط.

> **متتبّع الإصدار:** [databayt/hogwarts#1](https://github.com/databayt/hogwarts/issues/1) — متتبّع رئيسي موحّد للحضور والامتثال. راجع كذلك [Aldar Epic 01](https://kun.databayt.org/en/docs/aldar#01--adek-attendance-submission).

## المسارات

كلها تحت `src/app/[lang]/s/[subdomain]/(school-dashboard)/attendance/`. التبويبات: `/`, `/manual`, `/qr-code`, `/records`, `/excuses`, `/analytics`, `/reports`, `/settings`. صفحات خارج التبويبات (URL فقط): `/barcode`, `/bulk-upload`, `/bulk`, `/interventions`, `/geo`, `/analysis`, `/recent`. `/parent/attendance` يعيد التوجيه إلى `/attendance/records`. راجع جدول طرق الالتقاط أدناه لغرض كل route؛ راجع RBAC للوصول حسب الدور.

## الحضور اليدوي

شريط أدوات مسطّح inline (`PlatformToolbar`): محدّد الفصل، منتقي التاريخ، أزرار حالة جماعية، Save. `getCurrentPeriod()` يختار تلقائياً الفصل المطابق لحصة الجدول الحالية؛ الأسماء تُترجَم عند الطلب عبر `getText()`. لوحة المفاتيح: `P` / `A` / `L` لحاضر / غائب / متأخر، `Cmd/Ctrl+S` للحفظ. أولياء الأمور للطلاب الغائبين يُخطَرون تلقائياً عند الحفظ؛ `AbsenceIntention` معتمَد يحوّل ABSENT → EXCUSED.

## إشراك الأهل

`triggerAbsenceNotification()` يُطلَق عند الغياب: داخل التطبيق (أولوية عالية)، SMS عبر Twilio (إن كان مهيّأً)، بريد. يستخدم `preferredLanguage` للمدرسة.

أسباب العذر: طبّي، طارئ عائلي، ديني، نشاط مدرسي، نقل، طقس، أخرى. معتمَد → `EXCUSED`؛ مرفوض → غير معذور.

عرض الطالب / ولي الأمر: نسبة + اتجاه، تقويم تاريخي، تفصيل الحالات، تقديم عذر للغيابات غير المعذورة.

## الإنذار المبكّر

مستويات خطر US DoE:

| الخطر       | معدّل الحضور | معدّل الغياب |
| ----------- | ------------ | ------------ |
| مُرضٍ       | ≥95%         | `<5%`        |
| في خطر      | 90 – 94.9%   | 5 – 9.9%     |
| مزمن متوسّط | 80 – 89.9%   | 10 – 19.9%   |
| مزمن حادّ   | `<80%`       | ≥20%         |

تتبّع التدخّلات في `/attendance/interventions` — 14 نوع تدخّل (تواصل مع ولي الأمر، زيارة منزلية، إحالة لمرشد / أخصائي اجتماعي / إداري، عقد حضور، إحالة تغيّب، مورد مجتمعي، دعم أكاديمي، إرشاد، تحفيز، أخرى) مع حالات Scheduled → In Progress → Completed / Cancelled / Escalated. محرّك السياسات في `actions/policy.ts` يقيّم المُحفّزات ويدير الاستثناءات.

## التحليلات والتقارير

تحليلات بـ7 تبويبات في `/analytics`: Summary، Calendar، Trends، Patterns، Methods، Classes، At Risk.

التقارير في `/reports`: فلترة بالفصل، الطالب، الحالة، نطاق التاريخ، الحصة. تصدير: CSV، Excel (5 أوراق — summary، detailed، by status، by class، by date)، PDF (بهوية المدرسة).

الامتثال: `getComplianceDashboard()`, `getComplianceReport()`, `getScheduledReports()`.

## Schema

```prisma
model Attendance {
  id           String           @id @default(cuid())
  schoolId     String
  studentId    String
  classId      String
  date         DateTime         @db.Date
  status       AttendanceStatus
  method       AttendanceMethod @default(MANUAL)
  checkInTime  DateTime?
  checkOutTime DateTime?
  notes        String?
  markedBy     String?
  markedAt     DateTime         @default(now())
  deletedAt    DateTime?

  periodId    String?
  timetableId String?
  periodName  String?

  @@unique([schoolId, studentId, classId, date, periodId])
  @@index([schoolId, date, periodId])
}
```

## الامتثال للجهات التنظيمية (ADEK eSIS — الإمارات)

يمكن لمدارس الإمارات (`School.country === "AE"`) تفعيل رفع يومي لملف CSV
إلى بوابة ADEK eSIS على `esis.adek.gov.ae` بالإضافة إلى التواصل مع ولي
الأمر خلال ساعتين من الغياب غير المُبلَّغ (متطلب ADEK). الإطار العام —
إضافة جهة تنظيمية جديدة (هيئة التعليم السعودية، وزارة التعليم القطرية،
وزارة التعليم السودانية) تتطلب فقط إضافة قيمة إلى enum
`ComplianceProvider` ودليل تحت `src/lib/compliance/providers/<authority>/`.

| القطعة                 | المسار                                            | الغرض                                                     |
| ---------------------- | ------------------------------------------------- | --------------------------------------------------------- |
| واجهة الإعدادات        | `/{lang}/compliance`                              | تفعيل، الوضع، المهلة، وقت الإرسال اليومي، قائمة الإرسالات |
| إدارة عبر المستأجرين   | `/{lang}/(saas-dashboard)/compliance` (DEVELOPER) | مجموعات بيانات الاعتماد المشتركة لنمط Aldar               |
| كرون يومي              | `/api/cron/esis-submit` (`0 10 * * *`)            | إدخال طوابير + توزيع على المدارس                          |
| كرون متابعة            | `/api/cron/absence-followup` (`*/30 * * * *`)     | سجل امتثال ساعتين                                         |
| Webhook                | `/api/webhooks/adek`                              | استلام `ACCEPTED`/`REJECTED` غير متزامن                   |
| Worker (خارج المستودع) | `tools/adek-rpa-worker/` (Fly.io)                 | Playwright عند عدم توفر API                               |

أوضاع الموصّل — قابلة للضبط لكل مدرسة × مزوّد:

| الوضع          | الوظيفة                                                                                                        |
| -------------- | -------------------------------------------------------------------------------------------------------------- |
| `DRY_RUN`      | إنشاء CSV، حفظه للتدقيق، يرفعه المسجّل يدوياً إلى eSIS                                                         |
| `PIGGYBACK`    | الإرسال عبر بيانات اعتماد مجموعة مشتركة (نمط الدار، صف اعتماد واحد × N مدارس، قاطع دائرة عند التدفقات الفاشلة) |
| `OFFICIAL_API` | واجهة برمجية موثّقة من المزوّد (بانتظار وصول مطوّرين من ADEK)                                                  |
| `RPA`          | يلتقط عامل Playwright خارجي الصفوف `QUEUED` ويرفع عبر بوابة eSIS                                               |
| `DISABLED`     | تخطّي في الكرون                                                                                                |

### مهلة التواصل مع ولي الأمر — ساعتين

عند تفعيل `SchoolComplianceConfig.enabled = true`، فإن
`triggerAbsenceNotification` يُرسل عبر قنوات `in_app + email + whatsapp`
(بدلاً من `in_app` فقط) لكي تستهلك كرونات البريد و WhatsApp الصف. كل
30 دقيقة، يبحث كرون `absence-followup` عن صفوف ABSENT التي:

- `markedAt < now() - parentContactSlaMinutes` (افتراضي 120، محسوب بـ UTC)
- لا يوجد لها `AttendanceExcuse`
- لا يوجد لها `AbsenceIntention` معتمَد يغطّي اليوم
- لا يوجد لها `AttendanceIntervention` من نوع `PARENT_PHONE_CALL` /
  `PARENT_EMAIL` خلال آخر 24 ساعة

يُرسل إشعار `absence_unreported_followup` لكل ولي أمر، ثم يكتب صفّ
`AttendanceIntervention` (`PARENT_EMAIL`, parentNotified=true,
contactMethod=`in_app+email+whatsapp`). هذا الصفّ هو سجل التدقيق الموجّه
للجهة التنظيمية — يستعلم `getParentContactSlaReport()` عنه لتقديم
عدّ "تواصل ضمن المهلة" لتدقيقات ADEK.

### النموذج البياني (جانب الامتثال)

خمسة كيانات جديدة في `prisma/models/compliance.prisma` — ليس أيٌّ منها
خاصّاً بـ ADEK:

- `ComplianceProvider` enum (حالياً `ADEK_ESIS`، `CUSTOM`)
- `ConnectorMode` enum
- `ComplianceSubmissionStatus` enum + `CircuitBreakerState` enum
- `SchoolComplianceConfig` — صفّ واحد لكل مدرسة × مزوّد (تفعيل اختياري)
- `SharedComplianceCredentialGroup` — بيانات اعتماد مشتركة لمجموعة (نمط الدار)
- `SchoolComplianceCredential` — بيانات اعتماد لكل مدرسة (وضع OFFICIAL_API)
- `ComplianceSubmission` — صفّ لكل محاولة، مفهرس بـ `(schoolId, provider,
submissionDate, attemptNumber)`. تتسلسل المحاولات عبر `supersededBy`.
  يُحفظ CSV داخل `csvArtifactContent` ويُنزَّل عبر
  `/api/compliance/artifact/[submissionId]` (S3 URL موقّع قيد التخطيط).

تمّ توسيع `AuditLog.userId` ليقبل null في نفس الإصدار لكي يُسجَّل كرون /
عامل / webhook كأحداث نظام — تمرير `userId: null` في المكالمات.

أُضيف نوع `NotificationType.absence_unreported_followup` لمسار المهلة.
