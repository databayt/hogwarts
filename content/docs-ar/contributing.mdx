## أعراف الـ commit

الصيغة: `category(scope): message`. الفئات:

| الفئة              | الاستخدام                        |
| ------------------ | -------------------------------- |
| `feat` / `feature` | كود جديد، ميزات جديدة            |
| `fix`              | إصلاح أخطاء (اربط issue إن وُجد) |
| `refactor`         | تغيير كود ليس إصلاحاً ولا ميزة   |
| `docs`             | وثائق فقط                        |
| `build`            | تغييرات البناء أو التبعيات       |
| `test`             | اختبارات جديدة أو محدّثة         |
| `ci`               | تغييرات CI / GitHub Actions      |
| `chore`            | كل ما عدا ذلك                    |

استخدم اسم المجلد كـ scope:

```
feat(marketing)     # SaaS marketing
feat(operator)      # SaaS dashboard
feat(site)          # School marketing
feat(platform)      # School dashboard
feat(auth)          # Authentication
feat(onboarding)    # Onboarding flow
feat(ui)            # shadcn/ui primitives
feat(atom)          # Atomic components
feat(template)      # Templates / sections
fix(docs)           # إصلاحات الوثائق
```

راجع [conventionalcommits.org](https://www.conventionalcommits.org/) للمواصفة الكاملة.

## الاختبارات

```bash
pnpm test
```

كل PR يجب أن تجتاز الاختبارات. الميزات الجديدة يجب أن تتضمن اختبارات.

## أول PR لك

استهدف أقل من 15 دقيقة من الاستنساخ إلى PR.

```bash
git checkout -b docs/first-pr
pnpm install
pnpm dev
```

اختر مهمة صغيرة — تصحيح خطأ مطبعي، توضيح جملة، إضافة أمر مفقود. نفّذ التعديل. إن غُيِّرت الواجهة، التقط لقطة LTR + RTL.

```bash
pnpm build
git add -A
git commit -m "docs: clarify get-started step"
git push origin HEAD
```

افتح PR بعنوان قصير محدّد ووصف موجز يبيّن سبب الفائدة. اربط الـissues ذات الصلة. أهلاً بك.

## المجتمع

ندير اقتصاداً مشتركاً لمطوّري البرمجيات — مفتوح المصدر، عمل مدفوع عند توفّره، ومشاركة عادلة للإيرادات. راجع [الاقتصاد المشترك](/ar/docs/shared-economy).

### المبادئ

- لا مواعيد نهائية، لكن مساهمات مستمرة.
- شفافية — backlog عام، أولويات معلنة، changelog.
- عدالة — مكافأة الأثر المستدام والصيانة والموثوقية.
- شمولية — العربية + الإنجليزية، واجهات قابلة للوصول، تفكير صديق للوضع غير المتصل.

### كيف نتعاون

- async-first — الـissues والـPRs والوثائق تحمل السياق.
- انحياز لشحن خطوات صغيرة.
- توثيق القرارات في صفحة الوثيقة المعنية أو في الـPR.

### الأدوار (مرنة، اختيارية)

- **Maintainers** — يحافظون على صحة المستودعات ويراجعون الـPRs.
- **Contributors** — يبنون الميزات ويصلحون الأخطاء ويحسّنون الوثائق.
- **Coordinators** — يفرزون الـissues ويرتّبون خارطة الطريق.
