تستخدم Hogwarts Next.js App Router مع مكونات قائمة على الميزات. المسارات تعكس المكونات 1:1 — إن عرفت الـURL عرفت مكان الكود. راجع [الأنماط](/ar/docs/pattern) لأعراف الكود و[البنية المعمارية](/ar/docs/architecture) لشرح نمط المرآة.

## جذر المشروع

```text
hogwarts/
├── public/                # أصول ثابتة
├── prisma/
│   ├── schema.prisma      # datasource + generator + imports
│   ├── models/            # 73 ملف .prisma
│   └── seeds/             # سكربتات seed (تُشغَّل عبر pnpm db:seed:single)
├── src/
│   ├── app/               # مسارات Next.js App Router
│   ├── components/        # مكونات الميزات (مرآة لـapp/)
│   ├── lib/               # وحدات utilities
│   ├── types/             # تعريفات TypeScript عامة
│   ├── styles/            # أنماط عامة (typography.css)
│   └── proxy.ts           # Edge proxy — استخراج النطاق وإعادة الكتابة
├── content/docs-en/       # وثائق MDX (إنجليزية)
├── content/docs-ar/       # وثائق MDX (عربية، مرآة)
├── scripts/               # سكربتات بناء + seed
├── tests/                 # مجموعات E2E (Playwright)
├── .env                   # بيئة مركزية — لا .env.local أبداً
├── next.config.ts
├── package.json           # pnpm فقط
└── tsconfig.json          # @/* → ./src/*
```

## المسارات (`src/app/`)

كل المسارات تحت `[lang]/` لتوجيه اللغة (`ar`, `en`). أربع نقاط دخول تُنفَّذ عبر مجموعات App Router.

### نقاط الدخول

| المجموعة             | الـURL                          | الغرض              | الوصول                       |
| -------------------- | ------------------------------- | ------------------ | ---------------------------- |
| `(saas-marketing)`   | `ed.databayt.org/`              | تسويق SaaS عام     | عام                          |
| `(saas-dashboard)`   | `ed.databayt.org/dashboard`     | لوحة المشغّل       | DEVELOPER فقط                |
| `(school-marketing)` | `school.databayt.org/`          | موقع المدرسة العام | عام                          |
| `(school-dashboard)` | `school.databayt.org/dashboard` | منصة المدرسة       | أدوار المدرسة المصادَق عليها |

مقطع `s/[subdomain]/` **داخلي فقط**. الـmiddleware يعيد كتابة `school.databayt.org/dashboard` → `/[lang]/s/school/dashboard`. كود العميل يستخدم مسارات نسبية للمستأجر.

## المكونات (`src/components/`)

المسارات تعكس المكونات 1:1. ملف الصفحة رفيع — يستورد ويعرض `content.tsx` للميزة.

```tsx
// src/app/[lang]/s/[subdomain]/(school-dashboard)/(listings)/students/page.tsx
import { Content } from "@/components/school-dashboard/listings/students/content"

export default function StudentsPage() {
  return <Content />
}
```

## نمط الملفات الموحّد للميزة

كل مجلد ميزة يأخذ نفس الشكل. راجع [الأنماط](/ar/docs/pattern) للمرجع الكامل.

| الملف                                 | النوع          | الغرض                                             |
| ------------------------------------- | -------------- | ------------------------------------------------- |
| `content.tsx`                         | خادم           | الواجهة الرئيسية، جلب بيانات، تنظيم تخطيط         |
| `client.tsx`                          | عميل           | حدود `"use client"`، يستقبل بيانات الخادم         |
| `actions.ts`                          | server actions | تعديلات مع auth + tenant + validation             |
| `queries.ts`                          | خادم           | استعلامات قراءة فقط                               |
| `authorization.ts`                    | خادم           | فحوص وصول حسب الدور                               |
| `validation.ts`                       | مشترك          | Zod schemas                                       |
| `types.ts`                            | مشترك          | أنواع المجال + UI                                 |
| `config.ts`                           | مشترك          | ثوابت، enums، افتراضيات                           |
| `form.tsx`                            | عميل           | نماذج إنشاء / تعديل (RHF + Zod)                   |
| `table.tsx`                           | عميل           | غلاف DataTable مع أعمدة الميزة                    |
| `columns.tsx`                         | عميل           | تعريفات الأعمدة (يستخدم `useMemo` إن وُجدت hooks) |
| `card.tsx`, `detail.tsx`, `views.tsx` | عميل           | بطاقات KPI، صفحة تفاصيل، مبدّلات عرض              |
| `list-params.ts`                      | عميل           | URL search-param cache (nuqs)                     |
| `util.ts`                             | خالص           | helpers الميزة                                    |
| `hooks.ts` / `use-<x>.ts`             | عميل           | hooks الميزة                                      |
| `README.md`                           | وثيقة          | سياق على مستوى الكتلة (~40 سطر)                   |
| `ISSUE.md`                            | وثيقة          | issues نشطة، سجل منجَز (~40 سطر)                  |

## مخطط Prisma (`prisma/`)

```text
prisma/
├── schema.prisma                # datasource + generator + multi-file imports
├── models/                      # 73 ملف .prisma (302 نموذج، 149 enum)
│   ├── school.prisma
│   ├── auth.prisma
│   ├── students.prisma
│   ├── staff.prisma
│   ├── academic.prisma
│   ├── timetable.prisma
│   ├── attendance.prisma
│   ├── catalog.prisma
│   ├── exam.prisma
│   ├── finance-core.prisma
│   ├── subscription.prisma
│   ├── stream.prisma
│   ├── transportation.prisma
│   └── … 60 آخر
└── seeds/                       # سكربتات seed
    ├── index.ts                 # مدير — لا يُشغَّل مباشرة
    ├── single.ts                # مشغّل seed وحيد — استخدمه دائماً
    └── …
```

شغّل seeds عبر `pnpm db:seed:single <name>`. لا تشغّل `pnpm db:seed`. راجع [Safe seeds](/ar/docs/safe-seeds).

## القواعد المفتاحية

| القاعدة             | التفصيل                                                            |
| ------------------- | ------------------------------------------------------------------ |
| مدير الحزم          | pnpm فقط (متطلب Vercel)                                            |
| البيئة              | `.env` واحد — لا `.env.local` ولا `.env.development`               |
| عزل المستأجرين      | كل استعلام DB يتضمّن `schoolId`                                    |
| نمط المرآة          | مسارات `app/` تطابق 1:1 مجلدات `components/`                       |
| HTML دلالي          | لا فئات `text-*` / `font-*` مكتوبة                                 |
| توجيه اللغة         | كل المسارات تحت `[lang]/` لـ`ar` / `en`                            |
| توجيه النطاق الفرعي | مسارات المدرسة تحت `s/[subdomain]/` (داخلي فقط)                    |
| CSS منطقي           | `ms-*`, `me-*`, `ps-*`, `pe-*` — لا `ml-*`, `mr-*`, `pl-*`, `pr-*` |
| Import alias        | `@/*` → `./src/*`                                                  |
| المنفذ              | 3000 دائماً في التطوير                                             |
| Seeding             | دائماً `pnpm db:seed:single <name>` — لا `pnpm db:seed`            |
