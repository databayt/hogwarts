## الملحمة 01 — البنية التحتية

**الحالة:** `مُنفَّذ + تحسين`

الأساس متعدد المستأجرين. كل مدرسة نطاق فرعي (`school.databayt.org`)، يتم حله على الحافة، معزول بـ `schoolId`. انظر [البنية المعمارية](/ar/docs/architecture) و[تعدد المستأجرين](/ar/docs/multi-tenancy).

- مخطط Prisma متعدد المستأجرين مع تحديد نطاق `schoolId` على كل نموذج مملوك للمستأجر (302 نموذج، 73 ملف مخطط).
- توجيه النطاقات الفرعية عبر `proxy.ts` على حافة Next.js 16.
- ربط نطاق مخصص عبر Upstash Redis (`custom-domain:${host}`).
- `getTenantContext()` يحل `schoolId` من ترويسة `x-subdomain` لكل إجراء خادم.
- Vercel + Neon (PostgreSQL serverless) + AWS S3 + CloudFront (`cdn.databayt.org`).
- مراقبة أخطاء Sentry (تتعامل بأمان بدون DSN).
- 11 مهمة cron يومية للصيانة (الإشعارات، التذكيرات، الرسوم المتأخرة، التنظيف).
- CI/CD: 245 ملف اختبار مخصص، مجموعات Playwright للمصادقة / تعدد المستأجرين / RBAC / اختبار الدخان.
- نظافة حزمة الحافة: `outputFileTracingExcludes` يبقي الوظائف serverless تحت حد Vercel 300 MB.
- `experimental.serverActions.bodySizeLimit: "10mb"` لتحميلات الملفات المباشرة.

## الملحمة 02 — المصادقة والمستخدمون

**الحالة:** `مُنفَّذ`

NextAuth v5 مع البريد + كلمة المرور، OAuth (Google, Facebook), 2FA، تدفق الدعوة الكامل، تسجيل التدقيق. انظر [المصادقة](/ar/docs/authentication).

- NextAuth v5 (beta.30) مع استراتيجية JWT ومحول Prisma متعدد المستأجرين مخصص.
- البريد + كلمة المرور (bcrypt تكلفة 12).
- موفرو OAuth: Google, Facebook.
- مصادقة ثنائية عبر `TwoFactorToken` + `TwoFactorConfirmation` (`src/components/auth/verification/2f-token.ts`, `2f-confirmation.ts`).
- 8 أدوار: `DEVELOPER`, `ADMIN`, `TEACHER`, `STUDENT`, `GUARDIAN`, `ACCOUNTANT`, `STAFF`, `USER`.
- قدرات CASL (`@casl/ability` + `@casl/prisma`) للتفويض الدقيق.
- RBAC على مستوى المسار عبر بوابات الأدوار.
- تدفق إعادة تعيين كلمة المرور مع التحقق من البريد.
- التحقق من البريد عند التسجيل.
- إدارة الجلسة مع التجديد التلقائي.
- صفحات الملف الشخصي لكل دور (متغيرات admin / teacher / student / parent / staff).
- نظام دعوة المستخدم (`src/lib/invitation-utils.ts`, `src/app/[lang]/accept-invite/page.tsx`, مهمة cron يومية `/api/cron/expire-invitations`).
- تسجيل التدقيق للعمليات الحساسة (نموذج `AuditLog`, `src/lib/audit-log.ts`).
- حماية القوة الغاشمة عبر `LoginAttempt` وحدود المعدل.
- نماذج الامتثال + الموافقة القانونية (`LegalConsent`, `LegalDocument`, `ComplianceLog`).

- **تقديم عام من 9 خطوات** في `/[lang]/s/[subdomain]/application/...`: شخصي ← اتصال ← موقع ← أكاديمي ← رسوم ← مرفقات ← عرض ← دفع ← نجاح.
- حفظ تلقائي مع استئناف `ApplicationSession`.
- **تتبع حالة بـ OTP** — يتحقق المتقدمون من الحالة بدون حساب.
- **تحميل وثائق** مع خطوة المرفقات.
- **مراجعة وثائق بالذكاء الاصطناعي** (`src/components/school-dashboard/admission/ai/`):
  - تصنيف وثائق قائم على Anthropic (`classify.ts`).
  - تسجيل اكتمال (`completeness.ts`).
  - محرك جدارة لدرجات القبول / المقابلة / الجدارة / الترتيب (`merit-engine.ts`).
  - استخراج إيصال بنكي للتحقق من الدفع غير المتصل.
- **حجز جولة** مع نموذج `TourBooking` وإدارة فتحات وقت المسؤول.
- **تدفق القبول / العرض** — `Application.admissionOffered`, `offerDate`, `offerExpiryDate`, `offerAccepted`.
- **تحويل تسجيل تلقائي** — قبول العرض يُشغِّل `enrollStudentInGradeClasses` (`src/lib/enrollment-sync.ts`)، منشئاً Student + User + Guardian + StudentYearLevel + AcademicGrade + FeeAssignments + UserInvoices + StudentDocuments في معاملة واحدة.
- **تسجيل الجدارة + الترتيب** مع جدول جدارة المسؤول.
- **حملات** (`AdmissionCampaign`) مع منتقي تقويم نطاق التاريخ.
- **دفع رسوم متعدد البوابات** — انظر الملحمة 09.
- تحليلات القبول، الإعدادات، التوضيع بالجملة، جدولة المقابلات.
- إشعارات (بريد + واتساب + push) عند كل انتقال حالة.
- 5 ملفات اختبار مخصصة: `actions.test.ts`, `enrollment-notifications.test.ts`, `enrollment-pipeline.test.ts`, `offer-expiry.test.ts`, `validation.test.ts`.

- **قوالب البريد** في `src/lib/email-templates/` مدعومة بـ React Email.
- 11 مهمة cron يومية تسلم الاتصالات المجدولة (نشر / انتهاء صلاحية الإعلانات، طوابير البريد / الواتساب، تذكيرات الأحداث، تذكيرات الواجبات، إشعارات الرسوم المتأخرة، تذكيرات المعلمين).

- مشغل فيديو مع تتبع التقدم.

- محرر نص ثري (Tiptap).
- لوحة تحليلات المسؤول.
- إشعارات بريد التسجيل / الإكمال.
- صفحة الدورة الرئيسية (بطل، منهج، مهارات).

- **تكامل ClickView CDN `مُنفَّذ + تحسين`** — مكتبة الصور تم تنزيلها جزئياً (~32–40 من ~986 درس)؛ الباقي يتراجع إلى عناوين CDN ClickView. متتبع في [خارطة الطريق](/ar/docs/roadmap).

## مفاجآت ما وراء نطاق MVP الأصلي

هذه لم تكن في MVP المبكر لكنها شُحنت على طول الطريق. تظهر داخل الملحمة ذات الصلة أعلاه؛ هذه القائمة فحص سريع لأي شخص يراجع المجموعة.

- تصنيف وثائق + ترتيب جدارة + استخراج إيصال بنكي قائم على الذكاء الاصطناعي للقبول (الملحمة 05، Anthropic).
- تكامل Plaid + Dwolla المصرفي (الملحمة 09).
- ست بوابات دفع مع واجهة موفر قابلة للتوصيل (الملحمة 09).
- 11 طريقة التقاط حضور بما في ذلك بيومتري، جيوفنس، وكشك (الملحمة 06).
- بنك أسئلة مع توليد بالذكاء الاصطناعي، تقطيع المواد المصدر، وتحليلات (الملحمة 07).
- التحقق من الشهادة بكود قصير (الملحمة 07).
- جسر WhatsApp Evolution API مع إعادة محاولة + تراجع (الملحمة 10).
- خدمة ذاتية للنطاق المخصص عبر Upstash Redis (الملحمة 01).
- نظام تصريح ممر، تلعيب، تدخلات إنذار مبكر (الملحمة 06).
- سير عمل مساهمة مجتمع الكتالوج مع اقتراحات + موافقات (الملحمة 14).
- وحدة مبيعات + عميل محتمل مع كانبان (`sales.prisma` منفصل — يواجه المشغل).
- تخصيص السمة (`UserTheme`).
- 11 مهمة cron يومية للصيانة (الملحمة 01).
- AGENTS.md.draft + مجموعة `.claude/` شاملة (50 وكيل، 47 أمر، 16 مهارة، 13 دليل CLAUDE.md على مستوى الكتلة).
- 4 مناهج دول مزروعة على القرص مع أسماء مواد ثنائية اللغة خاصة بالسودان (الملحمة 14).
