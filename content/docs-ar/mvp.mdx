## 01 — البنية التحتية · `مُنفَّذ + تحسين`

الأساس متعدد المستأجرين. راجع [البنية المعمارية](/ar/docs/architecture) و[تعدد المستأجرين](/ar/docs/multi-tenancy).

- 302 نموذج Prisma عبر 73 ملف schema، مع `schoolId` على كل نموذج خاص بالمستأجر.
- توجيه نطاق فرعي عبر edge `proxy.ts` في Next.js 16.
- ربط نطاقات مخصصة عبر Upstash Redis.
- Vercel + Neon (PostgreSQL serverless) + AWS S3 + CloudFront (`cdn.databayt.org`).
- مراقبة Sentry، تتعامل بسلاسة بدون DSN.
- 11 cron job يومياً (إشعارات، تذكيرات، تأخّر رسوم، تنظيف).
- 245 ملف اختبار مخصّص، ومجموعات Playwright لـauth، multi-tenant، RBAC، smoke.
- نظافة الحزم: `outputFileTracingExcludes` تُبقي الدوال serverless تحت حد Vercel البالغ 300 ميغابايت.

## 02 — المصادقة والمستخدمون · `مُنفَّذ`

NextAuth v5 مع بريد + OAuth + 2FA + دعوات + audit. راجع [المصادقة](/ar/docs/authentication).

- NextAuth v5 (beta.30)، استراتيجية JWT، Prisma adapter متعدد المستأجرين.
- بريد + كلمة مرور (bcrypt cost 12)، OAuth Google، Facebook.
- 2FA عبر `TwoFactorToken` + `TwoFactorConfirmation`.
- 8 أدوار: `DEVELOPER`, `ADMIN`, `TEACHER`, `STUDENT`, `GUARDIAN`, `ACCOUNTANT`, `STAFF`, `USER`.
- صلاحيات CASL (`@casl/ability` + `@casl/prisma`) لتحكّم دقيق.
- إعادة تعيين كلمة المرور، التحقق من البريد، تجديد الجلسة تلقائياً.
- صفحات ملف لكل دور (admin / teacher / student / parent / staff).
- الدعوات: `src/lib/invitation-utils.ts`, `accept-invite/page.tsx`, cron يومي `expire-invitations`.
- نموذج `AuditLog` + حماية brute-force عبر `LoginAttempt`.
- نماذج الامتثال: `LegalConsent`, `LegalDocument`, `ComplianceLog`.

## 03 — الإعداد · `مُنفَّذ`

معالج onboarding من 18 خطوة ينشئ المدارس end-to-end. راجع [Onboarding](/ar/docs/onboarding).

- الخطوات: about-school · branding · capacity · description · discount · finish-setup · import · join · legal · location · price · schedule · stand-out · subdomain · title · visibility · congratulations · overview.
- نموذج `School` مع year، term، period، و`YearLevel` / `AcademicLevel` / `AcademicGrade`.
- `StreamType`: `GENERAL`, `SCIENCE`, `ARTS`, `COMMERCE`, `TECHNICAL`, `RELIGIOUS`.
- محرّرات إعدادات في `school/configuration/` لـ branding، capacity، discount، domain، hero، legal، location، modules، name format، plan، price، schedule، title، visibility.
- رفع شعار، منتقي ألوان، سمة (`UserTheme`)، منطقة زمنية.
- تدفّق طلب نطاق مخصّص.

## 04 — الأشخاص · `مُنفَّذ`

CRUD لأنواع الأشخاص الأربعة. راجع [الطلاب](/en/docs/students)، [المعلمون](/en/docs/teachers)، [القوائم](/en/docs/listings).

| الكيان                | خطوات المعالج  | أبرز الميزات                                                                                                   |
| --------------------- | -------------- | -------------------------------------------------------------------------------------------------------------- |
| الطلاب                | 12             | شخصي، تواصل، رسوم، تسجيل، موقع، صحة، أكاديمي، تعليم سابق، ولي أمر، مرفقات، صورة. ملف بـ8 تبويبات. استيراد CSV. |
| المعلمون              | 9              | معلومات، تواصل، توظيف، خبرة، مؤهلات، تجربة، موقع، صورة، مرفقات.                                                |
| الأهل / أولياء الأمور | متعدّد الخطوات | معلومات، تواصل. إجراء ربط طفل.                                                                                 |
| الكادر                | CRUD كامل      | `StaffMember`, `StaffAccessCard`, `StaffSalaryStructure`, `StaffSalarySlip`, `StaffTimesheetEntry`.            |

انتقالات الحالة: `enrolled`, `transferred`, `graduated`. تصدير دفعي، ورفع صور إلى S3 + CloudFront.

## 05 — القبول · `مُنفَّذ`

بوابة عامة مع مراجعة مستندات بـAI وتحويل تلقائي للتسجيل. راجع [القبول](/ar/docs/admission).

- 9 خطوات تقديم عام: personal → contact → location → academic → fees → attachments → offer → payment → success.
- حفظ تلقائي مع استئناف عبر `ApplicationSession`.
- متتبّع حالة بـOTP — يفحص المتقدّم حالته بدون حساب.
- AI في `school-dashboard/admission/ai/`: `classify.ts`, `completeness.ts`, `merit-engine.ts`، استخراج إيصالات بنكية.
- حجز جولة (`TourBooking`) مع إدارة المواعيد للأدمن.
- تدفّق القبول: `admissionOffered`, `offerDate`, `offerExpiryDate`, `offerAccepted`.
- تحويل تسجيل في معاملة واحدة عبر `enrollStudentInGradeClasses` — ينشئ Student + User + Guardian + StudentYearLevel + AcademicGrade + FeeAssignments + UserInvoices + StudentDocuments.
- ترتيب جدارة، `AdmissionCampaign` بمنتقي مدى تواريخ.
- دفع رسوم متعدّد البوابات — راجع الملحمة 09.
- إشعارات (بريد + WhatsApp + push) عند كل انتقال حالة.
- 5 اختبارات مخصّصة تغطّي actions، إشعارات التسجيل، خط أنابيب التسجيل، انتهاء العرض، التحقق.

## 06 — الحضور · `مُنفَّذ + تحسين`

حضور يومي بـ11 وضع التقاط. ترحيل section-centric جارٍ. راجع [الحضور](/ar/docs/attendance).

- 11 وضع `AttendanceMethod`: `MANUAL`, `GEOFENCE`, `QR_CODE`, `BARCODE`, `RFID`, `FINGERPRINT`, `FACE_RECOGNITION`, `NFC`, `BLUETOOTH`, `BULK_UPLOAD`, `KIOSK`.
- 24 مساراً تشمل manual، kiosk، bulk-upload، QR، barcode، geofence، hall-pass، intentions، excuses، letters (auto-gen)، interventions، gamification، early-warning، AI، analytics، reports.
- 33+ نموذج Prisma موزّعة بين `attendance.prisma`, `attendance-enhanced.prisma`, `geo-attendance.prisma`.
- الحالات: حاضر / غائب / متأخر / معذور، إشعار الأهل بالغياب + تتبّع التأخير.
- تحليل اتجاهات، ملخصات شهرية، تقارير الفصل / الطالب.
- **ترحيل section-centric `قيد التنفيذ`** — راجع `.bmad/planning/architecture-section-timetable-attendance.md`. اليوم: توافق رجعي مع `StudentClass`. التالي: `Section.id` في كل مكان.

## 07 — الامتحانات والتقييم · `مُنفَّذ`

دورة حياة التقييم end-to-end. راجع [الامتحانات](/en/docs/exams) و[معالج الامتحان](/en/docs/exam-wizard).

- 40+ نموذج Prisma عبر `exam.prisma`, `school-exam.prisma`, `school-qbank.prisma`, `quiz.prisma`, `quiz-game.prisma`, `quick-assessments.prisma`, `grading-scheme.prisma`, `school-assignment.prisma`, `progress-reports.prisma`.
- جدولة الامتحانات بأنواعها: midterm، final، quiz، mock.
- بنك أسئلة مع توليد بـAI، تقطيع مادة المصدر، تحليلات، تدفّق مراجعة.
- توليد ورقة الامتحان مع غلاف / رأس / تذييل / قوالب أنواع الأسئلة.
- تصحيح مع rubrics + معايير + حدود درجات + تجاوزات.
- كروت الدرجات، الدرجات، transcripts (PDF + CSV).
- شهادات بتحقّق-بكود في `/certificate/[shareToken]` و`/verify/[code]`.
- تدفّق ترقية: `PromotionBatch`, `PromotionCandidate`, `PromotionPolicy`, `Transcript`.

## 08 — الجدول الدراسي · `مُنفَّذ`

منشئ بصري مع اكتشاف تعارضات. راجع [الجدول](/en/docs/timetable).

- جدولة حصص بالسحب والإفلات مع إدارة فواصل.
- توزيع غرف عبر `RoomConstraint`.
- اكتشاف تعارضات عبر المعلمين والغرف والطلاب.
- مشاهد لكل معلم / فصل / غرفة.
- بدائل (`SubstitutionRecord`)، قيود معلم + فترات غير متاحة.
- محرّك توليد مع حلّ تعارضات، قوالب (`TimetableTemplate`, `TemplateApplication`).
- استثناءات جدول، تحليلات، تصدير PDF / طباعة.

## 09 — المالية والفوترة · `مُنفَّذ`

نظام مالي شامل — أكبر بكثير من MVP المعتاد. راجع [الرسوم](/ar/docs/fees) و[الفاتورة](/ar/docs/invoice).

- 36 نموذج Prisma عبر 7 ملفات schema مالية.
- **محاسبة مزدوجة القيد** — `ChartOfAccount`, `JournalEntry`, `LedgerEntry`, `AccountBalance`, `FiscalYear`.
- **6 بوابات دفع** في `src/lib/payment/providers/`: Stripe, bankak, tap, bank-transfer, cash, mobile-money.
- **بنوك** — Plaid + Dwolla، `BankAccount`, `BankReconciliation`, `Transaction`, `Transfer`, `PlaidItem`, `DwollaCustomer`.
- **رسوم** — `FeeStructure`, `FeeAssignment` مع جدول أقساط، `Payment`, `Refund`, `Scholarship`, `ScholarshipApplication`, `Fine`.
- **فوترة** — `UserInvoice` ببنود، عناوين، توقيع، إعدادات.
- **إيصالات** — PDF عبر `@react-pdf/renderer`.
- **رواتب** — `PayrollRun`, `SalaryStructure`, `SalarySlip`, `SalaryAllowance`, `SalaryDeduction`, `TimesheetEntry`, `TimesheetPeriod`.
- **ميزانية** — `Budget`, `BudgetAllocation`, `Expense`, `ExpenseCategory`, `Wallet`, `WalletTransaction`.
- **اشتراكات على مستوى المدرسة** — `Subscription`, `Invoice`, `BillingHistory`, `BillingPaymentMethod`, `BillingPreferences`, `SubscriptionTier`, `UsageMetrics`, `Discount`, `AppliedDiscount`, `CreditNote`.
- **تقارير + صلاحيات** — `FinancialReport`, `FinancePermission`.

## 10 — الاتصال والمراسلة · `مُنفَّذ`

إعلانات، رسائل فورية، إشعارات، WhatsApp. راجع [الرسائل](/en/docs/messages).

- **إعلانات** (`announcement.prisma`، 5 نماذج): النطاق (مدرسة / فصل / دور)، الأولوية، نشر مجدول، انتهاء صلاحية، تثبيت / إبراز، إشعارات قراءة، قوالب.
- **رسائل** (`messages.prisma`، 11 نموذج): `Conversation`, `ConversationInvite`, `ConversationParticipant`, `Message`, `MessageAttachment`, `MessageDraft`, `MessageReaction`, `MessageReadReceipt`, `PinnedMessage`, `StarredMessage`, `TypingIndicator`. Realtime عبر مخدّم Socket.io مخصّص (`server.js`).
- **إشعارات** (`notifications.prisma`، 7 نماذج): in-app + بريد + push.
- **WhatsApp** (`whatsapp.prisma`، 5 نماذج) — جسر Evolution API كامل مع إعادة محاولة بـ exponential backoff.
- قوالب React Email في `src/lib/email-templates/`.
- 11 cron job يومياً يسلّم الرسائل المجدولة.

## 11 — Stream / LMS · `مُنفَّذ + تحسين`

LMS بنطاق المدرسة. موجة أصول US جزئية. راجع [US curriculum](/en/docs/us-curriculum).

- 8 نماذج Prisma (`stream.prisma`): `StreamCourse`, `StreamCategory`, `StreamChapter`, `StreamLesson`, `StreamEnrollment`, `StreamLessonProgress`, `StreamCertificate`, `StreamAttachment`.
- إدارة CRUD للمساقات + الفئات، محرّر Tiptap rich-text.
- مشغّل فيديو مع تتبّع تقدّم، تسجيل طلاب + شهادات إكمال.
- تدفّق دفع Stripe (cancel / success)، حصة تخزين لكل مدرسة (`videoStorageUsedBytes` / `videoStorageQuotaBytes`).
- **source CDN جزئي** — ~32–40 من ~986 درساً مُنزّلة، الباقي يعتمد على source CDN. يُتابَع في [Roadmap](/ar/docs/roadmap).

## 12 — المكتبة · `مُنفَّذ`

كتالوج، مساهمة، سجلات إعارة، إدارة المدرسة. راجع [المكتبة](/en/docs/library).

- مكونات في `src/components/library/`: admin, book-detail, book-list, catalog, contribute, my-profile.
- النماذج: `Book`, `Textbook`, `SchoolBook`, `BorrowRecord`.
- تصفّح + بحث + إعارة للطلاب، CRUD للأدمن + تقارير دوران.
- لكل مدرسة عبر `SchoolBook` مع الاستفادة من كتالوج `Book` / `Textbook` العام.

## 13 — النقل · `مُنفَّذ`

سائقون، مركبات، مسارات، صعود geofence، إشعارات الأهل. أحدث ملحمة — مدمجة 2026-04 (PR #291).

- 7 نماذج Prisma (`transportation.prisma`): `Driver`, `Vehicle`, `Route`, `RouteStop`, `RouteAssignment`, `Trip`, `TripBoarding`.
- المسارات: drivers, vehicles, routes, trips, assignments, reports, settings.
- ترتيب توقّفات بالسحب والإفلات عبر dnd-kit.
- صعود مرتبط بـgeofence مع إشعارات الأهل عند الصعود / النزول.
- Mapbox + Leaflet لتصوير المسارات.
- تقارير الرحلات: استخدام، ساعات، إحصاءات، معاينة رسوم.
- دليل الكتلة في `src/components/school-dashboard/transportation/CLAUDE.md`.

## 14 — الكتالوج · `مُنفَّذ + تحسين`

مكتبة محتوى تعليمي على مستوى المنصة. راجع [الكتالوج](/en/docs/catalog).

- 22+ نموذج عبر `catalog.prisma`, `curriculum.prisma`, `chapter.prisma`, `lesson.prisma`, `material.prisma`, `book.prisma`, `textbook.prisma`, `video.prisma`, `image.prisma`, `document.prisma`, `assignment.prisma`, `qbank.prisma`, `quiz.prisma`, `bridge.prisma`، إضافة إلى `proposal.prisma`, `enrollment.prisma`.
- **نمط الجسر** (`bridge.prisma`) — المدارس تشترك في كتالوج مشترك وتطبّق `ContentOverride`, `BookSelection`, `SubjectSelection`, `InstructorPreference`.
- 4 مناهج على القرص: SD (السودان، 12 صفاً، مواد ثنائية اللغة)، US (246 مجلد مادة)، IN (246)، FR.
- المخزون المستهدف: 62 مادة، 201 فصل، 986 درساً.
- لوحة المشغّل في `/[lang]/(saas-dashboard)/(catalog)/catalog/...`.
- تدفّق مساهمة مجتمعية مع `Proposal` (proposed → reviewed → approved → published).
- مكتبة US المصدر: [نظرة عامة](/en/docs/us-curriculum) — 62 مادة، 240 سجل كتالوج خاص بالصف، ثلاث مراحل (ابتدائي/متوسط/ثانوي).
